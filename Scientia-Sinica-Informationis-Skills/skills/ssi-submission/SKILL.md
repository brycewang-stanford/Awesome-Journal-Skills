# 《中国科学：信息科学》投稿审计（Submission）

在把稿件上传到《中国科学：信息科学》(SCIENTIA SINICA Informationis, SSI) 投稿系统之前，
按本清单端到端过一遍。SSI 是中国科学院与国家自然科学基金委员会共同主办、《中国科学》杂志社
出版的国家级大信息学科综合中文旗舰，定位高、以重大原创与评述为主。以下条目均据官网
(scis.scichina.com)《投稿须知》于 2026-07-09 核对（经搜索渲染，双源交叉核实），
凡随周期变动或未双源确认者标 **待核实**，投稿前请重新打开官网当期须知。

## 一、投稿系统与文件准备

- 访问 SSI 官网，注册"作者账户"，按提示填写投稿信息，将**全文 PDF** 上传至数据库服务器。
- 稿件要求用 **LaTeX** 排版，官方模板从官网"下载中心"获取。注意官方模板可能使用过时的
  CCT 中文支持，在现代 TeX 发行版可能无法直接编译，需社区修订版（非官方，仅排版参考）。
- 通讯作者信息、各作者单位**全称** + 所在城市 + 邮政编码务必准确。
- 基金资助项目名称与**编号**齐全（这是本刊与国家自然科学基金委关系密切的著录重点）。

## 二、栏目自选（必须与稿件类型匹配）

| 栏目 | 篇幅/机制 | 是否适合你的稿件 |
| --- | --- | --- |
| 评述 | 约 20 页，一般编委邀请/提议 | 有观点的系统综述 + 未来方向；投前宜与相关编委沟通 |
| 论文 | 主体，无固定短限 | 完整原创研究：科学思想 + 创新点 + 实验/理论依据 + 应用前景 |
| 快报 | ≤4 页，优先发表 | 重要且亟需优先发表的核心成果，4 页内自洽 |
| 学术介绍 | 介绍性 | 客观介绍院校/实验室/重大课题 |

- 选错栏目是常见退修原因：增量结果投评述会被降格；重要快讯写成长论文会拖慢发表。

## 三、中英文双语摘要与关键词

- SSI 要求**中、英文双语摘要**。中文摘要概括目的、方法、结果与结论；**英文摘要须可独立阅读**，
  是国际可见性关键，不能是中文的机翻残句。
- 中英文**关键词**齐备，术语与正文一致。
- 摘要不含未在正文出现的缩写与引用，不堆套话（"随着…的发展"）。

## 四、格式与规范细节

- **量与单位**采用法定计量单位；变量斜体、单位正体、矢量/矩阵按规范。
- **图表**清晰、编号连续、图注自足；中文图表标题规范；避免正文与图表重复陈述数据。
- **参考文献**著录格式统一（是否严格按 **GB/T 7714** 以当期须知为准，**待核实**）；
  中外文文献均需完整著录，避免只引英文忽略国内相关工作。
- **中图分类号**是否强制**待核实**；若须提供，按《中国图书馆分类法》信息科学相关类目填写。

## 五、大信息学科的稿件适配

SSI 覆盖计算机、控制、通信、微电子等多个子学科，评审专家可能来自相邻而非同一子方向，
因此稿件要对"跨子学科可读性"负责：

- **术语首现加注**：子方向专有术语（如体系结构、鲁棒控制、信道编码、EDA）首次出现给出
  中文全称与必要英文缩写，避免只有本子方向读者能懂。
- **贡献一句话可懂**：用一两句让非本子方向的信息学科同行理解创新点与意义。
- **动机高于技巧**：SSI 看重科学价值与创新高度，先讲"为什么重要、新在哪里"，再展开技术细节。
- **应用前景落地**：论文栏目要求交代应用前景，避免纯技巧堆叠而无学科意义。

## 六、伦理与学术规范

- **一稿多投/重复发表**：不得同时投多刊；尤其注意 SSI（中文）与英文姊妹刊
  **Science China Information Sciences (SCIS)** 为相对独立的两个刊物，应**择一投稿**，
  中英文互转/两投政策**以编辑部当期说明为准（待核实）**——不可自行中英文一稿两投。
- **署名与贡献**：作者排序、通讯作者、贡献声明清晰；无挂名、无遗漏。
- **利益冲突与基金**：如实声明利益冲突与资助来源。
- **数据/图像真实性**：不篡改、不选择性报告；引用他人成果规范标注。
- **生成式 AI 使用披露**：政策**待核实**，如使用须按当期要求声明。

## 七、退稿/退修高风险项排查

| 审计发现 | 严重度 | 处理 |
|---|---|---|
| 栏目选错（增量结果投评述、长文投快报） | 退修级 | 改栏目或重构稿件 |
| 英文摘要缺失或质量差 | 退修级 | 重写英文摘要至可独立阅读 |
| 基金编号/单位全称缺失 | 编辑退修 | 补全著录信息 |
| 一稿多投 / 中英文两投嫌疑 | 学术规范风险 | 撤回一处，向编辑部说明并获同意 |
| LaTeX 无法编译 / 未用官方模板 | 技术退修 | 用官方/社区模板重排 |
| 创新高度不足、仅增量刷点 | 科学价值不达标 | 补强贡献，或改投兄弟刊（计算机学报等） |
| 参考文献著录不统一、漏引国内工作 | 退修 | 统一格式，补齐中外文献 |

## 八、投稿周操作顺序

1. 早冻结正文论证；参考文献可微调，主张不可动。
2. 用官方/社区 LaTeX 模板编译出干净 PDF，逐页目视检查图表与公式。
3. 打磨**中英文双语摘要**与关键词，确保英文摘要独立可读。
4. 核对单位全称、城市、邮编、基金编号、作者排序与通讯作者。
5. 在投稿系统内**如实自选栏目**，填全所有字段，声明利益冲突与资助。
6. 上传后**重新下载已上传的 PDF** 冷读一遍，确认是最终版本。

## 快报（≤4 页）专项检查

若投快报栏目，额外确认：

- 全文（含图表、参考文献）严格控制在 **4 页**以内，信息密度高、无冗余背景。
- 只保留支撑核心结论的关键证据；细节与扩展留待后续完整论文（须避免与本刊或他刊重复发表）。
- 说明成果的**时效性与重要性**，以支撑"优先发表"的定位。
- 中英文摘要仍需齐备，但更精炼。

## 九、每次投稿前重新核实

- 版面费金额、是否分栏目差异化（**待核实**）。
- 参考文献是否严格 GB/T 7714、中图分类号是否强制（**待核实**）。
- 三审三校的轮次与外审机制、AI 使用披露政策、数据/代码是否强制公开（**待核实**）。
- 中英文姊妹刊择一投稿/互转的当期细则（**待核实**）。

```text
[SSI 投稿状态] 就绪 / 阻塞 / 待完善
[栏目] 评述/论文/快报/学术介绍——是否与稿件类型匹配？
[排版] LaTeX 官方模板？PDF 可编译？图表/公式规范？
[摘要] 中英文双语齐备？英文摘要可独立阅读？
[规范] 基金编号/单位全称/参考文献/中图分类号(待核实)？
[伦理] 一稿多投/中英文两投风险？利益冲突已声明？
[待办队列] <按优先级排序，含负责人与时间>
```
