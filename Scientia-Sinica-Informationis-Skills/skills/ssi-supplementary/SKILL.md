# 《中国科学：信息科学》附录与补充材料（Supplementary）

一篇投往《中国科学：信息科学》(SCIENTIA SINICA Informationis, SSI) 的稿件，常需在**正文、
附录、补充材料/多媒体附件**之间分配内容。核心原则只有一条：**决定录用的证据必须在正文里**，
补充材料是给愿意深入的读者与复核的审稿人看的，不能承载评审据以判断科学价值的关键论证。
本刊各栏目篇幅约束不同（评述约 20 页、快报 ≤4 页），划分策略也随之不同。具体附件格式与容量
**以官网当期《投稿须知》为准（待核实）**。

## 一、划分的第一原则：按"决定录用性"分

问自己：**如果审稿人只读正文、不看补充，能否判断本文的核心贡献成立？**

- **能** → 划分合理。
- **不能** → 有决定录用的证据被放到了补充材料，必须移回正文。

绝不能把"证明创新点成立的关键实验/推导"藏进补充材料来规避篇幅或凑数。

## 二、各内容的归属

| 内容 | 建议归属 | 说明 |
|---|---|---|
| 核心方法与主结果 | 正文 | 决定贡献成立，必须在正文 |
| 主实验对比与消融 | 正文 | 支撑主张，正文呈现 |
| 关键定理与证明思路 | 正文；完整证明可入附录 | 思路在正文，冗长推导可移附录 |
| 完整推导 / 引理证明 | 附录 | 支撑但不打断正文论证 |
| 额外实验、超参表、更多数据集结果 | 补充材料 | 增强说服力但非决定性 |
| 数据集细节、预处理脚本 | 补充材料 / 可复现包 | 服务复现 |
| 演示视频、系统运行录像、音频 | 多媒体附件 | 直观展示，非核心证据 |
| 大型表格/图集 | 补充材料 | 避免正文臃肿 |

## 三、附录的用途

- 承载**冗长但必要**的内容：完整证明、推导细节、算法伪代码全量版、符号表。
- 附录仍是正式发表内容的一部分，需同等严谨、编号规范（附录 A、附录 B…）。
- 正文在相应位置**明确引用**附录（"完整证明见附录 A"），不留断链。

## 四、补充材料

- 补充材料通常随投稿系统上传，或以在线附件形式提供；**格式与容量以当期须知为准（待核实）**。
- 内容：额外实验、扩展数据、更多案例、细节参数、数据/代码可用性材料。
- 补充材料应**自足可读**（有必要的说明），但读者不读它也不影响理解正文结论。
- 与可复现包配合，见 [`../ssi-reproducibility/SKILL.md`](../ssi-reproducibility/SKILL.md)。

## 五、多媒体/演示附件

- 系统类、机器人/控制类、可视化类成果可提供演示视频或交互附件，直观展示效果。
- 多媒体是**佐证与展示**，不是核心证据；正文仍须有可量化的实验/理论支撑。
- 注意匿名/保密：视频中避免泄露涉密信息、单位标识（若评审有匿名要求，**待核实**）。
- 附件命名规范、体积可控；提供简短说明文档（附件里放什么、如何查看）。
- 若多媒体依赖特定播放器/格式，优先选用通用格式（mp4/pdf），降低审稿人打开成本。
- 见刊后多媒体是否随正式版一并存档，以编辑部说明为准（**待核实**）。

## 六、按栏目差异化

| 栏目 | 补充材料策略 |
|---|---|
| 评述（约 20 页） | 正文容量大，可将扩展文献表、方法分类细表放补充；核心观点与判断在正文 |
| 论文 | 主结果+主消融在正文，完整推导入附录，额外实验入补充 |
| 快报（≤4 页） | 正文极简，只放核心证据；细节与扩展几乎全部依赖补充材料/后续完整论文 |

- 快报尤其考验取舍：4 页内讲清核心，其余放补充，但**核心结论证据仍须在 4 页正文内**。

## 七、常见失误

| 失误 | 后果 | 纠正 |
|---|---|---|
| 决定性证据放补充材料 | 审稿人据正文无法判断贡献 | 移回正文 |
| 正文塞满可移出的推导 | 论证被打断、超篇幅 | 冗长推导入附录 |
| 补充材料与正文不一致 | 显得草率、被质疑 | 交叉核对编号与数据 |
| 附录引用断链 | 读者找不到内容 | 正文明确指向附录编号 |
| 多媒体当核心证据 | 缺乏可量化支撑 | 正文补量化实验 |

## 八、自查表

- [ ] 只读正文即可判断核心贡献成立。
- [ ] 完整证明/冗长推导已移附录，正文保留思路。
- [ ] 补充材料自足、与正文一致、编号清晰。
- [ ] 多媒体附件仅作展示，核心证据在正文。
- [ ] 快报严格控制在正文页数内，核心证据未外溢。
- [ ] 补充材料格式/容量符合当期须知（**待核实**）。

## 九、划分决策的三个反面案例

以下都是"把决定录用的证据错放到补充材料"的典型，需纠正：

- **案例一（错）**：论文主张方法在真实数据集上显著优于基线，但唯一的公平对比表放在补充材料，
  正文只给自家方法的曲线。**纠正**：主对比表必须进正文——它决定贡献是否成立。
- **案例二（错）**：一个"可证明时延上界"的理论主张，正文只给结论，全部推导（含关键引理）在补充。
  **纠正**：证明思路与关键引理入正文/附录，正文须让审稿人相信结论可证，而非仅"相信作者"。
- **案例三（对而更好）**：正文给主结果 + 主消融；6 个额外数据集、超参搜索全表、失败案例分析
  放补充材料。**这是正确用法**：补充增强说服力，但不承载决定性论证。

判断口诀：**"抽掉补充材料，正文的贡献是否还站得住？"** 站得住才算划分正确。

补充说明：审稿人不一定会逐页翻阅补充材料，尤其在初审与首轮外审阶段；因此凡是希望审稿人
"一定看到"的证据，都应放在正文而非寄望于补充材料。补充材料更适合承载"想深入的读者才需要"
的扩展内容。见刊后补充材料是否随正式版长期存档，以编辑部说明为准（**待核实**）。

## 十、每次核实

- 补充材料/多媒体附件的格式、容量、上传方式（**待核实**）。
- 是否有匿名评审对附件的要求（**待核实**）。
- 各栏目对附录/补充的具体限制（**待核实**）。
- 与英文姊妹刊 Science China Information Sciences (SCIS) 对补充材料要求的差异（**待核实**）。

## 输出格式

```text
[划分检验] 仅读正文能否判断贡献成立？能/不能（不能则移回）
[归属] 核心→正文 / 完整证明→附录 / 扩展→补充 / 演示→多媒体
[栏目] 评述/论文/快报——划分策略是否匹配篇幅
[一致性] 补充与正文编号/数据一致？附录引用无断链？
[待核实] 附件格式/容量、匿名要求
```
