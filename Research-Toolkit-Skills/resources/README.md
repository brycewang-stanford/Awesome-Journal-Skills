# Research Toolkit Resources

This toolkit is intentionally venue-neutral. It does not vendor a `code/` kit or
an `official-source-map.md` because live venue facts belong in the selected
journal pack's own `resources/official-source-map.md`, and reusable methods live
once in the repository-level shared resources.

## Canonical Shared Resources

- `shared-resources/journal-selection/journal-match.md` and
  `shared-resources/journal-selection/venue-index.tsv` define the journal-match
  and resubmission-ladder surface.
- `shared-resources/empirical-methods/execution-with-mcp.md` maps empirical
  designs to StatsPAI / Stata MCP execution and audit tools.
- `shared-resources/empirical-methods/replication-package.md` defines the
  Data-Editor package manifest and validation checklist.
- `shared-resources/submission-readiness/readiness-checklist.md`,
  `shared-resources/submission-readiness/simulated-referee.md`, and
  `shared-resources/submission-readiness/response-to-referees.md` define the
  cross-journal readiness, review rehearsal, and R&R response loops.

## Validation Boundary

The toolkit skills should remain compact entry points. They must route to the
shared resources and to the target journal pack instead of duplicating live
submission facts, fee schedules, editor rosters, or code snippets locally.
