# 《自动化学报》选题与栏目定位

在动笔前，用本技能判断选题是否契合《自动化学报》(Acta Automatica Sinica, AAS)。AAS 由中国自动化
学会与中国科学院自动化研究所共同主办、创刊于 1963 年，是控制与自动化学科的中文旗舰月刊。选错刊或
选错栏目，会在编辑初审阶段就以"不符宗旨/无新意/重复前人"被退。以下口径于 2026-07-09 核验（见
`resources/official-source-map.md`），易变项标 **待核实**。

## 一、本刊学科边界（是否对口）

本刊聚焦**控制理论与控制工程、模式识别与智能系统、机器人、人工智能在自动化中的应用**，具体范围含：

| 方向 | 典型主题 |
|---|---|
| 自动控制 | 自适应/鲁棒/最优/预测控制、稳定性、多智能体一致性 |
| 系统理论与系统工程 | 建模、辨识、复杂系统、优化与决策 |
| 自动化工程技术与应用 | 过程控制、运动控制、工业自动化落地 |
| 机器人 | 运动规划、感知、协同、人机交互 |
| 人工智能与智能控制 | 强化学习控制、智能优化、学习型系统 |
| 模式识别与图象处理 | 分类、检测、跟踪、图像/视频分析 |
| 信息处理与基于网络的自动化 | 网络化控制、信息服务、分布式系统 |

**判断准则**：稿件的核心贡献应落在"控制/自动化/智能系统"这条主线上。若贡献本质是纯算法/纯系统
软件/纯电路，则更契合兄弟刊。

## 二、与兄弟中文刊的分工

| 期刊 | 侧重 | 与本刊边界 |
|---|---|---|
| 《自动化学报》(AAS) | 控制、模式识别、机器人、智能系统 | 本刊；贡献落在控制/自动化 |
| 《计算机学报》(CJC) | 计算全学科综合 | 纯计算理论/体系结构改投 CJC |
| 《软件学报》 | 软件工程/系统软件/程序语言 | 软件方法学改投软件学报 |
| 《电子学报》 | 电子、通信、器件、信号 | 硬件/通信物理层改投电子学报 |
| 《计算机研究与发展》 | 计算机综合、系统与应用 | 计算系统类改投 |
| 《计算机科学》 | 计算机综合、面广 | 一般计算主题改投 |

同一份多智能体/强化学习工作，若强调**控制律与稳定性**投本刊，若强调**分布式系统实现**则更像计算刊。

## 三、三类栏目如何选

| 栏目 | 定位 | 适用 |
|---|---|---|
| **综述** | 系统梳理 + 批判性评价某方向 | 对某控制/识别方向有全局把握、能提出研究议程 |
| **论文** | 完整原创研究长文 | 方法 + 理论分析 + 仿真/实物实验齐备 |
| **短文** | 篇幅小、贡献聚焦 | 单点创新、结论明确但体量有限 |

综述稿不是文献堆砌，要有分类框架、对比表、未解问题与趋势判断；论文稿要有"问题形式化—理论分析—
实验验证"三段闭环。

## 四、创新性自检（对抗初审退稿理由）

初审常见退稿理由：不符宗旨、无新意、重复前人、观点模糊、逻辑混乱、学术不端。逐条自检：

1. **贡献可数**：能否用 1~3 条陈述？每条是否可被实验或理论验证？
2. **delta 明确**：相对最接近的已有工作，新增了什么（新模型/新证明/新约束/新场景）？
3. **对口本刊**：贡献主线是否在控制/自动化/智能系统上？
4. **证据匹配**：理论贡献有证明，方法贡献有仿真/实物实验与公平基线。
5. **非增量堆叠**：不是"把 A 换成 B 再跑一遍"式的微增量。

## 五、选题风险分诊

| 症状 | 风险 | 处置 |
|---|---|---|
| 贡献本质非控制/自动化 | 不符宗旨 | 改投对口兄弟刊 |
| 只有仿真无理论、或只有理论无验证 | 深度不足 | 补齐理论分析或实验 |
| 相对已有工作 delta 模糊 | 无新意 | 重定位贡献或换角度 |
| 综述写成文献罗列 | 观点模糊 | 建分类框架 + 批判评价 |
| 主题太宽/太散 | 逻辑混乱 | 收敛到单一核心问题 |

[栏目] 综述 / 论文 / 短文 —— 依据：___

[兄弟刊排除] 为何不投 CJC/软件/电子学报：___

## 七、常见误区

- 把控制味不足的纯算法工作硬投本刊——初审即退；先判断贡献主线。
- 综述选题过大（"人工智能综述")无法在有限篇幅讲透——收敛到具体方向。
- 追热点但无自己的控制/系统视角——本刊看重方法与理论增量而非蹭词。
- 把《自动化学报》(AAS) 与英文《IEEE/CAA Journal of Automatica Sinica》混为一刊选题——二者语种、
  体例不同，先确认目标是中文 Acta Automatica Sinica。

## 八、从热点到可发表选题的收敛

热点方向（如强化学习控制、多智能体、具身智能）竞争激烈，需收敛到有自己增量的具体问题：

```text
[大方向] 强化学习控制
[收窄] 含约束的强化学习控制
[具体问题] 状态/输入约束下保证安全的策略学习与稳定性
[本文角度] 用控制理论工具（如控制屏障函数）给出安全保证 ← 这是本刊看重的控制视角
```

本刊看重的是**控制/自动化视角的贡献**，而非单纯套用热门模型；有自己的理论或系统增量才易被接受。

## 九、选题与证据的匹配预判

选题时就预判所需证据，避免写到一半发现无法支撑：

| 贡献类型 | 需要的证据 | 若缺 |
|---|---|---|
| 理论贡献（新控制律/新定理） | 严格稳定性/收敛性证明 | 补证明或降级为经验方法 |
| 方法贡献（新算法） | 公平基线 + 充分实验 + 消融 | 补对照实验 |
| 应用贡献（新系统落地） | 实物/真实场景验证 | 补实物或高保真仿真 |
| 综述贡献 | 系统检索 + 分类 + 批判 | 补检索策略与 taxonomy |

## 十、与其他技能衔接

选题定稿后，进入 `aas-workflow` 排期，用 `aas-related-work` 坐实 delta，用 `aas-writing-style`
落体例，用 `aas-experiments` 搭证据，最后经 `aas-submission` 完成投稿审计。始终以《自动化学报》
(Acta Automatica Sinica, AAS) 的控制/自动化/智能系统定位为锚，确认目标是中文刊而非英文 JAS。
