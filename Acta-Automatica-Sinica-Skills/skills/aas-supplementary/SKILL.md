# 《自动化学报》附录与补充材料组织

本技能帮你决定投向《自动化学报》(Acta Automatica Sinica, AAS) 的稿件中，哪些内容留在正文、哪些
移入附录或补充材料。AAS 有综述长文传统但正文仍应紧凑，冗长证明与额外实验宜移入附录。以下方法于
2026-07-09 核验框架（见 `resources/official-source-map.md`）；补充材料具体形式以官网当期为准
（**待核实**）。

## 一、划分原则：决定性

判断标准是**"读者理解主贡献是否必需"**：

| 内容 | 位置 | 理由 |
|---|---|---|
| 主定理与核心证明思路 | 正文 | 理解贡献必需 |
| 冗长的完整证明推导 | 附录 | 支撑但不打断主线 |
| 关键实验与主结果 | 正文 | 支撑贡献 |
| 补充仿真、额外场景、参数扫描 | 附录/补充 | 增强但非核心 |
| 系统模型完整推导 | 视情况 | 核心入正文，细节入附录 |
| 大表、超参全表 | 附录 | 正文放摘要版 |

## 二、正文应保留什么

- 问题形式化、核心方法/控制律、主定理与稳定性结论。
- 最能支撑可数贡献的实验：主对比、关键消融、代表性响应曲线。
- 让外审在正文内就能判断创新性与正确性。

## 三、宜移入附录/补充的内容

- 完整证明的每一步推导、引理的技术性证明。
- 额外被控对象/数据集的补充实验、更多参数扫描、鲁棒性细节。
- 超参数完整列表、更多可视化、算法伪代码全版。
- 实物实验的更多细节、标定过程。

## 四、补充材料与多媒体附件

- 若有视频（机器人实验、控制过程演示）、代码、数据等，按补充材料/附件组织。
- 文件**命名清晰、与正文引用对应**（如"附录A 证明""附件S1 演示视频"），正文处给出指引。
- 附件同样需**匿名**：文件名、元数据、视频画面不得泄露作者身份（双盲，见 `aas-submission`）。

## 五、正文与附录的引用衔接

- 正文在移出细节处明确标注"详见附录X/补充材料S y"，让读者可追踪。
- 附录内容自成体系、可独立阅读，但不重复正文已述。
- 综述稿的大对比表若过长，可正文放精简版、附录放完整版。

## 六、组织自检清单

```text
[决定性] 每块内容问：理解主贡献是否必需？必需→正文，否则→附录
[正文紧凑] 冗长证明/额外实验是否已移出，主线是否清晰？
[附录完整] 完整证明/补充实验/超参全表是否齐备？
[附件] 视频/代码/数据命名清晰、与正文对应？
[匿名] 附录与附件是否无身份泄露？
[引用衔接] 正文是否指明"详见附录/补充"？
[待核实] 本刊补充材料具体形式与上限：待核实
```

## 七、常见问题

- 把所有证明塞进正文，主线被淹没——移冗长推导入附录。
- 附录成了正文的重复，没新增信息——附录只放正文放不下的细节。
- 补充附件命名混乱、正文无指引——统一命名并在正文标注。
- 附件泄露作者身份（视频含实验室标识）——匿名化处理。
- 综述超长对比表占满正文——正文精简、附录完整。

## 八、综述稿的补充材料特点

综述栏目常有大量对比数据与分类明细，补充材料的组织与论文稿不同：

| 内容 | 位置 | 说明 |
|---|---|---|
| 核心分类框架图/表 | 正文 | 综述的骨架，必须在正文 |
| 完整方法对比大表 | 附录 | 正文放精简版，附录放全表 |
| 各方法参数/数据集清单 | 附录 | 便于读者查阅，不打断正文 |
| 时间线/演化图 | 视情况 | 关键脉络入正文，细节入附录 |
| 检索与纳入标准 | 附录 | 说明综述的文献筛选方法，增强可信度 |

综述的附录还可包含"文献检索策略"（数据库、关键词、时间范围、纳入/排除标准），这是让综述可复现、
显严谨的重要部分。

## 九、控制/自动化稿的典型附录内容

- **完整稳定性证明**：正文给主定理与证明思路，附录给逐步推导与引理证明。
- **额外被控对象仿真**：正文用代表性对象，附录补更多对象/工况验证普适性。
- **参数敏感性完整扫描**：正文给关键结论，附录给完整扫描曲线。
- **实物实验细节**：平台照片（匿名）、标定过程、更多试次数据。
- **算法伪代码全版**：正文给核心步骤，附录给完整伪代码与复杂度分析。

## 十、正文—附录一致性核对

定稿前做一次一致性核对，避免正文引用与附录内容脱节：

```text
[引用一致] 正文每个"详见附录X"都能在附录找到对应内容？
[编号一致] 附录章节/图表/式编号与正文引用一致？
[无重复] 附录未简单重复正文内容？
[无孤儿] 附录每块内容都被正文引用到？
[匿名一致] 附录与附件均无身份泄露？
```

## 十一、补充材料的匿名与提交

- 补充材料随主稿一并在 aas-cn 系统提交（具体入口以系统提示为准，**待核实**）。
- 与主稿一样需**匿名**：文件、元数据、视频画面均不得泄露作者身份。
- 大文件（视频、数据集）如系统受限，说明获取方式（匿名链接），勿指向作者主页。
- 命名清晰、与正文引用编号对应，便于外审逐一对照。
- 退修时若补充材料随之更新，答复信中说明新增/修改了哪些附件，便于外审核对。
- 录用后（见 `aas-camera-ready`）确认最终见刊版与补充材料的引用编号仍然一致。

## 十二、与本刊定位

《自动化学报》(Acta Automatica Sinica, AAS) 允许把工作讲透，但正文应服务于清晰传达可数贡献。用
附录/补充材料承接细节，既满足外审对严谨性的要求，又保持正文可读。与 `aas-experiments`、
`aas-writing-style`、`aas-reproducibility` 联动，确保正文—附录划分一致、复现材料与附录呼应，让
Acta Automatica Sinica 的外审既能看到紧凑主线，又能查到完整证据。
