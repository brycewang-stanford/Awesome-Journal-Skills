# 《自动化学报》投稿全流程编排

本技能把一次《自动化学报》(Acta Automatica Sinica, AAS) 投稿拆成可执行阶段，并在每个节点指明调用
哪个 aas- 技能。AAS 是中国自动化学会与中国科学院自动化研究所共同主办、创刊于 1963 年的控制与
自动化学科中文旗舰月刊，实行**双盲三审制**。以下时间为经验值，官方权威周期以官网《稿件处理流程》
为准（**待核实**）。核验日期 2026-07-09。

## 一、全流程阶段图

```text
选题定位 → 写作与证据 → 匿名化+作者承诺 → 在线投稿 → 编辑初审
   → 双盲外审 → 退修/复审(多轮) → 主编终审 → 录用 → 定稿校样 → 见刊+稿酬
```

## 二、阶段-技能对照表

| 阶段 | 主用技能 | 准出标准 |
|---|---|---|
| 选题定位 | `aas-topic-selection` | 贡献可数、对口本刊、栏目确定 |
| 全流程规划 | `aas-workflow`（本技能） | 里程碑与责任人排期完成 |
| 写作体例 | `aas-writing-style` | 中英文摘要/关键词/中图分类号/GB/T 7714 就位 |
| 相关工作 | `aas-related-work` | delta 明确、代表作覆盖 |
| 实验/仿真 | `aas-experiments` | RQ、公平基线、稳定性分析、消融 |
| 可复现性 | `aas-reproducibility` | 环境/种子/数据固定，复现包 |
| 补充材料 | `aas-supplementary` | 正文/附录按决定性划分 |
| 投稿审计 | `aas-submission` | 双盲、承诺、查重、字段核对 |
| 审稿建模 | `aas-review-process` | 各环节作者杠杆清晰 |
| 退修答复 | `aas-author-response` | 逐条答复 + 修改对照 |
| 录用定稿 | `aas-camera-ready` | 校样、稿酬、排期 |
| 代码数据 | `aas-artifact-evaluation` | 可用性声明现状讲清 |

## 三、里程碑时间线（经验值，非官方承诺）

| 里程碑 | 相对时间 | 关键动作 |
|---|---|---|
| T0 投稿 | 第 0 天 | 匿名稿 + 作者承诺（签名+公章）同时上传 aas-cn 系统 |
| 初审反馈 | T0 后数日~数周 | 编辑判断是否送外审 |
| 外审 | 约 1~3 个月/轮 | 双盲专家评审（**待核实**精确周期） |
| 退修 | 收到意见后 | 按编辑部给定期限完成大/小修 |
| 复审 | 大修常重新送审 | 外审复核修改 |
| 主编终审 | 复审通过后 | 录用与否终裁 |
| 录用→见刊 | 数月 | 定稿校样、排版、见刊 |
| 稿酬 | 见刊后 3 个月内 | 一次性支付，赠样刊 2 本 |

## 四、准入准出闸门（每阶段一句话）

- **投稿前闸门**：跑完 `aas-submission` 审计；双盲无泄露、作者承诺齐、查重达标，方可上传。
- **外审前闸门**：编辑部收到《作者承诺》后才初审；漏交即卡住。
- **退修闸门**：在编辑部给定期限内提交修改稿 + 逐条答复（`aas-author-response`），超期可能作退稿或
  按新稿处理。
- **录用闸门**：主编终审通过后进入定稿（`aas-camera-ready`）。

## 五、常见延误点与规避

| 延误点 | 后果 | 规避 |
|---|---|---|
| 漏交/未盖章的作者承诺 | 卡在初审前，白等 | 投稿即备好签名+公章扫描件 |
| 双盲泄露被退回补正 | 多一轮往返 | 投稿前逐位置匿名化 |
| 退修拖到超期 | 可能退稿/改按新稿 | 收到意见即排期，预留缓冲 |
| 大修改动过大重新送审 | 周期拉长 | 修改聚焦审稿人核心关切 |
| 投错到 ieee-jas 英文系统 | 不受理/延误 | 确认投中文 aas-cn 系统 |

## 六、并行与串行建议

- **可并行**：写作体例、相关工作、实验可并行推进；复现包与补充材料随实验同步整理。
- **必串行**：匿名化+作者承诺 → 投稿 → 审稿 → 退修 → 录用 → 定稿，按闸门顺序推进。
- 综述稿把更多时间放在分类框架与对比表；论文稿把时间放在理论证明与实验闭环。


## 八、按栏目分化的流程侧重

不同栏目在流程上的时间分配不同：

| 栏目 | 前期重点 | 审稿预期 |
|---|---|---|
| 综述 | 分类框架、检索策略、对比表 | 强调全面性与批判性，外审看视野 |
| 论文 | 理论证明 + 实验闭环 | 强调正确性与充分性，可能多轮大修 |
| 短文 | 单点创新讲透 | 篇幅小但要求贡献清晰 |

把选题阶段确定的栏目（见 `aas-topic-selection`）传导到整个排期，避免用论文体例硬写综述或反之。

## 九、跨阶段的一致性维护

一次投稿会产出多份材料，需保持一致：

```text
[术语] 正文/摘要/图表/答复信术语统一
[数据] 正文数字 = 实验记录 = 复现包结果
[贡献] 摘要/引言/相关工作的 delta 口径一致
[匿名] 投稿稿/复现材料/附件均无身份泄露
[版本] 每轮退修后同步更新正文、复现包与补充材料
```

## 十一、与投稿须知的同步纪律

每次进入新阶段前，回到官网当期《投稿须知》《稿件处理流程》核对易变项（周期、费用、模板、AI 披露
政策），凡未双源确认的一律标 **待核实**，不写入面向作者的确定结论。始终确认目标是中文《自动化
学报》(Acta Automatica Sinica) 而非英文 JAS，投稿走 aas-cn 系统。
