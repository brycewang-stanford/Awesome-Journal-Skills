# 《自动化学报》相关工作与文献综述

本技能帮你为《自动化学报》(Acta Automatica Sinica, AAS) 写出**以 delta 为先**的相关工作与综述。
AAS 只发表创新性成果，初审对"重复前人、无新意"很敏感，相关工作是证明创新性的主战场。以下方法于
2026-07-09 核验框架（见 `resources/official-source-map.md`）。

## 一、delta 优先原则

相关工作不是文献罗列，而是**用他人工作反衬本文增量**。每引一组工作，都要落到"他们做了什么、局限
在哪、本文如何突破"。

```text
[主题簇] ___（如：含饱和的多智能体一致性）
[代表作] 文献[a][b][c]
[他们的局限] a: 依赖全局信息；b: 仅局部稳定；c: 未考虑饱和
[本文 delta] 仅邻居信息 + 全局一致性 + 反饱和
```

## 二、文献覆盖策略（控制/自动化方向）

| 层次 | 覆盖什么 |
|---|---|
| 奠基工作 | 该方向经典模型/定理（一致性、Lyapunov 稳定、经典识别方法） |
| 最接近工作 | 与本文问题设定最相近的 3~5 篇，逐一对比 |
| 近两年进展 | 反映当前前沿，含本刊与兄弟刊近作 |
| 交叉与反例 | 相邻方向、可能质疑本文的工作 |

优先引本领域高质量来源；本刊《自动化学报》(Acta Automatica Sinica) 及控制/自动化主流刊的近作应有
适当覆盖，体现对本刊话语体系的了解。

## 三、对比表（强烈建议）

用一张表把本文与最接近工作对齐，列维度可含：系统模型、假设、信息需求、稳定性结论、是否处理约束、
实验验证方式。表末一行是"本文"，直观呈现 delta。

| 工作 | 系统 | 信息需求 | 稳定性 | 约束处理 | 验证 |
|---|---|---|---|---|---|
| 文献[a] | 一阶 | 全局 | 渐近 | 无 | 仿真 |
| 文献[b] | 二阶 | 邻居 | 局部 | 无 | 仿真 |
| 本文 | 二阶 | 邻居 | 全局渐近 | 饱和 | 仿真+实物 |

## 四、综述栏目的特别要求

投综述栏目时，相关工作升级为**全文主体**，需要：

1. **分类框架**：按方法/问题/应用维度建立清晰 taxonomy，而非时间流水账。
2. **批判性评价**：指出各类方法的适用条件、优劣与失效场景。
3. **对比表/图**：横向对比代表性方法的关键属性。
4. **未解问题与趋势**：提出研究议程，这是综述的核心价值。

## 五、创新性自检（对抗"无新意"退稿）

- 能否用一句话说清"本文相对最接近工作新增了什么"？
- delta 是**实质增量**（新模型/新证明/新约束/新场景）还是微调？
- 是否遗漏了会削弱本文新颖性的近作？（遗漏易被外审抓）
- 自引是否克制、必要，且以中性第三人称保双盲？

## 六、引用规范

- 用 **GB/T 7714 顺序编码制**，正文按引用顺序上标编号；著录完整、类型标识规范（见
  `aas-writing-style`）。
- 引用要准确到"他们主张什么"，不曲解、不拉大旗；对被对比工作保持公允。
- 避免只引英文而无视本领域重要中文成果，也避免只堆中文而缺国际前沿。

## 七、相关工作写作工作表

```text
[目标刊] 《自动化学报》(Acta Automatica Sinica, AAS)　[栏目] 综述/论文/短文
[主题簇] 1.___ 2.___ 3.___
[每簇代表作与局限] ...
[对比表维度] 系统/假设/信息需求/稳定性/约束/验证
[本文 delta 一句话] ___
[综述专属] 分类框架? 批判评价? 未解问题?
[自检] 无遗漏削弱新颖性的近作? 自引中性保双盲?
```

## 八、文献检索策略（尤其综述）

系统检索是 delta 可信的前提，综述稿更须写清检索方法：

| 要素 | 内容 |
|---|---|
| 数据库 | CNKI/万方/维普 + Web of Science/IEEE/Scopus |
| 关键词 | 中英文对照的领域术语组合 |
| 时间范围 | 近若干年 + 奠基经典 |
| 纳入/排除 | 主题相关性、质量、可获取性 |
| 追踪 | 高被引文的引文网络（引用与被引） |

综述可把检索策略写进正文或附录（见 `aas-supplementary`），显著提升可信度与可复现性。

## 九、相关工作的位置与篇幅

- **论文栏目**：相关工作可独立成节，也可融入引言；篇幅适中，聚焦最接近工作。
- **综述栏目**：相关工作即全文主体，需完整 taxonomy 与批判评价。
- **短文栏目**：相关工作压缩，只点最关键的对比对象与 delta。

避免相关工作过长喧宾夺主（论文），也避免过短无法支撑创新性主张。

## 十、把 delta 传导到摘要与引言

相关工作确立的 delta 要在摘要、引言贯通一致：

```text
[相关工作] 指出最接近工作的局限 → [引言] 收束为本文 gap → [贡献] 可数陈述 → [摘要] 报道性呈现
```

三处口径一致，外审才能形成清晰的创新性判断，避免"摘要说的贡献在正文找不到对应"的割裂。

## 十一、常见问题

- 相关工作写成"某某做了X，某某做了Y"的清单，无对比、无 delta——改为差异驱动。
- 只夸自己不谈他人局限，或贬低他人失公允——外审反感，需公正。
- 综述无分类框架，沦为文献堆砌——先立 taxonomy 再填内容。
- 忽视《自动化学报》(Acta Automatica Sinica) 与兄弟刊近作，显得不了解本刊语境——适当覆盖本刊话语。
- 遗漏最接近的竞品工作，被外审一击命中——先系统检索再动笔。
- delta 在相关工作、引言、摘要三处口径不一——统一贯通。
