# 制度背景（jfr-institutional-background）

## 触发时机

- 制度背景段笼统、像教科书，没服务识别与机制
- 担心监管口径/市场分层/所有制写错
- 审稿人可能问"你的政策冲击到底改了哪条规则"

## 为什么这是《金融研究》的加分项也是雷区

该刊编委多熟悉中国金融制度，**制度细节写准是基本盘，写错直接出局**。制度背景不是凑字数，而是为**识别外生性**和**机制方向**提供依据。

## 必须讲准的三类制度细节

| 类别 | 易错点 | 要求 |
|------|--------|------|
| 监管规则 | 混淆监管主体（央行/金监总局/证监会）、时点、口径 | 写清规则名、生效时点、约束对象与边界 |
| 市场分割 | 银行间 vs 交易所、A/H/B、信贷 vs 债券 | 明确所研究的是哪个市场、为何分割可用于识别 |
| 所有制与主体 | 国有/民营、国有大行/股份行/城商行/农商行 | 区分主体类型，因其约束与行为不同 |

## 与近年政策冲击对接（供识别用）

把制度段落与可用冲击挂钩，常见者：

- **MPA 宏观审慎评估**：约束对象、考核维度、对信贷投放的影响
- **资管新规（2018）**：打破刚兑、净值化、非标限制——影响表外与影子银行
- **LPR 改革（2019）**：贷款定价基准切换——影响利率传导
- **注册制（科创板 2019 / 全面注册制 2023）**：发行与定价机制变化——影响一级市场与信息
- **存款利率市场化、再贷款/结构性工具、房地产融资"三道红线"** 等

> 写制度段时给出**规则要点 + 生效时点 + 约束对象**三要素，并说明它如何制造准实验变异。具体条款以官方文件为准，不确定处标注而非杜撰。

## 自检清单

- [ ] 监管主体、规则名、生效时点、约束对象准确
- [ ] 市场分层/所有制分类与研究对象一致
- [ ] 制度细节服务于识别外生性（不是背景装饰）
- [ ] 政策冲击的"改了哪条规则"说得清
- [ ] 不确定的条款已核对官方文件或明确列为需复核项

## 微型走查：把"绿色信贷政策"写成可识别的制度段

示意稿《绿色信贷指引对企业污染排放的影响》，制度段如何从"装饰"变"识别依据"（数字均为虚构演示）：

- **弱写法**："近年来国家高度重视绿色金融，出台了一系列绿色信贷政策。" —— 无规则名、无时点、无约束对象，识别无从挂钩。
- **强写法**：锁定 2012 年《绿色信贷指引》（原银监会发布，约束对象=银行业金融机构，要求对环境违规企业限贷）。说明该规则使**高污染行业（两高一剩）企业**在 2012 年后面临外生的信贷收紧，而非企业自身选择，由此构造行业 × 时点的 DID 变异。
- **三要素填表**：规则名=绿色信贷指引；生效时点=2012 年；约束对象=银行→高污染企业信贷。
- **变异来源说明**：政策按行业"两高一剩"名录划线，企业难以瞬时跨行业规避 → 处理强度外生性较强。

> 注意区分发布主体：绿色信贷指引由原银监会（今金监总局前身）系统发布，**不是**人民银行或证监会单独出台；混淆主体是该刊审稿人一眼能看穿的硬伤。具体名录与口径以官方文件为准。

## 审稿人追问模式与本刊语境下的修法

| 审稿人追问 | 背后担心 | 本刊语境修法 |
|------------|----------|--------------|
| "这条监管到底约束谁、改了哪条规则？" | 制度细节空泛，识别站不住 | 补"规则名 + 生效时点 + 约束对象"三要素，并量化覆盖范围 |
| "政策是否内生于被监管主体的游说/预期？" | 政策外生性存疑 | 用"按名录/阈值机械划线 + 企业难瞬时规避"论证外生 |
| "为什么用这个时点，不是更早的征求意见稿？" | 处理时点选择随意 | 区分征求意见稿与正式生效，事件研究检验抢跑 |
| "把所有银行/企业当同质，合理吗？" | 忽略主体异质 | 按所有制/银行类型分层，制度约束本就不同 |

## 校准锚点

《金融研究》已刊论文的制度背景多为**独立小节或并入引言后段**，篇幅约 800–1500 字，通常配一段政策时间线说明，而非长篇政策综述。制度细节服务识别，点到为止。具体篇幅与体例以编辑部最新稿约为准。

## 反模式

- 制度背景写成金融常识科普，与识别无关
- 监管口径/主体张冠李戴（如把资管新规说成证监会单独发布）
- 笼统说"受政策影响"，不指明规则与时点
- 忽略主体异质（把所有银行当同质）
- 把征求意见稿、试点、正式生效三个时点混为一谈

## 输出格式

```
【制度对象】<规则/市场/主体>
【三要素】规则名□ 生效时点□ 约束对象□
【服务识别】是 / 否 <如何制造变异>
【准确性存疑点】无 / <列出需复核项>
【下一步】jfr-identification
```
