# 复现与自查脚本思路 — 《计算机学报》(Chinese Journal of Computers, CJC)

本目录把《计算机学报》(Chinese Journal of Computers, CJC) 作者常用的复现与投稿前自查动作沉淀为
可脚本化的思路。它刻意保持精简：本刊的**政策**（三审制、双盲外审、GB/T 7714、学术不端 30% 红线、
代码数据可用性现状）在各 `SKILL.md` 与 [`../official-source-map.md`](../official-source-map.md)
中；这里只放**跨稿件复用的工具性检查**。

## 一、复现包冒烟检查（思路）

在把复现包随稿提交前，做一次"干净环境冒烟测试"：

```bash
# 1) 从零构建
python3 -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
# 2) 一键复现主结果
bash scripts/run_all.sh
# 3) 核对产物是否对上论文表图（人工比对 results/ 与论文表6/图3）
```

要点：固定随机种子、记录硬件与耗时、把大数据走稳定托管而非塞进仓库（见
[`../../skills/cjc-reproducibility/SKILL.md`](../../skills/cjc-reproducibility/SKILL.md)）。

## 二、投稿前自查清单（脚本化思路）

- **查重**：投稿前用查重工具把总文字复制比压到 30% 红线以下，重点核对自引与会议版本重叠。
- **匿名化**：清理 PDF 文档属性、致谢、基金显名与图源水印；扫描仓库中的用户名、邮箱、机构路径。
  ```bash
  # 示意：检查提交物中可能泄露身份的痕迹
  grep -rniE '(@[a-z0-9.]+\.(edu|ac)\.cn|/home/|/Users/|课题组|实验室)' artifact/ | head
  ```
- **参考文献**：核对 GB/T 7714 著录、中文文献中英文对照、正文序号与文末列表一一对应。
- **六件套**：中英文摘要、关键词、中图分类号(TP)、基金、作者简介、参考文献是否齐全。

## 三、本刊特有的检查（通用工具无法替代）

- **注册+提交同步**：确保系统注册与全文上传相差不超过 1 天。
- **双盲外审下的材料匿名**：不仅正文，补充材料、附件、复现仓库也要去作者身份。
- **会议扩展版声明**：若为会议论文扩展版，显著声明扩展关系并规范引用会议版本、说明实质增量。
- **可用性声明诚实**：与实际提供的制品一致；不可公开部分说明原因与替代核验途径。

本目录不是计量经济学/Stata 套件，CJC 的证据也不是排行榜分数：请勿把 DiD notebook 或榜单式框架
搬进来。政策仍以官网当期《投稿须知》与各技能为准。
