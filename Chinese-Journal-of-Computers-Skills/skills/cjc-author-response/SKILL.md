# 《计算机学报》审稿意见答复与修回说明

《计算机学报》(Chinese Journal of Computers, CJC) 作为 CCF A 类综合性中文月刊，采用编辑初审→
专家外审→复审退修→主编终审的三审制。绝大多数最终录用的长文都经历过**一到多轮退修**：编辑结合
2~3 名外审专家意见做出退修决定，较大修改后通常需要**重新送审**。因此，一封结构清晰、逐条对应、
态度诚恳而有理有据的**答复信 + 修回说明**，往往是决定稿件命运的关键材料。本技能给出面向 CJC 的
答复写法。事实核验日期 2026-07-09，流程细节以系统内退修通知为准，未确认项标 **待核实**。

## 一、先给修回定性

收到退修通知后，第一步是判断它属于哪一档，因为投入策略不同：

| 退修类型 | 典型信号 | 应对基调 |
|---|---|---|
| 小修 (Minor) | 仅文字、格式、个别澄清；不再送外审 | 快速精准补正，避免引入新问题 |
| 退修/大修 (Major) | 要求补实验、补证明、重构论证；需重新送审 | 系统性修改 + 完整逐条答复，按"再审"标准准备 |
| 有条件再议 | 专家分歧明显、编辑要求二选一路线 | 先厘清核心争点，给出可验证的证据回应 |

判断依据主要看：是否明确要求"修改后重新送审"、外审评分与结论、编辑退修信的措辞强度。

## 二、答复信的总体结构

面向 CJC 主编与外审专家的答复信建议采用如下结构：

1. **致谢与总述**：感谢编辑与各位审稿专家，用 3~5 句概述本轮主要修改（新增了什么实验/证明/章节），
   让终审的主编一眼看到修改的体量与方向。
2. **逐条对应回复**：把每位审稿人的每一条意见**原文引用**，紧跟"回复"与"修改位置"。
3. **修改稿差异标注**：正文中用颜色或修订标记标出改动，并在答复中指明页码/章节号，方便复审比对。
4. **分歧调和说明**：若两位专家意见冲突，说明你如何权衡并给出折中或二者兼顾的方案。

## 三、逐条回复的标准模板

对每一条意见，使用统一三段式，做到"引用—回应—定位"闭环：

```text
【审稿人 X · 意见 n】（原文引用审稿意见）
【回复】我们同意/部分同意/审慎说明……（先表态，再给理由与证据）
【修改】已在正文第 __ 页第 __ 节 / 图 __ / 表 __ 处作如下修改：……（可引用改后关键句）
```

要点：
- **先表态后论证**。同意就落实并指明改处；不同意要给数据、引证或推理，语气克制、对事不对人。
- **每条都要有可核验的落点**。空回复"已修改"而不指明位置，会拖慢复审、损害信任。
- **补做的实验/证明单列**。新增结果给出表号图号，并在答复信中摘录关键数值，便于专家快速确认。

## 四、面向"重新送审"的准备

CJC 大修通常重新送审，且**多为原审稿人复审**，因此：
- 修改稿要能**独立自洽**：复审专家会重读全文，不要依赖只在答复信里出现的解释。
- 把答复信中承诺的每一处改动都真正落到正文，避免"信里说改了、文中没改"的致命不一致。
- 对未采纳的意见，给出经得起再审推敲的理由，而不是回避；专家最反感被无视。

## 五、答复信写作纪律

- **通篇中文、术语统一**：与正文保持一致的中文术语与符号，英文缩写首次出现给全称。
- **不夹带情绪**：即便认为审稿人误读，也以"可能是我们表述不清，已澄清如下"的姿态处理。
- **不夸大改动**：如实说明做了什么、没做什么以及为什么；主编终审看重诚实与增量。
- **控制篇幅**：答复信服务于快速核验，长度与修改量匹配即可，不堆砌客套。

## 六、常见失误对照

| 失误 | 后果 | 纠正 |
|---|---|---|
| 只笼统说"已按意见修改" | 复审无法定位，反复拉锯 | 每条给页码/章节号与改后关键句 |
| 与审稿人争辩语气强硬 | 激化对立，终审印象差 | 对事不对人，用证据说话 |
| 答复承诺与正文不符 | 再审发现不一致，直接减分 | 提交前逐条比对信与稿 |
| 忽略某位专家某条意见 | 被判定敷衍，可能再退修 | 建立意见清单，逐条勾销 |
| 新增实验只在信里报数 | 专家无法核验 | 结果写入正文表图，信中摘录 |

## 七、修回时限与流程衔接

- 按退修通知给定的**修回期限**提交；确需延期应提前向编辑部说明。
- 大修重新送审后，可能进入第二轮甚至更多轮退修，直至满足录用标准后由主编终审定夺。
- 录用后进入定稿与校样阶段，交由 `cjc-camera-ready` 处理。

## 八、一个完整回复样例（供仿写）

```text
【审稿人 2 · 意见 3】文中方法只与两个较早的基线比较，缺少与近两年主流方法的对比，
  难以判断其先进性。
【回复】感谢专家指出。我们同意仅比较早期基线不足以体现先进性。为此，我们补充了 2023—2024 年
  三个代表性方法(文献[21][27][33])在相同数据划分与评测协议下的对比实验，并统一了算力条件。
【修改】已在第 12 页第 5.2 节新增表 6，报告新增三个基线的结果(均值±标准差，5 次运行)；
  在第 13 页新增一段分析(第 5.2 节第 3 段)讨论本文方法在其中 4 个指标上显著更优
  (配对 t 检验, p<0.05)，并说明在 1 个指标上持平的原因。
```

样例要点：先明确表态→说明补做了什么→精确指出新增表号、页码、章节与统计口径，让复审专家能在
一分钟内核验。

## 九、与编辑部沟通礼仪

- 一切正式沟通走投稿系统或编辑部邮箱 `cjc@ict.ac.cn`，措辞正式、简洁、对事不对人。
- 确需延长修回期限，**提前**说明理由并给出可行的新时间，不要临近截止或逾期后才联系。
- 不催稿、不越级；对编辑与匿名审稿人始终保持尊重，这也是《计算机学报》(Chinese Journal of
  Computers, CJC) 学术共同体的基本礼仪。
- 若对某条意见确有异议且事关结论，可在答复中据理说明，但保留最终由编辑与主编裁量的姿态。

## 十、修回提交前的终检

- [ ] 每位审稿人每条意见都已逐条回应、无遗漏？
- [ ] 每条"已修改"都指明了页码/章节/图表号？
- [ ] 答复信承诺与正文改动逐条一致？
- [ ] 新增实验/证明已写入正文并在信中摘录关键结果？
- [ ] 修改稿能独立自洽（不依赖只在信里出现的解释）？
- [ ] 差异标注(修订色/标记)清晰，便于复审比对？

## 输出格式

```text
[修回类型] 小修 / 大修(重新送审) / 有条件再议
[修改总述] 本轮新增：实验__ / 证明__ / 章节__（供主编终审速览）
[逐条清单] 审稿人×意见n → 表态 / 证据 / 修改位置(页/节/图/表)
[分歧处理] 冲突意见如何权衡：___
[一致性核验] 答复承诺与正文改动逐条比对：通过/待改
[时限] 修回截止：___；是否需延期沟通：是/否
```
