# 《计算机学报》投稿就绪审计

在把稿件上传到《计算机学报》(Chinese Journal of Computers, CJC) 在线投稿系统之前，先跑完这份
审计。CJC 是由中国计算机学会(CCF)与中国科学院计算技术研究所主办、创刊于 1978 年的 CCF A 类
计算机学科**综合性**中文旗舰月刊，以高水平原创**长文**为主。它按期刊稿件走三审制，因此投稿被当作
一篇要经受编辑初审、专家外审、主编终审的完整学术论文来评判，而不是一次会议式的一轮定稿。以下事实于
2026-07-09 通过官网、CNKI 期刊主页与 CCF 推荐中文科技期刊目录交叉核验（见
`resources/official-source-map.md`），凡本刊未公开确认的数字均标注 **待核实**，请以投稿系统内
最新《投稿须知》为准。

## 一、投稿系统与账号

- 本刊**只接受网上投稿**，投稿入口为官网 `http://cjc.ict.ac.cn/`；编辑部邮箱 `cjc@ict.ac.cn`。
- **注册登记与稿件提交须同步**：官方要求二者时间相差不超过 1 天。先在系统内注册、随后拖延数日再
  上传全文，可能造成稿件状态异常。请把注册与上传安排在同一个工作时段完成。
- 注册时填写的题名、作者、单位、通信作者与研究方向决定编辑分派外审专家的匹配度，务必用**真实终稿
  信息**，不要用占位标题。
- 稿件注册后一般 1 周左右会收到编辑部邮件反馈初步状态。

## 二、格式与长文篇幅

- 使用本刊**官方模板**（提供 Word 与 LaTeX 两种；LaTeX 模板含示例论文与编译说明）。中文正文按
  A4 幅面排版，提交 PDF（必要时附源文件）。
- CJC 属一级学报，**篇幅偏长文**：一篇典型研究论文通常远超会议短文的容量，允许把方法、证明、系统
  设计与充分实验完整展开。请把这理解为"给足空间把工作讲透"，而非"凑字数"。具体页数/字数上限以模板
  与编辑部要求为准（**待核实**具体数值）。
- 稿件类型可为：论文、技术报告、短文、研究简报、综述。选定类型后按对应体例撰写。

## 三、中文科技论文的硬性组件

《计算机学报》的稿件必须齐备以下中文科技期刊的标准部件（详见 `cjc-writing-style`）：

| 组件 | 要求 |
|---|---|
| 中文题名 + 英文题名 | 英文题名与中文一致、可独立检索 |
| 中文摘要 + English Abstract | 采用**报道性摘要**：问题与目的、方法、结果、结论四要素齐全 |
| 中文关键词 + Keywords | 一般 3~8 个，中英文一一对应 |
| 中图分类号 | 计算机类以 **TP** 打头（如 TP18、TP311、TP393），据主题选取 |
| 作者简介与基金项目 | 注明资助项目及编号；作者简介含研究方向 |
| 参考文献 | **GB/T 7714**；中文文献需给出**中英文对照**，著录完整 |

## 四、学术规范与伦理红线

- **学术不端检测**：初审即查重，官方口径为总文字复制比**超过 30% 作退稿或改投处理**。自查时先用
  查重工具压到远低于该线，尤其注意自引与前序会议版本的重叠。
- 若本文是作者已发表会议论文的**期刊扩展版**，必须显著声明扩展关系、指明新增贡献（新增理论、新增
  实验、新增系统模块通常需达到实质比例），并规范引用会议版本，避免一稿多投与自我剽窃。
- 署名、贡献、基金、数据来源须真实；不得伪造、篡改、买卖数据或代写。
- 一稿多投是明确禁止项：投稿即视为该稿未在其他刊物同时评审。

## 五、退稿风险分诊表

| 审计发现 | 风险等级 | 处置 |
|---|---|---|
| 重复率 > 30% | 初审退稿级 | 重写重叠段落、补足原创增量后再投 |
| 未用官方模板 / 缺中英文摘要或中图分类号 | 初审退回补正 | 套用官方模板补齐全部部件 |
| 无新意、重复前人工作、观点模糊 | 初审退稿理由 | 回到 `cjc-topic-selection` 重估贡献 |
| 参考文献不符 GB/T 7714 / 中文文献缺英文对照 | 需补正 | 逐条重排著录格式 |
| 会议扩展版未声明、增量不足 | 学术规范风险 | 显著声明并补足实质新增贡献 |
| 主题超出计算机学科范围 | 不符宗旨 | 改投更契合的专业刊 |
| 一稿多投 | 严重违规 | 撤回其一，确认唯一在审 |

## 六、投稿前最后一轮操作顺序

1. 冻结正文论证主线；参考文献可微调，核心贡献陈述不再改。
2. 用官方模板做一次通排，确认中英文摘要、关键词、中图分类号、基金、作者简介、参考文献六件套齐全。
3. 跑查重，把总复制比压到安全线以下，逐条核对自引与会议版本重叠。
4. 校对图表：图题表题中英文规范、量和单位用法定计量单位、图内文字清晰可辨。
5. 在同一工作时段内完成系统注册与全文上传，保证二者相差不超过 1 天。
6. 上传后重新下载 PDF 冷读一遍，确认提交的是最终版文件、无乱码、无残留批注。

## 七、每轮需复核的易变项

- 具体页数/字数上限、模板版本、审理费与版面费金额（**待核实**，以编辑部通知为准）。
- 是否新增代码/数据可用性声明要求或生成式 AI 使用披露要求（**待核实**）。
- 栏目设置与当期征稿重点、专题(special issue)是否开放。

## 八、图表、公式与源文件提交细节

- **图表**：矢量图优先（可无损缩放），位图保证印刷分辨率；图内文字随正文缩放后仍清晰。图题在图下、
  表题在表上，中英文对照。折线/柱状图在黑白打印下用线型或纹理区分而非仅靠颜色。
- **公式**：用公式编辑器或 LaTeX 排版而非截图；对齐、编号连续；正文引用公式用编号。
- **源文件**：若用 LaTeX，提交前用官方模板本地完整编译一次，确认无缺字体、无未定义引用、参考文献
  正常生成；打包 .tex、.bib、图源与自定义宏，方便编辑部排版。
- **文件命名**：稿件文件名不含作者姓名、单位等身份信息（双盲外审）。

## 九、投稿信(cover letter)与系统字段

- 投稿系统通常要求填写题名、摘要、关键词、领域分类、作者与通信作者信息、基金、推荐/回避审稿人等。
  逐项据实填写；领域分类要与稿件主题精确匹配，直接影响外审专家分派。
- 若填写回避审稿人，仅出于真实利益冲突，并简要说明理由；不要用来排除潜在的严格评审者。
- 投稿信简明说明：本文主要贡献、为何适合《计算机学报》(Chinese Journal of Computers, CJC)、
  是否为会议扩展版及其增量、以及无一稿多投与利益冲突的声明。

## 十、投稿时间线上的常见踩坑

| 场景 | 后果 | 规避 |
|---|---|---|
| 注册后拖几天才上传 | 违反"≤1天同步"，状态异常 | 同一时段完成注册+上传 |
| 临投稿才发现重复率超标 | 被迫延期重写 | 投稿前数天先自查查重 |
| LaTeX 在编辑部环境编译失败 | 排版返工、延误 | 提交前用官方模板干净编译 |
| 领域分类填错 | 分到不对口专家、外审慢 | 精确匹配主题选分类 |
| 图表身份泄露(水印/路径) | 违反双盲 | 提交前清理图源与文件属性 |

## 输出格式

```text
[CJC 投稿状态] 就绪 / 阻塞 / 待完善
[系统] 注册与上传是否同步(≤1天)？是/否
[格式] 官方模板？中英文摘要/关键词/中图分类号/参考文献六件套齐全？
[查重] 总复制比 ___%（红线 30%）
[规范] 会议扩展版是否已声明并声明增量？一稿多投排查通过？
[栏目] 目标栏目/稿件类型：___
[待办队列] <按优先级排序，标注负责人与时限>
```
