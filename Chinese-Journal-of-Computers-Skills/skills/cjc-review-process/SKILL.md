# 《计算机学报》审稿流程

理解《计算机学报》(Chinese Journal of Computers, CJC) 的审稿流程，才能合理安排投稿节奏、预判修回
轮次、把力气用在能改变结果的环节。CJC 是 CCF A 类综合性中文月刊，实行以同行评议为基础的**三审制**，
送外审前删除作者姓名、单位等信息（双盲式外审）。以下流程与时长于 2026-07-09 通过官网与二手期刊
资料交叉核验；周期为**经验值**，个案差异大，未确认项标 **待核实**，以系统内实时状态为准。

## 一、三审制全链路

```text
在线投稿(注册+上传, ≤1天同步)
        │
   ① 编辑初审 ── 不符要求 → 退稿/建议改投
        │ 通过
   ② 专家外审(2~3名同行, 双盲, 约20天/轮)
        │
   ③ 复审/退修决定 ── 小修/大修(常重新送审) ⇄ 作者修回
        │ 满足标准
   ④ 主编终审(每月定期) → 录用 / 退稿
        │ 录用
     定稿 → 校样 → 出版
```

## 二、① 编辑初审：先过形式与门槛

初审由编辑负责，以下情形常被**退稿或建议改投**：
- 不符合本刊宗旨与选题范围（超出计算机学科、或与本刊综合定位不符）；
- 无新意、重复前人工作、观点模糊、逻辑混乱；
- 学术不端检测**总文字复制比超过 30%**；
- 格式严重不符（缺中英文摘要、中图分类号、参考文献不规范等）。

作者杠杆：投稿前用 `cjc-submission` 自审，把形式与查重问题清零，避免在门口被拦。

## 三、② 专家外审：双盲同行评议

- 编辑将稿件送 **2~3 名外审专家**做同行评议，送审前删除作者身份信息。
- 官方口径外审时间原则上约 **20 天/轮**；实际因专家档期常有浮动。
- 外审关注：创新性与贡献增量、方法正确性、实验充分与可信、写作与规范、与已有工作的区分度。

作者杠杆：投稿时把贡献增量、实验可信度、与相关工作的 delta 讲清楚（见 `cjc-writing-style`、
`cjc-experiments`、`cjc-related-work`），直接影响外审判断。

## 四、③ 复审与退修：多轮是常态

- 编辑结合外审意见与本刊规范做退修决定；**较大修改后通常重新送审**，多由原审稿人复审。
- 退修可能不止一轮，直至满足录用标准；这是 CJC 长文走向录用的正常路径，不必因收到大修而气馁。

作者杠杆：用 `cjc-author-response` 写逐条对应、可核验、态度诚恳的答复信，是此环节的胜负手。

## 五、④ 主编终审：定期定夺

- 编辑部/主编**每月定期**对当月稿件做终审，结合外审意见与作者修改情况**最终决定录用与否**。
- 终审看重：修改是否到位、增量是否实质、全文是否自洽可信、是否符合本刊长文标准。

## 六、周期预期（经验值，非承诺）

| 阶段 | 经验时长 |
|---|---|
| 投稿→初审反馈 | 约 1 周 |
| 第 1 轮审理 | 约 3~4 个月 |
| 全流程(含修回) | 约半年上下，个案差异大 |

以上为**报告性经验值**，具体以系统状态为准（**待核实** 官方最新口径）。合理做法：预留充足时间，
不把见刊时间卡在硬 deadline 上。

## 七、各环节作者杠杆速查

| 环节 | 能改变结果的动作 | 对应技能 |
|---|---|---|
| 投稿前 | 清形式、清查重、讲清贡献 | cjc-submission / writing-style |
| 外审 | 增量、实验、相关工作 delta | experiments / related-work |
| 退修 | 逐条对应、可核验答复 | author-response |
| 终审 | 修改到位、全文自洽 | author-response / camera-ready |

## 八、状态与沟通

- 通过在线系统跟踪状态；长时间无更新可礼貌向编辑部咨询，避免频繁催稿。
- 收到退修务必看清**修回时限**与是否"重新送审"，据此定投入强度（见 `cjc-author-response`）。

## 九、读懂外审意见的类型

外审意见通常混杂几类，需分类应对（详细回复见 `cjc-author-response`）：

| 意见类型 | 含义 | 应对 |
|---|---|---|
| 硬伤类 | 方法错误、实验不成立、结论不被支撑 | 必须实质修正或补证，否则难录用 |
| 增量类 | 要求补基线、补消融、补数据 | 尽力补做并写入正文 |
| 澄清类 | 表述不清、定义模糊、符号歧义 | 改写澄清并指明改处 |
| 建议类 | 可选改进、延伸讨论 | 采纳则说明，不采纳给理由 |
| 格式类 | 参考文献、图表、单位、语言 | 按 GB/T 7714 与模板逐条订正 |

判断哪些是"录用前必须解决"的硬伤，优先投入；对建议类量力而行并礼貌回应。

## 十、退稿后的选择

- 若被退稿，先冷静通读意见，区分"选题不符/增量不足"(需重新定位，见 `cjc-topic-selection`) 与
  "工作可救、只是本轮未达标"(可大幅修改后改投)。
- 《计算机学报》(Chinese Journal of Computers, CJC) 与兄弟中文刊(计算机研究与发展、软件学报等)
  定位不同，改投前用选题技能重新评估最契合的落点，并针对性重写，而非原样转投。
- 若确信存在评审误判且事关重大，可通过编辑部正当渠道申诉，但需以证据与事实为据、态度专业。

## 十一、评审阶段的作者纪律

- 保持耐心：三审制 + 多轮修回本身耗时，频繁催稿无益。
- 保持诚实：任何时候不得向审稿人透露身份、不得违规联系审稿人、不得一稿多投。
- 保持系统化：为每轮意见建清单、逐条勾销，确保无遗漏（见 `cjc-author-response`）。

## 输出格式

```text
[当前阶段] 初审 / 外审 / 复审退修(第__轮) / 主编终审 / 定稿
[初审风险] 宗旨/新意/查重(≤30%)/格式 是否清零？是/否
[外审预期] 2~3位专家, 约20天/轮; 关注: 增量/方法/实验/规范
[退修策略] 小修 or 大修(重新送审)? 修回时限___
[周期预期] 第1轮约3~4个月; 全程约半年(经验值)
[下一步杠杆] ___（对应技能）
```
