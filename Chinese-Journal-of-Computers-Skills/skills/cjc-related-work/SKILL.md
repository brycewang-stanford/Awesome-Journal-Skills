# 《计算机学报》相关工作与文献综述

在《计算机学报》(Chinese Journal of Computers, CJC) 这样一本 CCF A 类**综合性**中文月刊里，相关
工作部分承担双重任务：既要让**跨学科读者**看懂你所在子领域的脉络，又要让外审专家一眼看到"本文相对
已有工作的增量在哪"。创新性是初审与外审的核心判据（无新意、重复前人工作是明确退稿理由），而创新性
正是通过相关工作的对比确立的。本技能给出面向 CJC 的相关工作与综述写法。核验日期 2026-07-09，著录
规范以官方模板为准。

## 一、综合性期刊的读者预设

- CJC 覆盖计算机全学科（理论、体系结构、软件、人工智能、数据库、网络、多媒体、图形学等），读者
  未必是你子方向的专家。相关工作要**先给最小必要背景**，再进入细节对比。
- 用一两句话交代子领域的大问题与主流路线，帮跨学科读者建立坐标，再落到与本文最相关的工作。

## 二、"delta 优先"的定位写法

相关工作不是文献罗列，而是**为本文贡献定位**。推荐"归类—对比—留白"结构：
1. **归类**：把已有工作按技术路线/问题设定分组（如 A 类基于……、B 类基于……）。
2. **对比**：指出每组的**共性局限**或未解决问题。
3. **留白**：明确指出本文正是填补哪个空白、与最接近工作的**关键差异(delta)**。

写作模板：

```text
现有 <某类方法> 主要通过 <思路> 解决 <问题>，但在 <场景/条件> 下存在 <局限>；
与最接近的 <文献[k]> 相比，本文的不同在于 <关键差异>，从而 <带来的收益>。
```

避免"某某做了X，某某做了Y"的流水账——每提一项工作都要落到与本文的关系。

## 三、文献覆盖的平衡

| 维度 | 要求 |
|---|---|
| 经典 vs 前沿 | 既引奠基性经典，也覆盖近 1~3 年最新进展，显示视野与时效 |
| 国内 vs 国外 | 兼顾国内外高水平工作；恰当引用本刊及国内顶刊相关成果 |
| 广度 vs 深度 | 对最相关的少数工作深入对比，对外围工作简要带过 |
| 正面 vs 反例 | 既引可比基线，也点出与本文路线不同的替代思路 |

## 四、引用规范

- 按 **GB/T 7714** 著录，正文序号与文末列表一一对应（详见 `cjc-writing-style`）。
- **中文文献给出中英文对照**著录。
- 文献类型标识齐全（[J]/[C]/[M]/[D] 等），信息完整可追溯。
- **避免过度自引**：自引须与本文实质相关；双盲外审阶段注意自引的匿名表述（见 `cjc-submission`）。
- **避免堆引**：一句话后挂一长串文献却不加区分，会被视为凑引用。

## 五、研究论文 vs 综述型稿件

- **研究论文中的相关工作**：服务于确立本文创新性，篇幅适中，聚焦对比。
- **综述(review)型稿件**：本身以文献梳理为贡献，要求系统性、批判性、脉络化，给出分类框架与未来
  方向，覆盖面与客观性要求更高。CJC 接收综述稿，若投综述须按综述体例组织（**待核实** 综述专门
  篇幅要求）。

## 六、常见失误对照

| 失误 | 后果 | 纠正 |
|---|---|---|
| 文献流水账、无对比 | 外审看不到创新性 | 每项工作落到与本文的 delta |
| 只引国外、忽略国内 | 显得视野不全 | 兼顾国内外高水平工作 |
| 只引经典、缺近年 | 显得脱离前沿 | 补近 1~3 年最新进展 |
| 过度自引/堆引 | 学术印象差 | 自引须实质相关，去冗余堆引 |
| 双盲下自引暴露身份 | 违反匿名 | 改用第三人称引述自己工作 |

## 七、自查清单

- [ ] 是否给跨学科读者提供了最小必要背景？
- [ ] 每项相关工作是否都落到与本文的关系/差异？
- [ ] 是否明确点出与最接近工作的关键 delta？
- [ ] 经典与近年、国内与国外是否平衡？
- [ ] 引用是否 GB/T 7714 规范、中文文献中英对照？
- [ ] 自引是否适度且在双盲下匿名？

## 八、文献检索与查全策略

- 用多个来源交叉检索：CNKI/万方(中文)、DBLP/ACM DL/IEEE Xplore/Google Scholar(英文)，避免只看
  单一库导致遗漏。
- 以**关键术语 + 同义词 + 关键作者 + 引文追溯(向前/向后)** 组合检索，确保覆盖最相关工作。
- 特别检索**本刊及国内顶刊**近年相关论文，恰当引用能显示你了解本领域中文社区的进展，也契合
  《计算机学报》(Chinese Journal of Computers, CJC) 综合性读者的关注。
- 建立文献矩阵：行是相关工作、列是"问题设定/方法/数据/局限/与本文差异"，一眼看出 delta 与空白。

## 九、综述型稿件的组织框架

若投的是综述(review)稿，相关工作即为核心贡献，建议采用：
1. **问题域界定**：明确综述范围、检索方法与纳入标准（可交代查全策略以示系统性）。
2. **分类框架**：提出一套自洽的分类维度，把已有工作纳入框架，而非简单编年罗列。
3. **批判性比较**：跨类比较优劣、适用条件与未解难题，给出对比表。
4. **趋势与展望**：指出开放问题与未来方向，体现综述的判断力。
综述对客观性、覆盖面、批判性要求更高，需避免"只介绍不评述"。

## 十、放置位置与篇幅

- 相关工作可作**独立一节**（常见于综述与理论性长文），也可拆分到引言与各技术节旁（"就近对比"）。
  《计算机学报》(Chinese Journal of Computers, CJC) 长文两种都可，取决于对比是否需要集中呈现。
- 篇幅与贡献匹配：研究论文中相关工作适中、聚焦最相关工作；不要为显示读得多而堆砌无关文献。
- 避免"相关工作与引言重复"：引言点问题与本文贡献，相关工作做系统对比，二者分工不重叠。

## 十一、相关工作终检

- [ ] 是否交叉多源检索、覆盖国内外与近年工作？
- [ ] 是否用文献矩阵梳理每项工作与本文的差异？
- [ ] 定位段是否 delta 优先、指明与最接近工作的关键区别？
- [ ] 引用是否 GB/T 7714、中文文献中英对照、序号对应？
- [ ] 自引是否适度、双盲下匿名？
- [ ] 若为综述稿，是否有分类框架与批判性比较？

```text
[delta] 与最接近工作[k]的关键差异：___
[平衡] 经典/前沿、国内/国外 覆盖：___

```
