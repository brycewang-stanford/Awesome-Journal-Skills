# 《计算机学报》选题与栏目定位

投稿前最高杠杆的一步，是判断选题是否**契合《计算机学报》(Chinese Journal of Computers, CJC)**，
以及应落到哪种稿件类型。CJC 是 CCF A 类计算机学科**综合性**中文旗舰月刊（中国计算机学会与中科院
计算所主办，创刊 1978，月刊），"不符合本刊宗旨与选题范围""无新意"是初审退稿的头号理由。选错刊或
定位不清，再好的工作也会在门口被拦。本技能帮你在投稿前把契合度、创新增量与栏目归属想清楚。核验
日期 2026-07-09。

## 一、CJC 的定位：综合性 + 长文 + 原创

- **综合性**：覆盖计算机全学科——计算机科学理论、体系结构、软件、人工智能、数据库、网络、安全、
  多媒体、图形学以及新技术应用等，不限于某个子方向。
- **长文传统**：以高水平**原创研究长文**为主，适合把方法、理论、系统与充分实验完整讲透的工作。
- **原创性门槛**：只发表创新性成果；增量不足、重复前人、综述性罗列而无新见的稿件难以通过。

## 二、与兄弟中文刊的分工（避免投错）

| 期刊 | 侧重 | 何时优先它 |
|---|---|---|
| **计算机学报 (CJC)** | 计算机全学科**综合**、原创长文 | 跨子领域、理论+系统兼备、需长文篇幅 |
| 软件学报 | 软件工程、系统软件、程序语言 | 强软件/系统方法学贡献 |
| 计算机研究与发展 | 计算机综合（与 CJC 近，亦综合性） | 综合性工作的平行选择 |
| 自动化学报 | 控制、自动化、部分 AI | 控制论/自动化色彩强 |
| 电子学报 | 电子、通信、信号、器件 | 偏电子通信/信号处理 |
| 计算机科学 | 计算机综合（覆盖面广） | 面向更宽读者的综合工作 |

判断口诀：**工作若"横跨计算机多个子领域、以原创长文形态呈现、理论与系统实验兼备"，CJC 是天然
落点**；若强烈偏向软件工程/系统软件→软件学报，偏控制→自动化学报，偏电子通信→电子学报。

## 三、稿件类型选择

CJC 接收多种类型，据工作成熟度与体量选择：

| 类型 | 适用 |
|---|---|
| 论文(研究长文) | 完整原创工作，方法+理论+充分实验 |
| 技术报告 | 系统/工程性较强、以技术方案与实测为主 |
| 短文 | 篇幅较小但有明确创新点的工作 |
| 研究简报 | 阶段性、快速报道的新结果 |
| 综述 | 系统批判性梳理某方向，给出分类与展望 |

（各类型的具体篇幅与体例要求以官方模板/投稿须知为准，**待核实** 细则。）

## 四、创新增量自检

在投稿前诚实回答：
- 本文相对最接近的已有工作，**新在哪一点**？（新问题/新方法/新理论/新系统/新发现）
- 这个增量能否用一句话讲清、并被实验或证明支撑？（见 `cjc-related-work`、`cjc-experiments`）
- 如果换掉某个"新颖点"，结论是否仍成立？若是，则真正贡献可能不在此处，需重新定位。

## 五、会议扩展版的定位

- 若本文是作者已发表**会议论文的期刊扩展版**，需评估新增贡献是否达到实质比例（新增理论、显著扩展
  实验、新增系统模块等），并在投稿时**显著声明扩展关系**、规范引用会议版本（见 `cjc-submission`）。
- 仅做文字扩写、无实质新增，难以满足 CJC 原创长文门槛。

## 六、栏目/专题机会

- 关注本刊是否开放**专题(special issue)** 征稿，契合的选题投专题可提升匹配度（专题安排与截稿
  **待核实**，以官网征稿通知为准）。

## 七、定位自查清单

- [ ] 选题是否落在计算机学科范围内、契合综合定位？
- [ ] 工作体量与形态是否适合长文？
- [ ] 创新增量是否清晰、可支撑、经得起"换掉新颖点"测试？
- [ ] 是否比兄弟专业刊更适合投 CJC？
- [ ] 稿件类型选择是否恰当？
- [ ] 会议扩展版是否声明并达到实质新增？

## 八、选题契合度打分表

投稿前给选题做一次自评（每项 0/1/2 分，合计供参考）：

| 维度 | 0 分 | 1 分 | 2 分 |
|---|---|---|---|
| 学科契合 | 超出计算机学科 | 边缘相关 | 计算机核心问题 |
| 综合性视角 | 极窄工具改进 | 单子领域 | 跨子领域/有普遍意义 |
| 创新增量 | 重复前人 | 增量有限 | 明确新问题/新方法 |
| 工作体量 | 单点小结果 | 中等 | 适合长文的完整工作 |
| 证据充分 | 缺实验/证明 | 部分 | 充分且可信 |

合计偏低说明或选题不符本刊、或工作尚未成熟；应回炉或改投更契合的刊。

## 九、与兄弟刊的"换刊测试"

问自己：如果把这篇工作原样投给软件学报/自动化学报/电子学报，会不会"更对口"？
- 若换到某专业刊明显更契合(如强软件工程→软件学报)，说明本文并非《计算机学报》(Chinese Journal
  of Computers, CJC) 的最优落点。
- 若本文横跨多个计算机子领域、以原创长文形态呈现、且理论与系统实验兼备，换到任一专业子刊都会
  "损失一部分"，则 CJC 的综合定位正是它的归属。

## 十、选题阶段的常见误区

- **把会议短文原样当期刊长文投**：CJC 期待完整长文，需实质扩展（见 `cjc-submission` 会议扩展版）。
- **选题过窄或过旧**：缺乏综合价值或已被充分研究，难过创新性关。
- **贪大求全**：一篇塞进多个不相干贡献，反而稀释主线；聚焦一个清晰贡献更易通过。

## 十一、选题终检

- [ ] 学科契合、综合定位、长文形态是否满足？
- [ ] 换刊测试是否表明 CJC 是最优落点？
- [ ] 创新增量是否清晰、可支撑、能过"换掉新颖点"测试？
- [ ] 稿件类型选择是否恰当？
- [ ] 若为会议扩展版，是否达到实质新增并将声明？

```text
[会议扩展] 是否扩展版？增量是否实质并声明？是/否
[栏目/专题] 目标栏目或专题：___（以官网为准）
```
