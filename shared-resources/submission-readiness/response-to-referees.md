# Response-to-Referees — turn a referee report into a point-by-point reply + revision

After an R&R (real) — or after the [`simulated-referee`](simulated-referee.md) rehearsal —
this drafts the **point-by-point response letter** and the **revision plan** that decide
whether the second round survives. It is the capability that closes the review loop:

```
simulated-referee (rehearse the attack)  →  fix via the pack skills + execution bridge
                                          →  response-to-referees (answer every point)
```

> **Every "we have revised…" must be backed by an actual change.** This drafts the reply
> *around real edits and re-runs*, not promises. Where a fix is empirical, it ties to the
> pack's skill + the execution bridge so the response cites a number you actually produced.

## Inputs

- The **referee + AE letter** (real R&R, or the simulated-referee synthesis).
- The **manuscript** and the diffs you will make.
- The target pack's **`*-rebuttal`** and **`*-referee-strategy`** skills (the venue's
  response conventions) and **`resources/official-source-map.md`** (response-letter
  format, word limits, resubmission deadline, whether a separate cover letter is expected).

## Method

1. **Inventory every comment.** Number them per referee + the AE. Miss none — an
   unanswered comment is a second-round rejection risk. Tag each: *major / minor* and
   *substantive / clarification*.
2. **Editor first.** Identify the AE's 1–3 decisive concerns and lead the response with
   them; the editor decides, the referees advise.
3. **Classify each response.** For every comment choose, honestly:
   - **Concede & fix** — make the change; cite the new section/table/number.
   - **Address partially** — do what's defensible; state precisely what and why not more.
   - **Push back with evidence** — disagree *respectfully*, with a specific reason or a
     new analysis, never a bare assertion. Pick this rarely and only when right.
4. **Back substantive fixes with a real re-run.** Map each empirical comment to the pack
   skill + execution bridge — e.g. *"R1: TWFE under staggered adoption → re-estimated with
   `callaway_santanna`; added `bacon_decomposition` (new Table IA.4); effect now 3.1pp."*
   The number in the reply must be one you actually computed (see
   [`../empirical-methods/execution-with-mcp.md`](../empirical-methods/execution-with-mcp.md)).
5. **Write the letter.** Quote each comment verbatim, then respond directly beneath it;
   point to the exact location of every change. Respectful, specific, complete.
6. **Summarize the changes** in a short table the editor can scan.

## Output format

```
【Cover note to the editor】thank; summarize how the decisive concerns were addressed; flag major changes
【Response — Referee 1】
  R1.1 (quote) → response: concede & fix / partial / push-back · what changed · where (§/table) · the new number
  R1.2 …
【Response — Referee 2】…
【Response — AE】(the decisive points, answered first/most fully)
【Summary of changes】table: comment → change → location
【Open/declined】comments not fully addressed, with the honest reason
```

## Anti-patterns

- **Silent omission** — skipping a comment. Answer every one, even to say why you didn't change it.
- **Defensive tone / blanket push-back** — referees decide your fate; disagree sparingly and with evidence.
- **Promise without delivery** — "we have addressed this" with no actual edit/number is fatal if the referee checks.
- **Burying the decisive point** — answer the AE's load-bearing concern first and fully.
- **Venue-blind format** — some venues cap the response length or want a specific structure; read the source-map.

## Hard rules

1. **Editor first**, then referees.
2. **Every claimed fix corresponds to a real change/re-run**; empirical claims cite a
   number produced via the pack's skill + execution bridge.
3. **Answer every comment**; none silently dropped.
4. **Venue conventions from the pack + source-map**, not from memory.
5. It drafts a *reply*; the author owns the final scientific judgment and sign-off.
