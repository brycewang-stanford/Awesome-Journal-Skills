# Computer Science Conference Skills

AI-first conference profiles for computer-science manuscript strategy. The pack contains **155 conference skills** plus **`cs-ai-conference-workflow`**, a router that starts with NeurIPS / ICML / ICLR / AAAI / IJCAI and then routes across data mining, vision, NLP, robotics, HCI, systems, security, SE/PL, databases, and theory.

## What This Pack Does

- Choose the right CS/AI conference before formatting a manuscript.
- Re-frame a paper for the venue's contribution type and evidence bar.
- Surface desk-reject risks: weak novelty, missing baselines, anonymity violations, artifact gaps, ethics gaps, and current-cycle policy mismatches.
- Force a live official-cycle check before submission-ready advice.

## Use First

```text
Use cs-ai-conference-workflow to route my paper to the right CS/AI conference, then invoke the selected conference skill.
```

## Official-Source Discipline

Conference policies change every year. These skills intentionally avoid hard-coding volatile page limits, deadlines, chair names, fees, locations, and acceptance rates. Before submission, always re-check the current official CFP, author kit, submission portal, ethics policy, AI-use policy, artifact/code policy, rebuttal instructions, and camera-ready requirements.

See `resources/conference-roster.md` and `resources/official-source-map.md`.
