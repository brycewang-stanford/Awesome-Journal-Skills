# Computer Science Conference Skills

面向计算机科学投稿策略的 AI-first 会议 profile 包。该包包含 **155 个会议技能**，以及 **`cs-ai-conference-workflow`** 路由技能：先从 NeurIPS / ICML / ICLR / AAAI / IJCAI 等 AI 顶会开始，再路由到数据挖掘、视觉、NLP、机器人、HCI、系统、安全、软件工程、程序语言、数据库与理论方向。

## 这个包做什么

- 在排版投稿前，先判断论文最适合哪个 CS/AI 会议。
- 按会议的贡献类型和证据门槛重写 framing。
- 提前暴露 desk-reject 风险：创新性不足、baseline 缺失、匿名违规、artifact 缺口、伦理缺口、年度政策不匹配。
- 强制在给出 submission-ready 建议前核验当年官方 CFP / author kit。

## 优先使用

```text
Use cs-ai-conference-workflow to route my paper to the right CS/AI conference, then invoke the selected conference skill.
```

## 覆盖范围

- AI / ML / 数据挖掘 / responsible AI：35
- 视觉 / 图形学 / 多媒体 / 医学影像：20
- NLP / 语音 / 信息检索：20
- 机器人与 embodied AI：12
- HCI / 可视化 / 无障碍：15
- 系统 / 网络 / 体系结构 / HPC：20
- 安全 / 隐私 / 密码实现：12
- 软件工程 / 程序语言 / 形式化方法：16
- 数据库与理论：5
- 路由技能：1

## 官方来源纪律

会议政策每年都会变化。本包刻意不把年度 deadline、页数限制、主席名单、费用、地点、录用率等易变事实写死。正式投稿前，必须重新核验当年官方 CFP、author kit、投稿系统、伦理政策、AI 使用政策、artifact/code 政策、rebuttal 规则和 camera-ready 要求。

详见 `resources/conference-roster.md` 和 `resources/official-source-map.md`。
