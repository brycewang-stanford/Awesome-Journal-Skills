# ICLR Skills

Twelve agent skills for International Conference on Learning Representations (ICLR) main-conference
papers. The pack focuses on OpenReview submission readiness, double-blind public review, author
discussion, revision discipline, reproducibility, artifacts, and camera-ready preparation.

Official basis checked on 2026-06-01: ICLR 2026 CFP, Author Guide, Reviewer Guide, Code of Ethics,
Code of Conduct, LLM-use policy updates, poster instructions, and the OpenReview conference group.
See `resources/official-source-map.md` for the exact source map.

## Skills

- `iclr-submission` - audit OpenReview, anonymity, page limit, reciprocal reviewing, LLM disclosure,
  dual-submission, and supplementary readiness.
- `iclr-author-response` - draft concise OpenReview discussion replies and revision notes after
  reviews.
- `iclr-camera-ready` - prepare accepted-paper metadata, camera-ready files, poster assets, video,
  and project-page links.
- `iclr-artifact-evaluation` - package code, data, demos, checkpoints, and instructions for an
  ICLR-style reproducibility review, even when no separate artifact badge is promised.
- `iclr-reproducibility` - strengthen evidence, randomization, compute disclosure, and robustness.
- `iclr-supplementary` - organize appendices, supplementary files, code, private links, and
  discussion-period revisions.
- `iclr-review-process` - reason about review dimensions, AC/SAC roles, public comments, restricted
  comments, and final recommendations.
- `iclr-writing-style` - revise for learning-representation contribution, clarity, and reviewer
  navigation.
- `iclr-related-work` - position against concurrent OpenReview/arXiv work and adjacent ICLR claims.
- `iclr-experiments` - audit baselines, ablations, scaling, statistics, robustness, and compute.
- `iclr-workflow` - manage a calendar from topic selection through poster upload.
- `iclr-topic-selection` - decide whether a project is ICLR-shaped or should be routed elsewhere.

## Maintenance notes

- Reopen current-cycle official sources before deadline-sensitive advice.
- Treat ICLR 2026 facts as historical anchors, not permanent rules.
- Keep OpenReview public visibility in mind: author responses, revisions, and accepted-paper
  discussion can be read as part of the scientific record.
