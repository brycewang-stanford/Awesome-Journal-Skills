# M&SOM Pre-Submission Checklist

Check the official M&SOM submission-guidelines page, editorial-board page, and
ScholarOne screens before submitting. This checklist baseline was refreshed on
2026-06-20.

## 1. Operations centrality

- [ ] An operations decision/problem is the **primary** contribution (not a backdrop)
- [ ] The work clearly belongs to one of the six editorial Departments

## 2. Anonymization (double-anonymous)

- [ ] No author names, affiliations, or acknowledgments in the manuscript PDF
- [ ] Self-citations neutral ("Author (2020)"), not "our prior work"
- [ ] Funding/acknowledgments supplied separately, not in the manuscript
- [ ] File metadata and file names scrubbed of identifiers

## 3. Format (official style files)

- [ ] Typeset on the official **M&SOM LaTeX/Word template** (one column, 11-pt, 1-inch margins)
- [ ] Main PDF **≤ 32 pages** including references, tables, figures, appendices
- [ ] Online supplement **≤ 16 pages**
- [ ] **Structured abstract ≤ 300 words**, all three required sections, jargon-free
- [ ] INFORMS author-year citations; references ordered by first author + #authors + year (v1.6)

## 4. Department routing

- [ ] Primary **Department** selected
- [ ] **Two preferred Department Editors** named
- [ ] Manufacturing-and-Supply-Chain first/second-choice rule respected
- [ ] Correct article type (standard research / OM Forum / Practice Platform)

## 5. Files & declarations

- [ ] Single anonymized **PDF** + online supplement uploaded
- [ ] Cover letter: operations problem, contribution, fit, not under review elsewhere
- [ ] Prior conference/working-paper versions disclosed
- [ ] ScholarOne payment screen reviewed; optional Open Option fee handled only if choosing OA

## 6. Data & code (replicability)

- [ ] Manuscript permits replication (enough detail/references)
- [ ] Prepared to share raw data on editorial request and to retain it
- [ ] Own code + access/linking instructions ready for licensed data (Census/Compustat/CRSP/FactSet/WRDS)
- [ ] Reuse of any shared data/code cites and acknowledges the source

## 7. Consistency

- [ ] Notation, construct/parameter names, and exhibit numbers consistent throughout
- [ ] Every in-text citation appears in the reference list and vice versa
- [ ] Closest analytical/empirical antecedents cited
