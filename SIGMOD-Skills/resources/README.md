# SIGMOD Resources

Working materials that complement the twelve `skills/`. The skills advise; this directory
lets an agent draft against a modeled SIGMOD opening, benchmark positioning against
award-anchored real papers, and pressure-test an artifact package before a PACMMOD round
deadline or an ARI submission.

## Contents

| Resource | Reach for it when... |
| --- | --- |
| [`worked-examples/01-introduction.md`](worked-examples/01-introduction.md) | You need to see the page-one contract and running-example device executed on a fictional storage-engine draft, before → after. |
| [`exemplars/library.md`](exemplars/library.md) | You want real, verification-anchored SIGMOD papers (Test-of-Time lineage plus an industry-track case) as positioning references — with a list of famous near-misses that are *not* SIGMOD papers. |
| [`official-source-map.md`](official-source-map.md) | You must confirm what was actually verified, from which URL, on 2026-07-08 — including the access-method caveat and the 待核实 ledger. |
| [`external_tools.md`](external_tools.md) | You need the live official surfaces (CFP, important dates, PACMMOD guidelines, ARI, CMT pointer) and the do-not-infer rules. |
| [`code/README.md`](code/README.md) | You want the shared artifact smoke-check kit, translated into data-systems terms for ARI-bound packages. |

## Scope note

This pack serves the SIGMOD **research track's** PACMMOD round machinery, with the
industrial track covered as a routing option. It is a data-management conference pack: do
not import econometrics kits, and do not treat the ML-conference smoke checker in `code/`
as a substitute for the workload/baseline discipline in
[`../skills/sigmod-experiments/SKILL.md`](../skills/sigmod-experiments/SKILL.md).

## A sensible path through

1. Route first: [`../skills/sigmod-topic-selection`](../skills/sigmod-topic-selection/SKILL.md)
   decides research track vs. industrial vs. PVLDB before any formatting work.
2. Shape the paper: [`../skills/sigmod-writing-style`](../skills/sigmod-writing-style/SKILL.md)
   with [`worked-examples/01-introduction.md`](worked-examples/01-introduction.md) open
   beside the draft, positioning checked against [`exemplars/library.md`](exemplars/library.md).
3. Harden the evidence: [`../skills/sigmod-experiments`](../skills/sigmod-experiments/SKILL.md),
   [`../skills/sigmod-reproducibility`](../skills/sigmod-reproducibility/SKILL.md), and the
   smoke check in [`code/README.md`](code/README.md).
4. Execute the round: [`../skills/sigmod-workflow`](../skills/sigmod-workflow/SKILL.md) and
   [`../skills/sigmod-submission`](../skills/sigmod-submission/SKILL.md), with every date
   re-confirmed through [`official-source-map.md`](official-source-map.md) and
   [`external_tools.md`](external_tools.md).
