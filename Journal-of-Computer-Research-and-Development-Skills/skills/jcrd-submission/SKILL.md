# 《计算机研究与发展》投稿就绪审计 (JCRD Submission)

在把稿件上传到《计算机研究与发展》(Journal of Computer Research and Development, JCRD) 的在线
投稿系统之前，先跑完这份审计。JCRD 是由中国科学院计算技术研究所与中国计算机学会 (CCF) 主办、
科学出版社出版、创刊于 1958 年的 CCF A 类中文旗舰**月刊**，以**专题(special topic)组稿**见长、
综述与原创并重、实行**双盲评审**。投稿被当作一篇要经受编辑初审、专家外审、复审与主编终审的完整
学术论文来评判。以下事实于 2026-07-09 通过官网、CNKI/万方与 CCF 目录交叉核验（见
`resources/official-source-map.md`），凡本刊未公开确认的数字均标 **待核实**，以投稿系统内最新
《投稿须知》为准。

## 一、投稿系统与账号

- 本刊经**官网在线投稿系统**投稿，入口为 `https://crad.ict.ac.cn/`；编辑部邮箱 `crad@ict.ac.cn`，
  电话 (010) 62620696，通信地址 北京 2704 信箱，邮编 100190。
- 注册时填写的题名、作者、单位、通信作者与研究方向决定编辑分派外审专家的匹配度，务必用**真实终稿
  信息**，不要用占位标题。
- **只接收中文稿，不受理英文稿**；投稿须承诺**未在正式出版物发表**、**不存在一稿多投**。
- 稿件提交后一般会在系统内看到状态流转（收稿→初审→外审→…）。审稿周期据投稿人经验约「总计 4 个月
  左右」，为个案转述而非官方承诺（**待核实**）。

## 二、双盲评审的匿名清扫（本刊硬门槛）

JCRD 实行**双盲评审**：外审专家只见稿件内容、不见作者身份。因此提交的**盲审版**必须彻底匿名：

```bash
# 对提交 PDF 与随附材料做机械匿名自查
pdfinfo paper.pdf | grep -Ei 'author|creator|producer'          # 文件属性泄露
pdftotext paper.pdf - | grep -nEi '致谢|基金|资助|课题组|本文作者|我们(实验室|课题组)' | head
pdftotext paper.pdf - | grep -nEi 'github\.com/[a-z0-9-]+|gitee|个人主页' | head
grep -rniE 'university|@[a-z0-9.]+\.edu|xx大学|xx实验室' artifact/ --include='*.md' | head
```

盲审版需处理：**题名页去作者/单位**、**致谢与基金项目号在盲审版隐去或中性化**、**自引改第三人称**
（「文献[x]提出…」而非「我们先前工作」）、**图表水印/页眉页脚不含单位**、**代码/数据链接重托管到
匿名服务**、**PDF 文件属性清空**。终稿（录用后）再补回作者、单位、基金号，见 `jcrd-camera-ready`。

## 三、格式与中文长文组件

- 使用本刊**官方模板**（Word / LaTeX；LaTeX 模板含示例与编译说明）。中文正文按 A4 排版，提交 PDF
  （必要时附源文件）。篇幅上限以模板与编辑部要求为准（**待核实**具体数值）。
- 稿件必须齐备下列中文科技期刊标准部件（详见 `jcrd-writing-style`）：

| 组件 | 要求 |
|---|---|
| 中文题名 + 英文题名 | 英文题名可独立检索、与中文一致 |
| 中文摘要 + 英文摘要 (Abstract) | 中文约 200~300 字（**待核实**上限），英文与中文对应 |
| 关键词 + Key words | 3~8 个，中英文对照 |
| 中图分类号 | 计算机类多为 **TP** 类 |
| 作者简介 / 通信作者 | 盲审版隐去，终稿补全 |
| 基金项目 | 盲审版中性化，终稿补全项目号 |
| 参考文献 | 按 **GB/T 7714** 顺序编码制、中英文对照著录 |

## 四、稿件类型与栏目/专题归属

- 稿件类型可为：**学术论文、技术报告、综述、研究热点论文**。选定后按对应体例撰写。
- **判断投常规栏目还是对口专题**：JCRD 每期常设专题，由客座编辑组稿。若工作强绑定当期专题主题、
  或具综述价值，投**对口专题**匹配度更高（受专题截稿时间表约束，**待核实**当期主题）。选路细节见
  `jcrd-topic-selection`。

## 五、学术规范自查

- 投稿前用正规工具**自查重复率**，规避文字复制、图表挪用与不当署名。
- 承诺**无一稿多投**：同一工作不得同时投他刊；已在会议发表的扩展稿须显著扩充并如实说明。
- 署名、贡献与利益冲突如实填写；引用他人成果规范标注，避免自我剽窃。

## 六、退稿风险分诊

| 审计发现 | 严重度 | 处置 |
|---|---|---|
| 提交英文稿 | 直接退稿级 | 本刊只收中文稿，改中文重投 |
| 盲审版泄露作者/单位身份 | 双盲违规 | 重新匿名清扫，重托管材料，清空文件属性 |
| 一稿多投 / 已正式发表 | 学术不端 | 撤回一处，核实会议扩展是否合规 |
| 缺中英文摘要/关键词/中图分类号 | 形式退修 | 补齐标准部件，对照 `jcrd-writing-style` |
| 参考文献未按 GB/T 7714 | 形式退修 | 按国标中英文对照重排 |
| 与本刊征稿范围/专题不符 | 选题退稿风险 | 重定栏目或改投姊妹刊，见 `jcrd-topic-selection` |
| 创新性或完整性不足 | 学术退稿风险 | 强化贡献与实验/综述深度，再评估 |

## 七、提交前操作顺序

1. 冻结正文论点，参考文献可继续微调。
2. 生成**盲审版** PDF，跑机械匿名自查（第二节脚本）。
3. 核对中英文题名/摘要/关键词、中图分类号、GB/T 7714 文献齐备。
4. 确认稿件类型与所投栏目/专题；投专题则核对当期截稿时间表。
5. 自查重复率、确认无一稿多投、填写利益冲突。
6. 在系统内填全字段（研究方向标签匹配证据），上传后重新下载 PDF 冷读一遍确认为终版。

## 八、逐周期复核

- 当期**专题主题、客座编辑与截稿日**（逐期变动，**待核实**）。
- 摘要字数、篇幅上限与**模板版本**（**待核实**）。
- 版面费/审理费政策与审稿周期（**待核实**，勿杜撰）。

## 九、常见误区（本刊特有）

- **误把英文稿投来**：《计算机研究与发展》(Journal of Computer Research and Development, JCRD)
  只受理中文稿，英文写作习惯者须整体中译，而非只译摘要。
- **只匿名摘要不匿名全文**：双盲要求正文、致谢、基金号、图表水印、文件属性、代码链接全部匿名，
  遗漏任何一处都可能被外审识别身份。
- **专题当常规栏目投**：投对口专题须在系统中选择专题并遵守其截稿时间表；错过截稿只能转常规栏目。
- **参考文献用英文期刊格式**：本刊按 GB/T 7714 顺序编码制、中英文对照著录，不能沿用 IEEE/ACM
  的著录风格。
- **贡献写成综述式罗列**：即便是原创稿件，引言也须给出可数的 delta，而非泛泛铺陈背景。

## 十、投稿材料清单

| 材料 | 盲审版 | 说明 |
|---|---|---|
| 正文 PDF | 匿名 | 题名页去身份，自引第三人称 |
| 中英文摘要与关键词 | 含 | 中图分类号 (TP) 齐备 |
| 图表与算法 | 匿名 | 水印/页眉不含单位 |
| 补充材料 | 匿名 | 见 `jcrd-supplementary` |
| 代码/数据链接 | 匿名托管 | 见 `jcrd-artifact-evaluation` |
| 基金与作者信息 | 隐去 | 录用后终稿补回 |
