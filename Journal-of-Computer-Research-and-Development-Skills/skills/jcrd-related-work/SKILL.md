# 《计算机研究与发展》相关工作与文献综述 (JCRD Related Work)

本技能帮助为《计算机研究与发展》(Journal of Computer Research and Development, JCRD) 稿件写好
**相关工作**或**文献综述**。JCRD **综述与原创并重**，且以**专题**组稿见长，综述型稿件是本刊显著
栏目（研究热点/综述）。无论综述型还是原创型，文献部分都要有**分类框架**与**清晰定位**，而非文献
罗列。

## 一、综述型 vs 原创型的文献部分

| 类型 | 文献部分目标 | 组织方式 |
|---|---|---|
| 综述 / 研究热点 | 系统梳理某主题的技术脉络与开放问题 | 分类框架 + 趋势判断 + 未来方向 |
| 原创研究 | 定位本文相对已有工作的 delta | 按技术路线归类 + 指出各自不足 → 引出本文 |

## 二、综述型稿件：构建分类框架

- **一维不够就多维**：按方法路线、应用场景、数据形态等维度建立分类体系（给分类图/表）。
- 每类给**代表工作 + 核心思想 + 优缺点**，而非逐篇复述。
- 明确**开放问题**与**未来方向**，这是综述的价值所在，也是审稿人重点看的部分。
- 附**系统性检索**说明：检索式、数据库（CNKI/万方/IEEE/ACM/DBLP 等）、时间范围、纳入排除标准，
  可放补充材料（见 `jcrd-supplementary`）。

## 三、原创稿件：delta 定位

- 相关工作按**技术路线**分组，每组一句话点出**共同思路**与**共同不足**。
- 指出与**最接近工作**的关键差异（问题设定/方法/证据），用一句可数的 delta。
- 避免「已有方法都不够好，所以我们…」式空泛贬低；要具体到失效模式。

## 四、覆盖度：国内外并重

- 兼顾**中文文献与英文文献**：JCRD 作为中文刊，既要覆盖国际前沿，也要恰当引用国内相关进展。
- 覆盖**近三年**主流工作 + 奠基性经典；避免只引自己熟悉的小圈子。
- 引用要**读过再引**，转引风险高；核对 GB/T 7714 著录格式（见 `jcrd-writing-style`）。

## 五、双盲下的自引

- 自引改**第三人称**：「文献[x]提出…」而非「我们先前工作」。
- 盲审版不因自引密集暴露身份；保持匿名（见 `jcrd-submission`）。

## 六、专题稿件的综述组织

- 投**对口专题**的综述要**紧扣专题主题范围**，与当期专题定位一致（**待核实**当期主题）。
- 专题综述常需与专题内其他文章形成互补，避免大面积重叠；可在框架中体现专题视角。

## 七、常见问题与修法

| 问题 | 修法 |
|---|---|
| 文献罗列、无框架 | 建分类体系，按类归纳共性与不足 |
| 只贬低不定位 | 给具体失效模式 + 本文可数 delta |
| 只引英文/只引中文 | 国内外并重，补齐两侧 |
| 自引暴露身份 | 改第三人称，盲审版匿名 |
| 覆盖过时 | 补近三年主流工作 |
| 综述无开放问题 | 明确列出待解难题与未来方向 |

## 八、综述分类框架示例

以「面向长文档的检索增强生成」主题为例，一个可用的多维分类框架：

```text
维度一（检索策略）：稀疏检索 | 稠密检索 | 混合检索
维度二（证据利用）：拼接输入 | 迭代检索 | 生成约束
维度三（评测重心）：事实一致性 | 答案正确性 | 效率
```

- 每个交叉格给**代表工作 + 核心思想 + 局限**，用表格或分类图呈现。
- 框架末尾归纳**共性瓶颈**（如长文档下噪声召回）与**开放问题**，为后文原创工作或未来方向铺垫。
- 《计算机研究与发展》(Journal of Computer Research and Development, JCRD) 的综述栏目尤其看重
  框架的**系统性**与**判断力**，而非文献数量。

## 九、文献检索与筛选记录

综述型稿件应给出可复核的检索过程（可放补充材料）：

| 项目 | 内容 |
|---|---|
| 数据库 | CNKI、万方、IEEE Xplore、ACM DL、DBLP、arXiv 等 |
| 检索式 | 关键词组合与布尔逻辑 |
| 时间范围 | 如近五年 + 奠基性经典 |
| 纳入标准 | 与主题相关、同行评审、可获取 |
| 排除标准 | 重复、非正式、离题 |
| 结果规模 | 初检 N 篇 → 精读 M 篇 |

## 十、检查清单

- [ ] 综述有多维分类框架 + 开放问题 + 未来方向。
- [ ] 原创稿相关工作按路线分组，指出各自不足与本文 delta。
- [ ] 国内外文献并重，覆盖近三年 + 经典。
- [ ] 系统检索说明（综述）：检索式、库、时间、纳排标准。
- [ ] 自引第三人称，盲审版匿名。
- [ ] 文献按 GB/T 7714 著录，读过再引。
- [ ] 专题稿综述紧扣当期专题主题。

## 小结

《计算机研究与发展》(Journal of Computer Research and Development, JCRD) 综述与原创并重，文献
部分无论出现在综述型稿件还是原创稿件中，都要有分类框架与清晰定位，而非文献罗列。综述型给多维
框架、开放问题与可复核的检索过程；原创型按技术路线归类并给出可数的 delta；两者都要国内外文献
并重、自引第三人称、按 GB/T 7714 中英文对照著录，投专题时紧扣当期专题主题。

## 十一、逐周期复核

- 综述型稿件是否符合本刊当期对综述/研究热点栏目的最新要求（**待核实**）。
- 若投对口专题，专题对综述覆盖范围与篇幅的具体约定（**待核实**当期启事）。
- 与专题内其他文章的分工需与客座编辑或责任编辑确认，避免主题重叠（**待核实**）。

## 输出格式

```text
[JCRD 相关工作] 通过 / 待改
[类型] 综述 / 原创相关工作
[框架] 分类维度 + 开放问题（综述）/ 路线分组 + delta（原创）
[覆盖] 国内外并重？近三年 + 经典？
[检索] 检索式/库/时间/纳排（综述）？
[双盲] 自引第三人称？
[专题] 紧扣当期专题主题（若投专题）？
```
