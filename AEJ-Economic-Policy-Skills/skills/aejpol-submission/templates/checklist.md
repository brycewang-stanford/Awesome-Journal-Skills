# AEJ: Economic Policy — Pre-Submission Self-Check

An 8-section checklist to run before submitting to the *American Economic Journal: Economic Policy*
via the AEA submission system. Pair with [`../SKILL.md`](../SKILL.md) and the
[`../../../resources/official-source-map.md`](../../../resources/official-source-map.md).
Volatile items (fee, codes, editors) — official AEA pages checked 2026-06-20; re-confirm at submission.

## 1. Policy fit & contribution
- [ ] A specific policy lever is named (instrument + who is affected)
- [ ] The policy question carries a welfare / cost-benefit / distributional stake
- [ ] Broad-interest test passed (a non-specialist AEA reader sees why it matters)
- [ ] Sibling check done (not JPubE field-only / not AEJ:Applied no-policy / not AER-scale)

## 2. Identification
- [ ] The policy variation and identifying assumption are stated explicitly
- [ ] Design diagnostics present (pre-trends / density / first-stage F / balance)
- [ ] Heterogeneity-robust estimator used where staggered timing applies
- [ ] Estimand is the policy-relevant one; inference clustered at the assignment level

## 3. Welfare / policy reading
- [ ] A welfare object is reported (MVPF / net cost-per-outcome / incidence / optimum)
- [ ] Sufficient statistics are identified by the design, not assumed
- [ ] Uncertainty propagated into the welfare number
- [ ] A distributional / incidence reading provided where the policy redistributes

## 4. Robustness
- [ ] Headline policy number (not just a coefficient) shown stable across specs
- [ ] The most likely referee threat is pre-empted with a dedicated exhibit
- [ ] Inference stress-tested (wild-cluster / AR / multiple-testing as relevant)
- [ ] Selection-on-unobservables addressed (Oster bounds or equivalent)

## 5. Exhibits & style
- [ ] No significance asterisks / boldface; standard errors (or CIs) shown
- [ ] One self-contained headline exhibit carries the policy magnitude
- [ ] Tables note sample, units, estimator, controls, clustering, N, dep-var mean
- [ ] Figures show effects with uncertainty bands; readable in greyscale

## 6. Front matter (single-blind)
- [ ] Title, byline, and author affiliations on the first page (single-blind — do not anonymize)
- [ ] Acknowledgement footnote: author emails, funding/grants, acknowledgments, IRB/pre-registration where applicable
- [ ] Abstract is 100 words or fewer

## 7. Files, codes & declarations
- [ ] Main manuscript with front matter + online appendix
- [ ] JEL codes + keywords supplied on the form
- [ ] Data and Code Availability Statement drafted (public vs. restricted; sources cited)
- [ ] AEA disclosure statement for every author (funding / interested parties)
- [ ] Submission fee ready / eligible category confirmed (verify current amount)
- [ ] Not under review elsewhere; AI not listed as an author

## 8. Reproducibility readiness (for conditional acceptance)
- [ ] Master script regenerates every exhibit and in-text number
- [ ] Software / package / `.ado` versions pinned; seeds set and reported
- [ ] Restricted-data access path documented; shareable code/extracts ready
- [ ] Deposit target = AEA Data and Code Repository (openICPSR); README per AEA Data Editor template
