# 《软件学报》录用后定稿与清样 (Journal of Software Camera-Ready)

稿件被《软件学报》(Journal of Software, JOS) 录用后，进入定稿→校样→清样→出版。这一阶段的
目标是把内容锁定、把格式做到符合出版要求、把错误在见刊前清干净。本刊按 2021 版模板出版，
排期与流程以编辑部通知为准（见
[`resources/official-source-map.md`](../../resources/official-source-map.md)）。

## 一、定稿：内容锁定

- 确认所有审稿意见已落实，正文与结论稳定，不再做实质性内容变更。
- 把修回中承诺的修改全部并入正稿；删除面向审稿人的临时标注/颜色。
- 核对贡献陈述、实验数字、结论在全文一致（含摘要与正文不冲突）。

## 二、格式终校（2021 版模板）

```text
[ ] 套用官网"作者园地"2021 版模板，正文双栏
[ ] 中文题目/英文题目、作者与单位、通信作者、基金脚注齐全
[ ] 中文摘要 + English Abstract 终校，语义与正文一致
[ ] 中文关键词 + English Keywords 一一对应
[ ] 中图分类号正确
[ ] 图表有中文题注、清晰可读、量纲规范
[ ] 公式编号、变量斜体、算法环境规范
```

## 三、参考文献 GB/T 7714 复核

- 逐条复核 GB/T 7714 著录：作者、题名、类型标识 [J]/[C]/[M]/[D]/[EB/OL]、刊名/出版者、
  年卷期页、DOI。
- 补齐修回期间新增引用；核对被引与正文引用编号一致、无悬空引用。
- 中文文献英文对照统一；URL 类文献给出访问日期。

## 四、图表与版权

- 图表最终清晰度达出版要求（矢量优先，位图足够分辨率）。
- 引用他人图表须获授权并标注来源；自绘图确保可编辑源留存。
- 截图/界面图去除无关敏感信息。

## 五、作者与基金信息

- 核对全部作者姓名（中英文）、单位、排序、通信作者与邮箱。
- 基金资助项目名称与编号准确（国家自然科学基金等）。
- 作者简介（如需）齐全规范。

## 六、校样纪律

- 校样阶段**只做勘误**（错字、格式、排版错误），不做内容增删或重写。
- 逐字通读校样，重点核对公式、数字、图表编号、参考文献。
- 在编辑部规定时限内返回校样；标注清晰、集中反馈，避免反复。

## 七、出版事务

- **版权协议**：按编辑部要求签署版权/出版协议。
- **版面费**：如涉及，按编辑部通知办理（具体金额本包标 **待核实**，以编辑部为准）。
- **清样确认**：最终清样确认后不再改动，等待排期见刊。
- 见刊后如需勘误，走编辑部勘误流程。

## 八、定稿自检清单

```text
[ ] 内容锁定，审稿意见全部落实
[ ] 2021 版模板格式完全符合
[ ] 中英文摘要/关键词/中图分类号终校
[ ] 参考文献 GB/T 7714 逐条复核，编号无悬空
[ ] 图表清晰、版权合规、题注规范
[ ] 作者/单位/通信作者/基金信息准确
[ ] 校样只勘误，未做实质改动
[ ] 版权协议、版面费等出版事务已办理
```

## 九、输出格式

```text
【定稿就绪度】就绪 / 需修正
【内容锁定】是否仍有实质改动：________
【格式合规】2021 模板问题：________
【GB/T 7714】参考文献问题：________
【信息核对】作者/基金/通信作者：________
【出版事务】版权协议/版面费/清样：________（待核实项以编辑部为准）
【下一步】返回校样，确认清样，等待见刊
```

## 十、中英文一致性终校（本刊特有的双语核对）

《软件学报》(Journal of Software) 要求中英文并存，定稿时要专门做一遍双语一致性核对，这是
中文期刊定稿区别于英文期刊的关键环节：

- **题目**：中文题目与 English Title 语义一致，英文术语用领域通用写法，大小写规范。
- **摘要**：中文摘要与 English Abstract 传达同一目的/方法/结果/结论，English Abstract 需可
  独立阅读、语法无误；不是中文的机器直译。
- **关键词**：中文关键词与 English Keywords 一一对应、顺序一致、数量相同。
- **术语**：正文中反复出现的核心术语，中英文译名与缩写全文统一，与摘要一致。
- **作者与单位**：中英文姓名、单位名称、通信作者信息一致无误。

建议请一位熟悉该领域英文表达的同行通读 English Abstract，避免见刊后留下英文硬伤。

## 十一、见刊前最后确认

```text
[ ] 清样与录用内容一致，无遗漏修改
[ ] 中英文题目/摘要/关键词双语一致
[ ] 参考文献编号与正文一一对应，无悬空
[ ] 图表清晰、页面无溢出、公式无错位
[ ] 版权协议已签、版面费（如有）已办
[ ] 材料/附录链接稳定有效
```

定稿一旦确认清样即不可再改，务必在返回前逐字通读，尤其核对数字、公式与参考文献。见刊后若
发现须更正的实质性错误，应通过《软件学报》(Journal of Software) 编辑部的正式勘误流程处理，
不要私自散布未经编辑部确认的更正版本。

## 十二、定稿阶段的心态

定稿不是"随便过一遍"，而是把一年多努力锁定成永久记录的最后一道关。此时最容易因松懈而留下
低级错误：作者姓名拼写、基金编号、参考文献页码、图表编号错位，这些在见刊后都难以更改，且
直接影响论文的专业观感与可检索性。建议由不同作者分工核对不同部分（一人查参考文献、一人查
图表、一人查中英文一致性），交叉复核比单人通读更可靠。把定稿当作与投稿同等重要的环节，
才能让《软件学报》(Journal of Software) 上最终呈现的版本干净、可信、经得起长期引用。

> 提醒：版面费金额、校样时限、见刊排期等属 **待核实** 或逐稿而异，请以《软件学报》(Journal
> of Software) 编辑部的正式通知为准。补充材料处理见
