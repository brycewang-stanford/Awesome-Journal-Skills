# 《软件学报》选题与栏目定位 (Journal of Software Topic Selection)

在动笔之前先判断稿件是否"《软件学报》(Journal of Software, JOS) 形态"。本刊是中国科学院软件
研究所与 CCF 联合主办的软件学科中文旗舰，CCF 推荐 A 类。选错刊会浪费一整轮审稿周期——本刊
与兄弟刊学科高度重叠但侧重不同。下列判据据 2026-07-09 官网与 CCF 目录的搜索渲染整理（见
[`resources/official-source-map.md`](../../resources/official-source-map.md)）。

## 一、《软件学报》的学科主线

《软件学报》聚焦**软件学科**，而非全计算机学科综合。核心方向：

- **软件工程**：需求、设计、测试、缺陷预测与定位、缺陷检测与修复、程序分析、软件维护与演化。
- **系统软件**：操作系统、运行时、编译技术与编译器、虚拟化、RISC-V 系统软件与软硬协同。
- **程序设计语言**：语言设计、类型系统、程序验证、形式化方法。
- **数据库与大数据**：数据库系统、查询处理、云原生数据库、数据管理。
- **网络与分布式系统**：分布式计算、云计算、并发与一致性。
- **软件安全**：漏洞感知与挖掘、供应链安全、可信赖软件。
- **AI × 软件交叉**：大模型用于软件质量保障、深度学习用于软件工程任务。

## 二、与兄弟刊的区分测试

| 兄弟刊 | 侧重 | 何时改投它 |
| --- | --- | --- |
| 《计算机学报》(Chinese Journal of Computers) | 全计算机学科综合 | 贡献跨体系结构/理论/AI 等，软件性不是主线 |
| 《计算机研究与发展》 | 计算机综合、系统与应用 | 偏系统工程或综合评述、软件学科纯度不高 |
| 《自动化学报》 | 控制、自动化、模式识别 | 贡献是控制/优化/感知，软件只是实现手段 |
| 《电子学报》 | 电子、通信、信号 | 贡献在器件/通信/信号处理 |
| 《计算机科学》 | 计算机综合（发表面更宽） | 增量型工作或综合方向 |
| 英文会议/期刊（ICSE/FSE/TSE 等） | 国际读者、英文 | 目标国际影响力且以英文写作 |

**判据**：如果把"软件"二字从你的贡献里抽走，贡献是否还成立？若成立（贡献其实是控制/信号/
纯理论），改投相应兄弟刊；若不成立（贡献本质是软件如何被构建/测试/维护/验证），《软件学报》
(Journal of Software) 是主场。

## 三、两个改投信号（route-out signals）

- **模型可替换测试**：若把论文里的深度模型换成另一个，软件工程教益是否仍在？若"模型就是全部
  论文"，这更像纯 AI 工作，应重构为软件学科问题或改投。
- **改标题测试**：若这篇论文换个题目就能变成一篇《自动化学报》或《电子学报》论文而无损失，
  说明软件学科定位不牢，需强化软件工程论点。

## 四、栏目选择

| 目标 | 栏目 | 要点 |
| --- | --- | --- |
| 原创方法/系统/实证 | 研究论文 | 贡献清晰、证据相称、可复现 |
| 梳理领域、建分类体系 | 综述 / 研究进展 | 覆盖全面、有开放问题、成为引用锚点 |
| 契合当前热点主题 | 专刊（special issue） | 走专刊通道，注意截稿与特约编辑 |

## 五、专刊（special issue）通道判断

《软件学报》(Journal of Software) 有稳定的专刊传统，常与 **CCF ChinaSoft 中国软件大会**联动，
由特约（客座）编辑组稿。近年主题包括形式化方法与应用、大模型下的软件质量保障、云原生数据库
技术与系统、编译技术与编译器设计、RISC-V 系统软件及软硬协同、新兴软件与系统的可信赖性与
安全等。判断是否走专刊：

- 你的主题是否正好落在当前开放的专刊征稿范围内？
- 专刊截稿时间是否来得及？（专刊有独立截稿，见征稿页）
- 是否需要先在 ChinaSoft 上作口头报告（部分专刊要求）？
- 专刊评审可能更快聚焦主题，但主题不契合会被直接建议转常规通道。

## 六、动笔前自检清单

```text
[ ] 贡献的核心是"软件如何被构建/测试/维护/验证/保护"？
[ ] 抽掉"软件"后贡献不成立（否则考虑兄弟刊）？
[ ] 模型可替换测试通过（软件工程教益独立于具体模型）？
[ ] 改标题测试通过（不易被无损改写为自动化/电子学报论文）？
[ ] 栏目已定（研究论文 / 综述 / 专刊）？
[ ] 若综述：有明确分类框架与开放问题，而非文献罗列？
[ ] 若专刊：主题契合、截稿前、了解特约编辑通道？
[ ] 已对标 exemplars/library.md 中同形态的《软件学报》论文？
```

## 八、常见误判

- 把一篇纯机器学习/纯优化工作硬套"软件工程应用"外壳——审稿人会识别为改投信号。
- 综述缺乏原创分类框架，只是把已有工作按时间罗列。
- 误把《软件学报》当《计算机学报》的软件子集——两刊独立评审、侧重不同，不可互相假定口径。
- 忽略专刊截稿，错过更契合的快速通道。

## 九、时机与"投中英文哪一边"

- 若目标是国内软件学界的**中文影响力**、参与国内学术共同体讨论，《软件学报》(Journal of
  Software) 是理想选择；若目标是国际读者与英文引用，考虑 ICSE/FSE/TSE/TOSEM 等英文场所。
- 会议工作扩展为期刊版投本刊时，需显著扩展（更完整的方法、更充分的实验、更深的分析）并
  说明与会议版的区别，避免自我重复。
- 专刊有独立截稿，且往往聚焦当下热点（如大模型软件质量、云原生数据库、RISC-V 系统软件），
  若你的工作正当其时，专刊是更快、更聚焦的通道。

## 十、快速决策路由

```text
不确定学科归属 → 做"抽掉软件"与"改标题"两个测试
偏纯AI/优化/控制 → 强化软件工程论点，否则改投自动化学报等
综合性/跨学科强 → 考虑计算机学报/计算机研究与发展
主题正好在开放专刊内且来得及 → 走专刊通道
以上都稳、软件学科纯度高 → 常规研究论文投《软件学报》
```

先跑完这张路由，再进入 [`jos-writing-style`](../jos-writing-style/SKILL.md) 搭骨架，可以避免
在错误的刊上耗掉一整轮审稿周期。

> 提醒：本刊各栏目的具体篇幅口径、专刊当前开放主题、以及影响因子等数值均可能变化，属
> **待核实**项，请以《软件学报》(Journal of Software) 官网当期目录、投稿指南与 ChinaSoft
> 专刊征稿页为准。全流程见 [`jos-workflow`](../jos-workflow/SKILL.md)。
