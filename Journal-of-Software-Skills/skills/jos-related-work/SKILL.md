# 《软件学报》相关工作与文献综述 (Journal of Software Related Work)

相关工作是《软件学报》(Journal of Software, JOS) 审稿人判断"增量是否足够"的地方。本刊读者是
中文软件学界同行，既看你是否覆盖国际前沿，也看你是否引用了本刊与国内相关工作。定位不清、
漏引近期工作，是常见退稿原因（见
[`resources/official-source-map.md`](../../resources/official-source-map.md)）。

## 一、软件学科文献分线

按方向把相关工作组织成清晰的"文献线"，而非一锅乱炖：

- **软件工程**：测试、缺陷预测/定位/检测/修复、程序分析、需求与设计、维护与演化。
- **系统软件**：操作系统、编译、运行时、虚拟化、系统性能。
- **程序设计语言与形式化**：语言/类型、验证、模型检验、定理证明。
- **数据库与大数据**：查询处理、存储、云原生数据库、数据管理。
- **网络与分布式**：一致性、并发、云计算、分布式协议。
- **软件安全**：漏洞挖掘/感知、供应链安全、可信赖软件。

每条线：概述该线做了什么 → 存在什么共同局限 → 你的工作如何补上。

## 二、差异优先 (delta-first) 定位

- 不要写成"编年史"（谁在哪年做了什么），要写成"论证"：**为什么已有工作不足以解决你的问题**。
- 对最接近的 2–3 项工作，逐一说明**关键区别**：问题设定、假设、方法、评测对象哪里不同。
- 用一句话能概括你的增量：如"已有 X 依赖 Y 假设，在 Z 场景失效；我们……"。

```text
定位模板：
  已有工作 A/B/C 通过 ____ 处理 ____，但都 ____（共同局限）。
  与最接近的 A 相比，本文在 ____ 上不同，因而能 ____。
  这一区别在实验 RQ__ 上得到验证。
```

## 三、引用的均衡与完整

- **国际 + 国内并重**：既引 ICSE/FSE/TSE/TOSEM 等国际前沿，也引《软件学报》《计算机学报》等
  中文期刊的相关工作——漏引本刊近期同主题论文是审稿人敏感点。
- 到本刊官网当期/过刊与 CNKI 检索同主题近 2–3 年论文，避免"只引英文、不知国内进展"。
- 引用要准确：核对作者、年份、卷期页；参考文献按 GB/T 7714 著录（见
  [`jos-writing-style`](../jos-writing-style/SKILL.md)）。

## 四、综述类文章的分类框架

若写综述/研究进展（本刊重要栏目）：

- **原创分类体系**是综述的核心贡献，不是文献罗列。给出一张分类图/表，让读者建立地图。
- 每类：代表工作、共性方法、评价方式、局限。
- 明确**开放问题**与未来方向，使综述成为后续工作的引用锚点。
- 覆盖要全面且更新到近年，说明检索范围与纳入标准（可借鉴系统综述方法）。

## 五、常见问题

- 相关工作与引言重复，没有新增信息。
- 只罗列不比较，读者看不出你的位置。
- 漏引最接近的竞争工作（审稿人往往就是被漏引者）。
- 只引英文顶会、不引本刊与国内相关工作。
- 综述无分类框架、无开放问题，沦为文献综述作业。

## 六、自检清单

```text
[ ] 相关工作按软件学科方向分线组织
[ ] 对最接近的 2–3 项工作逐一说明关键区别
[ ] 增量能用一句话概括，并在实验中验证
[ ] 国际与国内文献均衡，未漏引本刊近期同主题论文
[ ] 参考文献 GB/T 7714 著录准确完整
[ ] 若综述：有原创分类框架 + 开放问题
```

## 七、输出格式

```text
【相关工作就绪度】就绪 / 需补 / 重组
【文献分线】覆盖方向：________；缺口：________
【差异定位】与最接近工作的区别是否清晰：________
【引用均衡】是否漏引本刊/国内近期工作：________
【综述专属】分类框架是否原创、有无开放问题：________
【下一步】用 jos-writing-style 打磨表达 / 用 jos-experiments 验证增量
```

## 八、检索与覆盖策略

要让《软件学报》(Journal of Software) 审稿人相信你的覆盖是全面的，检索要有章法：

- **多库检索**：国际用 dblp、ACM DL、IEEE Xplore、Google Scholar；国内用 CNKI、万方、维普。
  中英文关键词都要检索，覆盖同一概念的不同译名。
- **近三年优先**：重点覆盖近 2–3 年工作，尤其本刊与兄弟刊的同主题论文；漏引近期工作最易
  被审稿人（往往就是作者本人）指出。
- **顺藤摸瓜**：从最接近工作的参考文献与被引列表向前后追溯，补全文献线。
- **纳入/排除标准**：综述类要写明检索范围、时间窗、纳入与排除标准，使覆盖可复核。

## 九、把相关工作与贡献绑定

- 相关工作不是孤立章节：每条"已有工作的局限"都应对应你论文的一个贡献或一个实验 RQ。
- 引言里点出的"现状不足"要在相关工作里展开、在实验里验证，形成"问题—定位—证据"闭环
  （见 [`jos-writing-style`](../jos-writing-style/SKILL.md) 与
  [`jos-experiments`](../jos-experiments/SKILL.md)）。
- 收尾用一小段把"本文相对全部已有工作的位置"再点一次题，便于审稿人形成整体印象。

## 十、快速对照

```text
[ ] 每条文献线都有"共同局限 → 本文如何补"
[ ] 与最接近工作的区别有实验证据支撑
[ ] 中英文多库检索、覆盖近三年
[ ] 未漏引本刊/兄弟刊同主题近期论文
[ ] 综述写明纳入/排除标准与分类框架
```

一段好的相关工作，读完能让《软件学报》(Journal of Software) 审稿人清楚回答三个问题：这个
问题此前被谁、用什么方式处理过；这些处理共同缺了什么；本文恰好补上了哪一块。做到这三点，
"增量是否足够"的疑虑往往就迎刃而解。
