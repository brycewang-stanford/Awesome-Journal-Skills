# External Tools - ICALP (EATCS)

Access date: 2026-07-09

Use these surfaces after reopening the current ICALP cycle pages. ICALP mechanics — the single
February deadline, the 15-page + appendix format, which track carries a rebuttal, the LIPIcs
production timeline — are decided per edition, and several of these hosts (the Royal Holloway site,
`callforpaper.org`, `wikicfp.com`, the mailing-list archives) may return 403 to a direct fetch (read
via search renderings and cross-check dblp and DROPS). ICALP is EATCS's European theory flagship;
do not import STOC/FOCS (ACM) or SODA (SIAM) rules.

## Cross-check surfaces (when the gateway blocks the primary page)

- dblp ICALP proceedings: https://dblp.org/db/conf/icalp/index.html (and per-year, e.g. `icalp2026.html`).
- DROPS/Dagstuhl LIPIcs ICALP volumes for the published version, page counts, and proceedings editors.
- The TYPES-announce and DMANET mailing-list postings of the ICALP CFP (machine-readable dates the
  glossy site paraphrases).
- ECCC (`eccc.weizmann.ac.il`) and arXiv (`cs.CC`, `cs.DS`, `cs.LO`, `cs.FL`) for the full versions
  that back Track A / Track B submissions.

## Author-side checks

- HotCRP profile on the **correct track server** (Track A vs Track B are separate HotCRP sites),
  conflicts, coauthor emails, and the **abstract-registration** step that precedes the full-paper
  deadline by a few days.
- Format: an extended abstract **≤15 pages excluding references and a clearly labelled appendix**;
  the appendix holds omitted proofs or the full version. Confirm whether the LIPIcs class is required
  at submission this cycle (optional in recent cycles; mandatory for the camera-ready).
- **Lightweight double-blind** sweep: no author names/affiliations/acknowledgements, third-person
  self-citation, no identifying links, and an arXiv posting that does not undercut anonymity.
- Student-only authorship flagged at submission for the **best student paper** award.
- No prior publication and no concurrent submission elsewhere.

## Do not infer

- Do not assume both tracks behave identically: **Track B has a rebuttal**; **Track A** contacts
  authors only for correctness issues. Confirm per cycle.
- Do not assume the page limit, the mandatory-class rule, the rebuttal window, or any AI-disclosure
  wording carry over between editions.
- Do not treat ICALP like STOC/FOCS/SODA: it is EATCS/LIPIcs, single-deadline, two-track, and
  open-access — a different ecosystem with different norms.
- If pages disagree, treat the newest CFP, the track HotCRP deadline page, or a direct PC-chair
  announcement as controlling, and record the conflict.
