# 政策含义（cte-policy-implication）

## 触发时机

- 文末政策建议只有"建议政府重视财税 / 金融改革"
- 政策建议是"加强、完善、推进、深化"四件套套话（本刊退稿雷区"政策含义空泛"）
- 审稿人指出"政策含义太薄 / 与本文证据脱节"

## 本刊政策含义的定位

《财贸经济》是财经政策导向的顶级刊，政策含义分量重，但**分量重 ≠ 写成部委备忘录**。定位是**"意义层 + 具体政策抓手"**：既要有对财税体制 / 金融改革 / 扩内需等的制度含义，又要**落到具体的政策对象与环节**（哪一类主体、哪一个政策工具、哪一档参数），与本文实证证据严格挂钩。

| 维度 | 合格（本刊偏好） | 不合格 |
|---|---|---|
| 层面 | 制度 / 原理层，但点明财经政策对象 | 笼统"政府应重视" |
| 落点 | 哪类主体 / 哪个工具 / 哪档参数 | 不指对象的"加强完善推进" |
| 依据 | 与本文机制 / 异质性结论呼应 | 与本文证据无关的泛泛建议 |
| 收尾 | 与理论贡献 + 重大财经战略呼应 | 复述政府工作报告 |

## 结构

### 三段式（推荐）

```
本文的研究具有以下政策含义。

第一，[原理层 + 政策落点]——本文揭示了[机制]，提示在[税制 / 财政转移 / 货币 / 监管 / 贸易等]政策设计中应重视……。
结合本文异质性发现，对[某类主体 / 某类地区]而言尤其如此。

第二，[宏观 / 战略层]——本文发现……，这意味着[财税体制改革 / 金融供给侧改革 / 扩大内需 / 共同富裕]在……方面仍有改进空间。

第三，[工具 / 参数层]（如适用）——就[某项政策工具的门槛 / 力度 / 覆盖范围]而言，本文提示……。
```

### 单段式（如果文章偏理论）

```
本文为理解中国[财政 / 金融 / 贸易]领域的[现象 / 制度]提供了新的证据。
本文发现……，这一结果意味着[制度意义]，对[财税体制 / 金融改革 / 相关理论]具有……启示。
```

## 必查清单

- [ ] 政策含义与本文实证证据 / 机制 / 异质性紧密挂钩
- [ ] 不出现"加强 / 完善 / 推进 / 深化"四件套撑段落
- [ ] 含义落到**具体政策对象 / 工具 / 参数**（哪类主体 / 哪个工具 / 哪档门槛），不是笼统"政府"
- [ ] 与引言中的"研究意义"段相呼应
- [ ] 呼应财税体制改革 / 金融改革 / 扩内需 / 共同富裕等战略议题，但不止于复述口号

## 反模式

- 政策建议用"加强、完善、推进、深化"四个动词撑满段落
- 把政策含义写成"政府工作报告"或"改革文件"的延伸
- 含义与本文研究无关（研究的是减税，建议却谈金融监管）
- 文末才提到中国财经制度背景 —— 制度意识应贯穿全文
- "本文是首次……" —— 这是贡献，不是政策含义

## 输出格式

```
【段数】X
【层级覆盖】原理 / 宏观战略 / 工具参数
【政策落点】明确（哪类主体 / 工具 / 参数）/ 笼统
【与本文证据挂钩度】高 / 中 / 低
【四件套套话命中】X 处
【下一步】cte-abstract
```

## 《财贸经济》操作审查

先锁定财经问题、政策 / 制度场景、识别链条、机制证据和可执行含义，再判断稿件是否回应财经审稿人通常同时追问的问题意识、政策场景、识别可信度与制度机制。

- **政策含义审查**：明确政策主体、作用边界、福利或实施权衡，并把每条政策建议落到已展示的识别或机制证据上。
- **决策账本**：返回"主张 / 证据 / 阻断点 / 下一处改稿"四列，便于下一轮直接改稿。
- **改投比较**：对照《经济研究》用于更一般理论贡献，《金融研究》用于纯金融机制，《财政研究》《税务研究》用于更专门财税政策；若相邻刊物读者匹配更强，先建议改投而非继续润色。
- **核验底线**：给投稿就绪判断前，必须重开 `resources/official-source-map.md`，列出仍可能改变建议的一个未核实事实。
