# 语言风格（cte-style）

## 触发时机

- 通读初稿后觉得"读起来像财经政策文件 / 工作汇报"
- 一段里出现 ≥ 2 个"具有重要意义" / "为高质量发展提供新思路"
- 政策建议读起来像"加强、完善、推进、深化"四件套
- 表格只报数字，没有落到主体的经济含义解读

## 对照表（黑名单 → 白名单）

| 问题类型 | 黑名单（删/改） | 白名单（替换为） |
|---|---|---|
| 空洞价值 | "具有重要的理论价值" | "为理解 X 提供了 Y 视角下的[企业 / 市场]行为框架" |
| 空洞意义 | "对经济发展具有重要参考价值" | "本文结果意味着 X 政策应将 Y 类主体纳入瞄准范围" |
| 四件套建议 | "应加强 / 完善 / 推进 / 深化……" | "X 部门应在 Y 环节针对 Z 类主体建立……机制" |
| 方法炫耀 | "本文使用复杂的因果识别方法" | "本文利用改革试点的分批推行构造交叠 DID，识别 X" |
| 贡献模糊 | "丰富了财经研究" | "首次利用企业面板识别了 X 政策对 Y 的因果效应" |
| 文献堆砌 | "（张三，2020；李四，2021；……）" | 按贡献分述：理论文献一段、财经实证一段、本文位置一段 |
| 表格无解读 | "表 3 报告了基准回归结果。" | "表 3 显示，减税使企业投资提高约 X%，约相当于样本均值的 Y%" |
| 机制空泛 | "促进了高质量发展" | "通过缓解企业融资约束、降低要素成本实现" |

## 行文规范

### 时态

- 描述本文方法 / 数据 → **现在时**：本文使用 / 本文构造
- 报告实证结果 → **过去时**：研究发现 / 我们检验了
- 解释结果的经济含义 → **现在时**：这意味着 / 这一发现表明

### 人称

- 避免"笔者""我们认为"出现在结论性段落
- 可用："本文发现 / 研究表明 / 结果显示"
- 英文版避免 "We feel that..." / "It is believed that..."

### 谦逊与边界

- 不要写"本文证明了"——经验研究只能说"本文提供了证据支持"
- 不要把统计显著当作经济显著——财经研究尤其要报告**经济量级**（如相当于均值的百分比、相当于多少元 / 多少个百分点）

## 一段一概括

每个段落第一句必须是**概括句**，后续句子是支撑。审稿人快速读完每段首句应能复原文章逻辑。

错误：
> 张三（2020）研究发现减税……。李四（2021）发现融资约束……。王五（2022）发现投资……。

正确：
> 既有研究就减税对企业行为的影响存在分歧（张三，2020；李四，2021）。一类研究强调……，另一类研究强调……。两者的分歧在于对政策内生性的不同处理。

## 行文自检清单

- [ ] 每段首句是概括性陈述
- [ ] 黑名单短语命中数 = 0
- [ ] 政策建议指明了"谁应该对哪类主体做什么"，无四件套套话
- [ ] 表格后有落到主体的经济含义解读，不仅报数字
- [ ] 机制阐述落到企业 / 银行 / 居民 / 市场可观测行为，不停在宏观叙事
- [ ] 时态、人称在全文统一
- [ ] 显著性与经济量级同时报告

## 反模式

- 通篇"具有重要的 X 意义" —— 一次都不应出现
- 政策建议用"加强、完善、推进、深化"四个动词撑满段落
- 方法描述长于结果阐释
- 机制和政策含义全是宏观口号，不见一个具体主体
- 引言里塞一长串并列文献 —— 拆开分述

## 输出格式

```
【黑名单命中】X 处，分别在：[...]
【四件套套话命中】X 处
【段首概括率】X / 总段数
【政策建议落点】明确到主体 / 笼统
【经济含义阐述】到位 / 缺失 [...]
【时态/人称一致性】一致 / 待统一
【下一步】cte-submission
```

## 《财贸经济》操作审查

先锁定财经问题、政策 / 制度场景、识别链条、机制证据和可执行含义，再判断稿件是否回应财经审稿人通常同时追问的问题意识、政策场景、识别可信度与制度机制。

- **文风审查**：把问题意识、识别证据、机制发现和政策含义放在段首主线中，删掉不改变本刊适配性的泛化表述。
- **决策账本**：返回"主张 / 证据 / 阻断点 / 下一处改稿"四列，便于下一轮直接改稿。
- **改投比较**：对照《经济研究》用于更一般理论贡献，《金融研究》用于纯金融机制，《世界经济》用于开放宏观 / 贸易；若相邻刊物读者匹配更强，先建议改投而非继续润色。
- **核验底线**：给投稿就绪判断前，必须重开 `resources/official-source-map.md`，列出仍可能改变建议的一个未核实事实。
