# EJ Pre-Submission Checklist

Verify the *current* EJ author guidelines, review model, and RES/EJ data-and-code policy on the
official OUP and Royal Economic Society pages before relying on any limit below. This list
captures the durable structure of a The Economic Journal submission via Editorial Express
(source refresh: 2026-06-20).

## 1. Format
- [ ] Manuscript is a **single PDF** including appendices (cover letter and past reports are separate)
- [ ] Section order is logical: introduction → model/theory → data & setting → empirical strategy → results → mechanisms/robustness → (counterfactuals) → conclusion
- [ ] Equations numbered; referenced in text as "equation (n)"
- [ ] Tables are self-contained (sample, period, unit, DV definition, specification, clustering level in notes)
- [ ] Figures legible in grayscale, axes labeled with units, confidence bands shown
- [ ] Short paper? within ~6,000 words and ~5 exhibits (AER:Insights style); full-length? exposition complete

## 2. References (house style)
- [ ] **Author-date** citations throughout — NOT numbered/bracket citations
- [ ] Reference list complete, alphabetical then chronological for each author
- [ ] Dataset and replication-package citations included where relevant
- [ ] Every in-text citation appears in the list; every list entry is cited at least once

## 3. Economic argument & broad interest (substance gate)
- [ ] The contribution is substantial and **of broad interest to economists at large** (not subfield-only)
- [ ] An explicit economic mechanism or model is stated, not just a net effect
- [ ] The estimated object has a clear economic interpretation
- [ ] General-equilibrium / selection / external-validity considerations addressed or bounded
- [ ] The leading rival mechanism is named and tested against
- [ ] The portable economic lesson is stated, not assumed

## 4. Review model (EJ is single-blind — verify current policy)
- [ ] **No anonymization needed** — EJ review is single-blind, so author identity is expected
- [ ] Title page includes authors, affiliations, acknowledgments, and **funding sources**
- [ ] Exposition checked so a generalist economist can follow the argument (EJ values clear exposition)

## 5. Data & code policy (RES / EJ Data Editor)
- [ ] RES/EJ data-and-code policy reviewed (DCAS-endorsed; EJ Data Editor reproducibility check)
- [ ] Replication package staged for the pre-acceptance check; deposit destined for **Zenodo**
- [ ] Restricted/proprietary data: **exemption requested at first submission**
- [ ] All variables documented; proprietary-format data also provided as ASCII/plain text
- [ ] Paper not under review elsewhere; no overlapping submission

## 6. Exhibits & supplementary
- [ ] Main-text exhibits report economic magnitudes (mean-relative / elasticity / welfare), not stars alone
- [ ] Number of main-text exhibits disciplined; heavy material moved to the online appendix
- [ ] Online appendix referenced from the main text and prepared
- [ ] Every exhibit number reproduces from the deposited code
- [ ] Restricted-data documentation prepared if applicable

## 7. Portal metadata (Editorial Express)
- [ ] Title and abstract entered; accepted-author abstract cap is 100 words
- [ ] JEL classification codes entered
- [ ] Keywords entered; accepted-author keywords are 20 characters or fewer each
- [ ] All co-authors and affiliations correct; corresponding author set
- [ ] Suggested referees (and any opposed referees) prepared with brief justification

## 8. Final pass
- [ ] Cover letter drafted (one page: question, contribution, fit with EJ's broad-interest mandate)
- [ ] Full-length vs. short-paper route confirmed
- [ ] New-submission fee category/payment path ready; resubmission fee exemption understood if applicable
- [ ] Manuscript number / acknowledgment email expected after submission
- [ ] Replication package re-run on a clean machine (ready for the EJ Data Editor)
