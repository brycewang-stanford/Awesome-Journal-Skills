# 管理启示（jmsc-managerial-insights）

## 触发时机

- 模型、命题、数值都有了，要写"管理启示/管理含义"一节
- 启示写成了"应加强、应完善、应推进"等政策口号
- 审稿质疑"管理含义与模型脱节 / 没有可操作的决策规律"
- 把别刊（管理世界）的政策建议腔搬了过来

## 核心：启示是从模型导出的决策规律

本刊的管理启示**必须由模型/命题/数值推出**，形态是**可操作的决策规律**："当参数 X 落在某区间时，最优策略是……"。它不是政策口号，也不是泛泛的"对企业有指导意义"。

## 决策规律的写法

每条启示绑定到具体结论，模板：

> **在【某条件/参数区间】下，决策者应【某动作】，因为命题 P/实验显示【某机制】。**

例：当需求波动 σ 超过阈值 θ 时，应采用提前承诺契约而非批发价契约，因为命题 3 证明前者此时 Pareto 占优。

## 启示来源对照表

| 模型产物 | 对应的管理启示 |
|----------|----------------|
| 阈值型最优策略 | "参数过/不过阈值"时切换决策 |
| 比较静态（单调） | 某参数上升时，最优决策朝某方向调整 |
| 均衡刻画 | 多主体下的协调/竞争后果与对策 |
| 敏感性拐点 | 哪些参数最值得管理者投入精力测量/控制 |
| 算法可扩展边界 | 实务中规模多大时该用近似/分解 |

## 自检清单

- [ ] 每条启示对应一条具体命题或实验结果（可溯源）
- [ ] 启示是"在何条件下做何决策"的规律，不是口号
- [ ] 用到了模型参数（阈值/区间/方向），而非泛泛而谈
- [ ] 区分了"模型证明的"与"数值提示的"，不夸大
- [ ] 没有超出模型假设范围的外推
- [ ] 命中政策口号黑名单 = 0

## 政策口号黑名单（命中即改写）

- "应加强 / 完善 / 推进 / 重视 / 优化（无具体决策内容）"
- "对企业（政府）具有重要的指导意义"
- "为……提供了理论支持 / 决策参考"（不说提供了什么规律）
- 把"政府应制定政策"当管理启示（本刊要决策规律，不是政策建议）

## 反模式

- 启示与模型无对应，谁的论文都能套
- 把敏感性的每个图都翻译成一句"……有影响"，无规律提炼
- 越过假设把局部结论说成普遍真理
- 用政策建议腔替代决策规律

## 本刊管理启示审稿期待与退稿模式

《管理科学学报》要的管理启示是"从模型可追溯导出的决策规律"，而非《管理世界》式政策建议。下表对齐本刊高频退稿语与修法：

| 退稿信号 | 根因 | 本刊期望的修法 |
|----------|------|----------------|
| "管理启示空泛" | 写成"加强完善推进"口号 | 改为"参数落某区间时最优策略如何"的规律 |
| "与模型结果脱节" | 启示无命题支撑 | 每条启示标注其来源命题 P? 或实验图 |
| "把政府政策当管理启示" | 搬经验刊政策腔 | 回到决策者视角，给企业可执行的决策动作 |
| "外推超出假设" | 把局部结论说成普遍真理 | 限定在模型假设范围内陈述，标出适用边界 |
| "每个图翻成一句'有影响'" | 无规律提炼 | 合并为少数几条带阈值/方向的决策法则 |

> 锚点：本刊已刊论文的管理启示节通常 3–5 条，每条形如"当参数 X 越过阈值 θ 时，决策者应由策略 A 切到策略 B（依据命题 k）"，并区分证明给出的与数值提示的。具体体例以编辑部最新稿约为准。

## 微型走查：供应链契约模型的启示导出

虚构稿件《需求波动下回购与收益共享契约的选择》。从命题/数值导出启示，走一遍（示意数字仅作演示）：

- **来源命题 P3**：当需求变异系数 CV>θ=0.45 时，收益共享契约的供应链总利润超过回购契约。
- **导出启示 1**（结构性，证明给出）：当需求波动较大（CV>0.45）时，制造商应主动提议收益共享契约而非回购契约，因命题 P3 证明前者此时使供应链利润 Pareto 改进约 6%。绑定 P3、参数化、未外推。
- **来源敏感性**：数值显示共享比例 φ 的最优值随零售商私有信息精度上升而下降。
- **导出启示 2**（数值提示，标注）：零售商预测越准，制造商越应让出更高利润份额（降低 φ）以换信息共享——此条由实验趋势提示，非解析证明，明示区分。
- **口号黑名单自检**：全文无"应加强/完善/推进"、无"对企业具有指导意义"，命中 0 处。
- **外推风险**：启示限定在两级供应链、单期、风险中性下，未推广到多级或风险厌恶，标出边界。

审稿人若追问"6% 改进是否稳健"，回应应指向敏感性结果（CV∈[0.45,0.8] 区间改进 4%–9%、始终为正），而非笼统说"显著改善"。

## 输出格式

```
【启示1】条件<参数区间> → 决策<动作>（依据 P?/实验?）
【启示2】…
【可溯源】每条对应命题/实验？是/否
【参数化】用了阈值/区间/方向？是/否
【口号黑名单命中】N 处 <列出>
【外推风险】无 / 有 <哪条>
【下一步】jmsc-notation-style / jmsc-submission
```
