# 复现与自查脚本思路 — 《模式识别与人工智能》(Pattern Recognition and Artificial Intelligence, PR&AI)

本目录把《模式识别与人工智能》(Pattern Recognition and Artificial Intelligence, PR&AI) 作者常用的
复现与投稿前自查动作沉淀为可脚本化的思路。它刻意保持精简：本刊的**政策**（初审→外审→终审、6 个月
审稿口径、GB/T 7714、重复率检测、代码数据可用性现状）在各 `SKILL.md` 与
[`../official-source-map.md`](../official-source-map.md) 中；这里只放**跨稿件复用的工具性检查**。

## 一、复现包冒烟检查（思路）

模式识别与人工智能类论文常含训练/推理流水线，投稿前做一次"干净环境冒烟测试"：

```bash
# 1) 从零构建
python3 -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
# 2) 一键复现主结果（固定随机种子）
bash scripts/run_all.sh --seed 20260709
# 3) 核对产物是否对上论文表图（人工比对 results/ 与论文表3/图4）
```

要点：固定随机种子、记录 GPU/CPU 型号与训练耗时、把大数据集与预训练权重走稳定托管而非塞进仓库
（见 [`../../skills/prai-reproducibility/SKILL.md`](../../skills/prai-reproducibility/SKILL.md)）。

## 二、投稿前自查清单（脚本化思路）

- **查重**：投稿前用查重工具把总文字复制比压到编辑部可接受线以下，重点核对自引与会议版本重叠
  （PR&AI 编辑部对稿件做重复率检测，未通过退回作者）。
- **匿名化（若本刊要求作者信息与正文分离）**：清理 PDF 文档属性、致谢、基金显名与图源水印；扫描
  仓库中的用户名、邮箱、机构路径。
  ```bash
  # 示意：检查提交物中可能泄露身份的痕迹
  grep -rniE '(@[a-z0-9.]+\.(edu|ac)\.cn|/home/|/Users/|课题组|实验室)' artifact/ | head
  ```
- **参考文献**：核对 GB/T 7714 著录、中文文献中英文对照、正文序号与文末列表一一对应。
- **首页部件**：中文题名+英文题名、中文摘要(约 200 字)、英文摘要(约 200 词)、中英文关键词(3~5)、
  中图分类号(TP)、基金、作者简介是否齐全。
- **字数**：全文控制在 18000~44000 字区间（含图表）。

## 三、本刊特有的检查（通用工具无法替代）

- **AI 细分方向的评测严谨性**：模式识别/机器学习论文需报告多次运行的均值±标准差、公平基线、消融、
  统计显著性；榜单式单点数字不足以支撑主张（见
  [`../../skills/prai-experiments/SKILL.md`](../../skills/prai-experiments/SKILL.md)）。
- **数据泄漏与评测污染**：训练/验证/测试划分固定、无标签泄漏、预训练语料与评测集不重叠。
- **会议扩展版声明**：若为已发表会议论文的期刊扩展版，显著声明扩展关系并规范引用会议版本、说明
  实质增量（新增理论/实验/系统模块）。
- **可用性声明诚实**：与实际提供的制品一致；不可公开部分说明原因与替代核验途径。

本目录不是计量经济学/Stata 套件，PR&AI 的证据也不是排行榜分数：请勿把 DiD notebook 或榜单式框架
搬进来。政策仍以官网当期《投稿指南》与各技能为准。
