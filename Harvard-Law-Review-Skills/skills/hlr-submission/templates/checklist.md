# HLR Pre-Submission Checklist

Verify the *current* Harvard Law Review submission guidance — length, channel, fee, and expedite/
seven-day policy — on the official submissions page before relying on any item below. Volatile specifics
are marked **(检索于 2026-06；以官网为准)**. This list captures the durable structure of a near-final
law-review submission to HLR's **own electronic submission system** (peer journals use Scholastica/ExpressO).

## 1. Manuscript readiness (HLR expects near-final)
- [ ] Argument is complete — this is not a draft you plan to finish during editing
- [ ] Introduction front-loads the problem, claim, contribution, and payoff (see `hlr-writing-style`)
- [ ] Body follows doctrine → theory → prescription, each Part with one job (see `hlr-argument-structure`)
- [ ] Length within HLR limits — Articles ≤ 25k (over 30k weighs against; over 37.5k rarely accepted),
      **counting text, footnotes, and appendices**; Essays 12k–17.5k
- [ ] **Anonymized for review**: name/affiliation/acknowledgments on a separate cover page; title on the
      first text page; self-citations neutralized
- [ ] Piece type is correct: Article / Essay / Book Review (Notes are student-written, unsigned)

## 2. Originality (preemption)
- [ ] Preemption check run across SSRN, Westlaw/Lexis, HeinOnline, Google Scholar (see `hlr-preemption-check`)
- [ ] Claim distinguished from the nearest prior work; any "first to" language is supported
- [ ] Preemption re-run close to submission (SSRN updates daily in season)

## 3. Footnotes & Bluebook (the make-or-break)
- [ ] Footnotes conform to **The Bluebook, 22nd edition**
- [ ] Every proposition pincited to the strongest available authority (see `hlr-sources-and-bluebook`)
- [ ] Bluebook signals (See, Cf., But see, Contra) match the relationships
- [ ] Short forms (Id., supra) correct and stable
- [ ] Academic law-review typeface conventions applied
- [ ] Authorities within each footnote Bluebook-ordered

## 4. Source-pull readiness
- [ ] Source file assembled: a copy/locator of every cited source with the cited page marked
- [ ] Each footnote re-read against its source page (proposition actually supported)
- [ ] Every quotation verified character-for-character and pinned
- [ ] Web sources archived with access dates (link rot)

## 5. Submission package (HLR's own submission system)
- [ ] Manuscript file (Word) with footnotes prepared
- [ ] Anonymized cover page kept separate from the manuscript body
- [ ] Abstract written
- [ ] CV / author information ready as the portal requires
- [ ] Any fee confirmed on the live page — not assumed (检索于 2026-06；以官网为准)

## 6. Placement & timing
- [ ] Submission set tiered: reach (incl. HLR) / match / floor (see `hlr-placement-strategy`)
- [ ] Submitting early in a season (Feb-Mar main window or smaller Aug window)
- [ ] Wave plan decided (broad at once, or core-then-widen)
- [ ] Expedite mechanics understood: expedite only with an offer in hand
- [ ] Seven-day decision window and offer deadlines will be tracked (检索于 2026-06；以官网为准)

## 7. Final pass
- [ ] Original, unpublished work; no overlapping commitment
- [ ] All cross-references and footnote numbers resolve
- [ ] Confirmation email retained (it carries the unique expedite link)
