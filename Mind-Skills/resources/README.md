# Mind Resources

Resources for the *Mind* (Oxford University Press / Mind Association) skill pack. The `skills/` give
advice; this directory lets an agent **model** a Mind-style section and **benchmark** against verified
exemplars. Pair these with the relevant `skills/mind-*/SKILL.md`.

**No code kit.** *Mind* is an **analytic-philosophy** journal — the paper *is* the argument, with no
data, statistics, or code. So, unlike the empirical-economics packs, this directory deliberately ships
**no `code/` pipeline**. The resources here are about building and positioning an *argument*.

## What's here

| Resource | Use it to... |
| --- | --- |
| [`worked-examples/01-introduction.md`](worked-examples/01-introduction.md) | See a before→after rewrite of an analytic-philosophy article **introduction** in Mind house style — a sharp deniable thesis, an explicit premises→conclusion argument, engagement with the target view, and the strongest objection anticipated. Illustrative **fictional** paper; rigorous prose, not data. |
| [`exemplars/library.md`](exemplars/library.md) | Benchmark against **real, web-verified Mind papers** (DOI `10.1093/mind/...`) organized by area (metaphysics / language / logic / philosophy of mind / ethics). Dialectical positioning only — no paraphrased arguments, with a sibling-journal "do-not-misattribute" list. |
| [`official-source-map.md`](official-source-map.md) | **Mind-specific policy & facts:** ScholarOne submission, ~8,000-word article limit, triple-anonymous review, 50–200-word abstract, scope/history, and the authoritative source map. |
| [`external_tools.md`](external_tools.md) | External tools and services referenced by the pack (PhilPapers, SEP, JSTOR, argument-mapping, LaTeX `lineno`, reference managers). |

## Suggested workflow

1. Frame the contribution with [`../skills/mind-topic-selection`](../skills/mind-topic-selection/SKILL.md)
   and position it with
   [`../skills/mind-literature-positioning`](../skills/mind-literature-positioning/SKILL.md).
2. Build the argument with
   [`../skills/mind-thesis-and-argument`](../skills/mind-thesis-and-argument/SKILL.md); stress-test it
   with [`../skills/mind-objections-and-replies`](../skills/mind-objections-and-replies/SKILL.md).
3. Arrange and polish with
   [`../skills/mind-structure-and-exposition`](../skills/mind-structure-and-exposition/SKILL.md) and
   [`../skills/mind-writing-style`](../skills/mind-writing-style/SKILL.md); model the introduction on
   [`worked-examples/01-introduction.md`](worked-examples/01-introduction.md).
4. Benchmark against verified Mind papers in [`exemplars/library.md`](exemplars/library.md); confirm
   submission policy, word limit, and review process in
   [`official-source-map.md`](official-source-map.md).
