# Language (LSA) Submission Preflight Checklist

Print this and clear every box before uploading to **Cambridge University Press ScholarOne**. Verify
volatile items (word limits, abstract length, accepted formats, APC/open-access terms, current
masthead, data policy) on the live CUP/LSA author pages immediately before submission — see
[`../../../resources/official-source-map.md`](../../../resources/official-source-map.md).

## 1. Section & length
- [ ] Section chosen: General Research Article / Research Report / Review Article / Discussion Note /
      Perspectives / Phonological Analysis / Language & Public Policy / Teaching Linguistics
- [ ] General Research Article word count computed on the **inclusive** rule (notes, charts, tables,
      appendices count; **references excluded**) and ≤ **18,000 words**
- [ ] If over **12,000 words**, the length is genuinely earned (stricter review applies)
- [ ] Review Article ≤ **5,000 words** (excluding references), if applicable

## 2. Double-anonymous anonymization
- [ ] No author names, affiliations, or institutional identifiers in the manuscript
- [ ] Acknowledgments, funding notes, and thanks removed from the manuscript body
- [ ] Self-citations neutralized (e.g., "Author 2021", no "in my earlier work…")
- [ ] File metadata / document properties stripped
- [ ] Separate title/cover page prepared if the portal requires it

## 3. Abstract & framing
- [ ] Abstract present, within the current word target (~150 words — verify)
- [ ] States the question, the data, and the finding
- [ ] Introduction front-loads the theoretical claim and its key prediction (not just description)
- [ ] The paper reads for a general linguistics audience, not one framework's insiders

## 4. Examples, glossing & transcription
- [ ] All examples numbered consecutively; sub-parts lettered (1a), (1b)
- [ ] Interlinear glosses follow the **Leipzig Glossing Rules**; object and gloss lines aligned
- [ ] Grammatical labels in small caps; non-standard abbreviations listed
- [ ] IPA in a Unicode font; phonemic `/ /` vs. phonetic `[ ]` used consistently
- [ ] Each non-elicited token has a source; judgment marks (`*`, `?`, `#`) consistent

## 5. Analysis & evidence
- [ ] Quantitative claims use appropriate models (mixed-effects where the design is clustered)
- [ ] Effect sizes and intervals reported, not stars alone
- [ ] No pseudoreplication (by-subject / by-item structure respected)
- [ ] Rival framework(s) engaged; the adjudicating prediction stated and tested

## 6. Exhibits & style
- [ ] Figures have alt text / descriptive captions; colorblind-safe; vector where possible
- [ ] Citations and reference list in the **Language / Unified Style Sheet** form (NOT APA/Chicago)
- [ ] Trees / tableaux / diagrams numbered and legible in grayscale

## 7. Data & ethics
- [ ] Data-availability statement included and accurate (no over-stated mandate)
- [ ] Analysis data + code shared where ethical (e.g., OSF); versions pinned, seeds set
- [ ] Consultant/community consent and anonymization respected; restricted access declared honestly

## 8. Ethics & logistics
- [ ] Not under review at another journal (no concurrent submission)
- [ ] Open-access / APC / waiver terms checked for Volume 102+ (fully OA from 2026)
- [ ] Overleaf/CUP template used or matched; accepted file format confirmed
- [ ] All co-authors approve the submission

> Final gate: if any box is unchecked, do not submit. Re-run `lang-submission` on the specific gap.
