# 表格与图形（mw-tables-figures）

## 触发时机

- 主表 8 列以上，审稿人吐槽"信息密度过高"
- 表格用了边框 / 阴影 / 颜色 —— 不符合《管理世界》规范
- 图形分辨率不足 / 配色不专业

## 表格规范（三线表 / booktabs）

### 必备结构

```
表 3  政策冲击对企业研发投入的影响
═══════════════════════════════════════════
                  (1)      (2)      (3)
变量           研发支出  研发支出  研发支出
───────────────────────────────────────────
政策×处理后  0.045***  0.038***  0.040***
              (0.012)   (0.011)   (0.013)
              ......
───────────────────────────────────────────
控制变量          否       是       是
个体固定效应       是       是       是
年份固定效应       是       是       是
观测值          10,234   10,234   10,234
R²               0.213    0.247    0.251
═══════════════════════════════════════════
注：括号内为聚类于企业层面的稳健标准误；***、**、* 分别表示 1%、5%、10% 显著水平。
```

### 严守约束

- **不超过 6 列**（极端情况 7 列，绝不 8 列）—— 列多就拆表
- **三线表**：顶线粗、中线细、底线粗，无竖线
- 显著性标记 `*** ** *` 统一用三档
- 标准误在系数下方括号内，**不要用 t 值或 z 值**
- 控制变量行写"是 / 否"，不要写出系数（除非论文核心是控制变量）
- 表号 + 表名在表上方，注释在表下方
- 注释**必须**包含：聚类层次、显著性符号定义

### 表格常见反模式

- 把系数写到小数点后 4–6 位（保留 3 位）
- 注释里"如表 3 所示"——废话
- 同一变量出现多次（已经做了交互项还把主项放回来）
- 用 stargazer / esttab 默认输出但不删冗余信息

## 图形规范

### 必备结构

- 字体：宋体 / Times New Roman 9–11 pt
- 颜色：黑白 + 1–2 种主色调（推荐深蓝 / 暗红），**不要彩虹色**
- 分辨率：≥ 300 dpi
- 图号 + 图名在**图下方**（与表相反）
- 注释：必须有数据来源 + 关键变量定义

### 常见图类

1. **平行趋势事件研究图** —— 95% CI，垂直虚线标处理时点
2. **政策时间线图** —— 与制度背景章节配套
3. **机制路径图** —— 用矩形 + 箭头，避免复杂阴影
4. **异质性森林图** —— 各子样本点估计 + CI

## 执行桥（StatsPAI / Stata MCP）

表格图形**从拟合结果生成**，不要手抄数字。完整映射见
[`execution-with-mcp`](../../../shared-resources/empirical-methods/execution-with-mcp.md)。《管理世界》重中国情境实证 + 政策可操作；识别 + 经济量级，定性/案例另循其标准。

- **表：**`etable`（多列）或 `did_summary_to_latex` 直接从 `result_id` 生成。
- **图：**`plot_from_result` / `enhanced_event_study_plot` / `event_study_table`，坐标单位与
  标准误/聚类注记自带。
- **每个表注**写明估计量与聚类层次，并以可解释单位报告经济量级。

完整“拟合结果 → 图表”链见 [JF 执行 walkthrough](../../../Journal-of-Finance-Skills/resources/worked-examples/02-execution-walkthrough.md)。
## 必查清单

- [ ] 主表 ≤ 6 列
- [ ] 三线表、无竖线
- [ ] 标准误格式统一
- [ ] 注释包含聚类层次
- [ ] 图形 300 dpi 以上
- [ ] 图表编号连续

## 反模式

- 表格里塞 R² + 调整 R² + 伪 R² + AIC + BIC + …… —— 选 1–2 个就够
- 图例用一长串说明
- 一张图叠 5 条曲线 + 5 种线型 + 5 种颜色
- 直接截图 Stata / R 输出贴进 Word

## 本刊表图审稿期待表

《管理世界》由国务院发展研究中心主办，是规范实证大刊，表图是审稿专家判断"研究是否规范"的最直观信号。编委对三线表、注释完整性、列数克制的要求接近硬约束。下表锚定本刊审稿期待与常见退稿模式：

| 维度 | 本刊期待 | 退稿/吐槽高风险信号 |
|---|---|---|
| 表格结构 | 三线表、无竖线、≤6 列 | 带边框/阴影/8 列以上 |
| 标准误 | 系数下方括号内、聚类标准误 | 用 t/z 值、未说明聚类 |
| 注释 | 含聚类层次 + 显著性符号定义 | 注释缺失或写"如表 3 所示"废话 |
| 图形 | ≥300dpi、≤2 主色、图名在下 | 彩虹配色、截图 Stata 输出 |

### 常见退稿模式（表图维度）

- 主表 8 列以上信息密度过高，审稿人要求拆表
- 系数保留小数点后 4—6 位，超出本刊惯例（保留 3 位）
- 表格塞满 R²、调整 R²、伪 R²、AIC、BIC，喧宾夺主
- 直接截图 Stata/R 输出贴进 Word，被判为不规范

## 微型走查示例：数字化转型主表与事件研究图

承接数字化转型稿件，按规范走查（数字为示意）：

- **主表（表 3）**：3 列（基准/加控制变量/加固定效应），核心系数"数字化×处理后"0.045***、0.038***、0.040***（示意），括号内为聚类于企业层面的标准误。
- **列数检查**：3 列 ≤6 列，合规；若再加机制列将拆为表 4，避免超列。
- **小数位**：系数统一保留 3 位；R² 仅报 R²，不堆 AIC/BIC。
- **注释**：写明"括号内为聚类于企业层面的稳健标准误；***、**、* 分别表示 1%、5%、10% 显著水平"——含聚类层次与符号定义，达标。
- **平行趋势事件研究图**：-5 至 -1 期系数 95% CI 跨零（示意），处理时点用垂直虚线标注，深蓝单色，300dpi，图名在图下方。
- **判定**：三线表、列数、注释、图形规范齐备 → 进入 mw-policy-implication。

## 审稿人追问模式 + 本刊语境修法

- 追问"标准误聚类在哪一层？" → 修法：在每张回归表注释中显式写明聚类层次，与 mw-identification 的聚类决策保持一致。
- 追问"表格信息密度太高？" → 修法：把控制变量系数折叠为"是/否"行，机制与异质性结果拆入独立表，主表控制在 6 列内。
- 追问"图形不像本刊风格？" → 本刊语境下，编委偏好黑白加 1—2 种主色（深蓝/暗红）、无复杂阴影；机制路径图用矩形加箭头即可。

## 校准锚点

本刊已刊实证论文主表普遍为三线表、≤6 列、聚类标准误置于括号内、注释含聚类层次与显著性定义；图形以单色或双主色、300dpi 以上、图名在下为常态。具体字号、配色与表格置入方式以编辑部最新稿约及同期已刊论文形态为准。
