# 修改回复（mw-rebuttal）

## 触发时机

- 收到《管理世界》R&R 通知
- 已有 1–3 份审稿意见和编委意见
- 准备开始修订正文 + 回复信

## 顺序原则

**先修订正文，再写回复信**。回复信是"导览图"，必须基于已修订的正文版本。

颠倒会导致：
- 回复信承诺了正文未真正做的修订
- 正文修订过程中发现承诺无法兑现
- 多稿来回不一致

## 回复信结构

### 总体结构

```
尊敬的主编、编委、各位审稿专家：

非常感谢编辑部和各位审稿专家在百忙之中对本文的审阅和宝贵意见。
我们已根据各位意见对论文进行了系统修订。下面我们逐条回复各位的意见。
为便于核对，我们在回复中标注了修订对应的正文位置（如"详见正文 P.X 第 Y 段"）。

致主编 / 编委的回复（如有编委意见）
审稿人 1 的回复
审稿人 2 的回复
审稿人 3 的回复（如有）

再次感谢各位的宝贵意见。
作者
20XX 年 XX 月 XX 日
```

### 单条回复结构

```
意见 X：[原文照抄审稿意见]

回复：感谢审稿专家的宝贵意见。我们已[做了什么具体修改]，理由如下：
（1）从理论角度，……
（2）从实证角度，……
（3）从写作角度，……
具体修改详见正文第 X 页第 Y 段（修订标记已用[蓝色字体 / 修订模式]标出）。
```

## 三种回复模式

### A. 完全接受

- 直接说明接受 + 描述修改 + 指明位置
- 不要找理由

### B. 部分接受

- 先肯定审稿人的合理性
- 说明部分接受的范围
- 解释未接受部分的原因（**给证据，不要狡辩**）

### C. 不接受（谨慎使用）

- 先充分肯定审稿人的考虑
- 给出严密的反驳论证（至少 3 段）
- 必要时新增稳健性 / 补充分析作为旁证
- **不接受的比例最多 1–2 条**，否则编委不会买账

## 必查清单

- [ ] 每条审稿意见都有对应回复（不能遗漏）
- [ ] 每条回复都指明正文修订位置
- [ ] 回复信开头有致谢段
- [ ] 不接受的意见有充分论证 + 旁证
- [ ] 正文修订处有标记（蓝色字体 / 修订模式）
- [ ] 修订版正文已通读，确保无前后矛盾

## 反模式

- "感谢审稿人的意见，我们已修改"——一行回复，等于挑衅
- 回复信比正文还长 —— 多半在凑字数
- 三个审稿人意见冲突，回复信里全部接受 —— 正文会自相矛盾
- 把不接受的回复写得很 defensive

## 本刊 R&R 审稿期待表

《管理世界》由国务院发展研究中心主办，审稿强调规范的因果识别、机制检验与政策含义。R&R 阶段编委关注的不是"作者是否态度恭敬"，而是"识别、机制、政策三条线是否真正补强、修订是否在正文落地、是否服务国家战略导向"。下表锚定本刊审稿期待与常见退稿模式：

| 意见类型 | 本刊期待的回应 | 致命应对 |
|---|---|---|
| 识别质疑 | 补异质性处理偏误诊断/安慰剂，正文落地 | 仅口头辩护不补分析 |
| 机制质疑 | 新增因果中介或子样本对比 | 把原结果换个说法 |
| 政策含义薄 | 重写分层建议，对接国家战略 | 加一句"建议政府重视" |
| 中外文献失衡 | 补本土顶刊文献并做对话 | 仅在回复信致谢不动正文 |

### 常见退稿模式（回复阶段）

- 回复信承诺的修订正文里找不到，二审被编委当场点破
- 三位审稿人意见冲突却全部"接受"，导致正文自相矛盾
- 不接受的意见超过两条且论证单薄，被判为"不愿改"
- 一行式"已修改"回复，等同挑衅审稿专家

## 微型走查示例：数字化转型稿件的二轮回复

虚构一份 R&R，按本 skill 规则走查（页码、系数为示意）：

- **意见 1（识别）**：审稿人质疑 staggered DID 的异质性处理偏误。回复模式 A（完全接受）：补 Goodman-Bacon 分解（坏比较权重 11%，示意）+ Callaway-Sant'Anna 估计（0.032，示意），详见正文 P.14 第 2 段。
- **意见 2（机制）**：审稿人问"为何数字化能提升 TFP"。模式 A：新增"政策→数字化程度→TFP"因果中介，详见正文 P.18 第 3 段。
- **意见 3（政策）**：编委指出政策含义偏弱。模式 B（部分接受）：重写三条分层建议并对接"数字中国"战略，但保留原结论框架，理由分三段说明，详见正文 P.22。
- **意见 4（异质性切法）**：审稿人建议加按要素禀赋切分。模式 C（不接受，谨慎）：先肯定其合理性，再以样本量不足（某子样本 N=42，示意）+ 已有三维度已足够支撑结论为由婉拒，并补一项稳健性作旁证。
- **判定**：不接受仅 1 条且有旁证，正文修订与回复信一一对应 → 可上传二审。

## 审稿人追问模式 + 本刊语境修法

- 追问"修订是否真的进了正文？" → 修法：每条回复标注"详见正文 P.X 第 Y 段"，并用修订模式/蓝色字体标出，便于编辑部核对。
- 追问"三位审稿意见冲突如何调和？" → 本刊语境下，编委接受"显式说明冲突 + 给出折中方案 + 解释取舍依据"，远胜于全盘接受导致正文矛盾。
- 追问"政策含义为何仍偏弱？" → 修法：联动 mw-policy-implication 重写分层建议并对接国家战略，而非在回复信里补一句空话。

## 校准锚点

本刊 R&R 的成功回复普遍是"先改正文、再写导览式回复信"，逐条标注正文位置、识别/机制/政策三线均有实质补强、不接受意见控制在 1—2 条且附旁证。回复信长度通常远短于正文修订量。二审轮次与上传格式以编辑部最新稿约为准。
