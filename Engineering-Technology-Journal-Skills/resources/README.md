# Engineering and Technology Breadth Resources

This layer supports routing across engineering, materials, energy, robotics,
communications, controls, mechanics, optics, and biomedical-engineering venues.
Use it when a manuscript could fit several technical communities with different
evidence standards.

## What to Load

- `official-source-map.md` - official-source anchors for current author
  guidelines and scope checks.
- `source-basis.md` - coverage basis, ranking logic, and verification rules.
- `worked-examples/venue-routing.md` - example routing diagnoses for hardware,
  theory, simulation, systems, materials, energy, and biomedical-engineering
  submissions.
- `exemplars/selection-patterns.md` - technical-community selection patterns
  and sibling-journal traps.

## Usage Rule

Start with `en-engtech-journal-workflow` when the manuscript's primary
community is unclear. Route by evidence artifact first: theorem/proof,
prototype, benchmark, device, system deployment, materials characterization, or
clinical-engineering validation. Then load only the most relevant venue cards.
