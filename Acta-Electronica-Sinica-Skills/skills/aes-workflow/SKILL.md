# 《电子学报》投稿全流程编排

本技能是《电子学报》(Acta Electronica Sinica, AES) 深度包的"总控台"，把 12 个技能按时间线串起来。
《电子学报》由中国电子学会 (Chinese Institute of Electronics) 主办、为月刊、走双盲多级审理，
全流程可能持续数月甚至更久，谋定而后动比边写边改更高效。以下时间线综合官网 (ejournal.org.cn)
渲染与中文学报通行节奏；未公开确认的时长标 **待核实**（核验日 2026-07-09）。

> 用法：在每个阶段，先看本页定位，再跳到对应 aes-* 技能执行细则。Acta / Electronica / Sinica
> 的每一步都有对口技能。

## 一、端到端时间线

```
① 选题定位   → aes-topic-selection    （动笔前）
② 文献综述   → aes-related-work        （动笔前/中）
③ 实验设计   → aes-experiments         （研究期）
④ 可复现建设 → aes-reproducibility     （研究期）
⑤ 写作规范   → aes-writing-style       （写作期）
⑥ 附录材料   → aes-supplementary       （写作期）
⑦ 制品开放   → aes-artifact-evaluation （投稿前）
⑧ 投稿审计   → aes-submission          （投稿前）
⑨ 审理跟踪   → aes-review-process      （投稿后）
⑩ 修回答复   → aes-author-response     （退修后）
⑪ 录用校样   → aes-camera-ready        （录用后）
```

## 二、各阶段目标与产出

| 阶段 | 目标 | 关键产出 |
|---|---|---|
| 选题 | 确认对口本刊学科与栏目 | 一句话创新点、栏目归属 |
| 综述 | 定位贡献、找准竞争工作 | 相关工作章节 + 贡献清单 |
| 实验 | 公平严谨验证每条贡献 | 主/对比/消融/敏感性结果 |
| 可复现 | 让结果可被重复 | 环境/种子/超参数记录 |
| 写作 | 合本刊体例 | 中英文摘要、GB/T 7714 文献 |
| 投稿 | 越过初审形式门槛 | 双盲匿名稿 + 评审费 |
| 审理 | 正确解读状态 | 状态研判、催稿时机 |
| 修回 | 逐条服人 | 修改说明 + 改痕稿 |
| 校样 | 零差错出版 | 定稿、揭盲回填、清样 |

## 三、关键决策点

- **投前分流**：面向中文旗舰学报影响力投《电子学报》；面向 SCI 国际读者考虑英文姊妹刊
  Chinese Journal of Electronics (CJE)。同稿不可同时投两刊（属一稿多投）。
- **对口判断**：主题若更贴计算机/控制学科，改投兄弟刊（《计算机学报》《自动化学报》等）更明智。
- **退修 vs 退稿**：退修是实质进展，全力修回；直接退稿则评估改投。

## 四、时间陷阱与规避

| 陷阱 | 规避 |
|---|---|
| 临投稿才建可复现/制品 | 研究期就记录环境、种子、超参数 |
| 双盲匿名化留到最后 | 写作期即用第三人称自引、占位基金 |
| 忽视 6 个月答复红线 | 记录投稿日，超期礼貌询问，勿一稿多投 |
| 修回逾期沉默 | 按退修信截止日排期，需要时提前申请延期 |
| 校样阶段大改 | 实质修改在修回期完成，校样只订错 |

## 七、阶段并行与前置

有些工作不必严格串行，合理并行能缩短总周期：

| 可前置/并行的工作 | 何时做 | 收益 |
|---|---|---|
| 可复现记录（环境/种子/超参数） | 研究期实时记录 | 投稿前不用回忆补齐 |
| 双盲匿名化（第三人称自引） | 写作期即执行 | 投稿前无需大改 |
| 制品打包与 README | 实验完成即整理 | 投稿即可附匿名链接 |
| 参考文献 GB/T 7714 整理 | 边写边录 | 定稿不临时爆量 |

前置这些"技术债"，能让投稿与修回阶段把精力集中在学术质量上。

## 八、投稿档案管理

- 为每次投稿建一个档案目录：稿件各版本、修改说明、审稿意见、往来邮件、评审费凭证。
- 记录关键日期：投稿日、各轮退修日与截止日、录用日，用于对照 6 个月红线与结题材料。
- 保留改痕稿与干净稿的对应关系，便于多轮再审时快速回溯（配合 `aes-author-response`）。

一份清晰的投稿档案，既是应对多轮审理的作战地图，也是成果登记与基金结题时的现成材料。
