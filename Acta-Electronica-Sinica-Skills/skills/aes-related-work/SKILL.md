# 《电子学报》相关工作与文献综述

相关工作章节是《电子学报》(Acta Electronica Sinica, AES) 外审专家判断"这篇文章是否清楚自己站在哪里"
的窗口。本刊由中国电子学会 (Chinese Institute of Electronics) 主办，审稿人对领域文献熟悉，**漏引关键
工作、把综述写成流水账、或说不清与已有方法的区别**，都会直接导致"创新性存疑"的质疑。本技能帮你
把相关工作写成一张有脉络的地图，并把本文的贡献钉在图上。以下为通行学术规范；本刊综述性稿件的
具体体例标 **待核实**（核验日 2026-07-09，官网 ejournal.org.cn）。

> 目标：读完你的相关工作，审稿人应能一句话复述"已有方法的局限"和"你补上的那块拼图"。
> 这是 Acta / Electronica / Sinica 稿件建立创新性的地基。

## 一、界定范围与检索

- 先划定综述边界：本文属于哪个子问题（如信道估计、目标检测、低功耗电路），不贪大求全。
- 系统检索中外文献：CNKI、万方、维普（中文）；IEEE Xplore、dblp、Scopus（英文）。
- 覆盖三类来源：**奠基性工作**、**最新代表性工作**、**与本文最接近的竞争工作**。
- 特别检索《电子学报》与英文姊妹刊 Chinese Journal of Electronics 上的相关论文，体现对本刊学术
  脉络的了解，也便于审稿人衡量本文在本刊语境中的位置。

## 二、按脉络组织，而非按时间罗列

| 写法 | 评价 |
|---|---|
| "A 做了…，B 做了…，C 做了…" 流水账 | 差：看不出脉络与批判 |
| 按**技术路线/方法族**分组，指出各族优劣 | 好：显示视野与判断 |
| 每组末尾点出**共同局限**，引向本文动机 | 最佳：自然导出创新点 |

组织维度可选：按方法族（传统/学习类）、按问题设定、按性能瓶颈、按应用场景。

## 三、批判性而非罗列性

- 每引一组工作，说清它**解决了什么、代价是什么、留下什么未解**。
- 用证据说话（其报告的指标、假设、适用条件），避免主观贬低他人工作。
- 把"已有局限"与"本文贡献"成对呈现，让创新点从综述里长出来，而不是硬塞。

## 四、定位本文贡献

- 在相关工作末尾或引言中，用一段明确写出：与最接近的 N 个工作相比，本文在**问题/方法/性能**上
  的具体区别。
- 若与本刊/CJE 已发表工作相近，尤其要讲清增量，避免被判自我重复或创新不足。
- 贡献表述可条目化（贡献1/2/3），与实验章节的验证一一呼应。

## 五、引用规范（GB/T 7714）

- 正文按出现顺序方括号角标引用 [x]；文末列表同序（见 `aes-writing-style`）。
- 引用要准确：不曲解他人结论，不转引未读文献，不堆砌无关引用充数。
- 中外文献并重，反映领域真实版图；避免只引中文或只引英文的偏食。

## 六、双盲期的自引匿名化

- 自引改第三人称："文献[x]提出…"而非"我们之前提出…"，避免暴露身份。
- 不写"我们课题组""本单位此前工作"等泄露线索。
- 录用后可恢复正常自引表述（见 `aes-camera-ready`）。

## 七、常见失误与规避

| 失误 | 规避 |
|---|---|
| 漏引最接近的竞争工作 | 用检索关键词矩阵反复扫，专门查本刊/CJE |
| 综述写成引文清单 | 按脉络分组 + 每组批判 |
| 创新点与综述脱节 | 局限—贡献成对呈现 |
| 自引暴露身份 | 双盲期第三人称化 |
| 只引英文/只引中文 | 中外并重 |

## 八、相关工作自检清单

- [ ] 领域边界清晰，检索覆盖奠基+最新+最接近工作。
- [ ] 已检索并恰当引用《电子学报》/CJE 的相关论文。
- [ ] 按技术脉络分组，每组有批判与局限。
- [ ] 本文贡献与已有工作差异明确、可与实验呼应。
- [ ] 引用准确、中外并重、无堆砌。
- [ ] 双盲期自引已匿名化。
- [ ] 参考文献 GB/T 7714 合规。

## 九、检索关键词矩阵法

为避免漏引最接近的竞争工作，建议用**关键词矩阵**系统检索：把"问题维度"与"方法维度"的同义词
分别列出，两两交叉组合成检索式，覆盖中外数据库。例如问题维度含"信道估计/channel estimation"，
方法维度含"深度学习/deep learning/神经网络"，交叉后逐一在 CNKI、IEEE Xplore、dblp 上检索。

| 维度 | 中文同义词 | 英文同义词 |
|---|---|---|
| 问题 | 信道估计、目标检测… | channel estimation, detection… |
| 方法 | 深度学习、稀疏表示… | deep learning, sparse coding… |
| 场景 | 大规模 MIMO、低信噪比… | massive MIMO, low-SNR… |

矩阵法能把"我以为查全了"变成"可核查地查全了"，显著降低漏引风险。

## 十、引用的质与量平衡

- 引用数量服从领域惯例，但**质重于量**：一篇精准的对比引用胜过十篇泛泛引用。
- 优先引用同行公认的高质量来源与一手文献；慎引预印本未定稿结论。
- 适度引用《电子学报》与 CJE 近年相关工作，既显示对本刊脉络的熟悉，也利于审稿人定位本文位置。

## 十一、相关工作写作小结

- 开篇一句话交代本节的组织维度（按方法族/问题设定/性能瓶颈），让读者有地图感。
- 每组工作末尾点出共同局限，自然引向本文动机，避免局限与贡献两张皮。
- 结尾用一段把本文贡献钉在这张地图上，说清与最接近工作的差异。
- 全节引用准确、中外并重、含本刊/CJE 相关文献，无堆砌无曲解。

```

范围界定：<清晰 / 过宽>

最接近工作：<已识别并对比 N 篇 / 疑漏引>
贡献定位：<与已有差异明确 / 模糊>
自引匿名：<通过 / 需处理>

```
