# 复现包与自查脚本思路（Computer Science /《计算机科学》, JSJKX）

本目录说明面向《计算机科学》(Computer Science) 投稿的**复现包结构**与**作者侧自查**思路。本刊未见公开的
独立 artifact 徽章评审制度（**待核实**，见 `../official-source-map.md`），因此复现包主要服务于：让单盲外审
专家更快确认你的方法与实验可信、可重复，并为录用后可能的补充材料做准备。

## 一、建议的复现包结构

```
repro/
├── README.md            # 环境、依赖、一键复现步骤、数据获取方式
├── env/                 # requirements.txt / environment.yml / Dockerfile
├── data/                # 数据说明与获取脚本（大数据用外链+校验和，勿入库）
├── src/                 # 方法与基线实现
├── scripts/             # run_all.sh：从原始数据到论文主表/主图
├── configs/             # 超参与随机种子（固定）
└── results/             # 预期输出（表/图）与日志
```

## 二、与论文对齐

- 论文每一张主表/主图，`scripts/` 里应有可定位的生成命令。
- 固定随机种子、库版本、硬件说明；报告均值±方差与运行次数。
- 数据划分（训练/验证/测试）与预处理脚本随包提供，杜绝数据泄漏。

## 三、作者侧自查（投稿前）思路

以下为**思路清单**（可用任意脚本语言实现，非官方要求）：

1. **格式自查**：检查中英文题名/摘要/关键词是否齐全；中图分类号(TP…) 是否存在；参考文献是否 GB/T
   7714—2015 顺序编码制（正文上标、文后按序编号）。
2. **图表自查**：统计图/表数量，核对每图每表是否有中英文题注与序号。
3. **查重自查**：投稿前用主流工具自查总文字复制比（阈值以编辑部为准，**待核实**）。
4. **匿名/署名**：本刊为**单盲**（审稿人知作者），故正文可保留署名，但仍需完整**利益冲突声明**与基金项目号。
5. **一键复现**：在干净环境执行 `scripts/run_all.sh`，确认能从数据复现主结果。

## 四、提交与外链

- 大体量数据/模型用可长期访问的仓库（如机构库、Zenodo）外链并给校验和；正文注明获取方式与访问日期。
- 若涉敏感数据，提供合规的脱敏子集或申请流程说明。
- 录用后如需提供补充材料/多媒体附件，参见 [`csj-supplementary`](../../skills/csj-supplementary/SKILL.md)。
