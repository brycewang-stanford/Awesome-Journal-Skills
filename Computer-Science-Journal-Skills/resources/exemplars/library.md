# 范文自选与核验指南（Computer Science /《计算机科学》, JSJKX）

本文件**不列举具体篇目**，也不杜撰任何论文的标题、作者或指标，而是教你如何在《计算机科学》(Computer
Science) 官网自行挑选、核验适合模仿的范文。原则：以官网 `www.jsjkx.com` 当期/过刊真实刊文为唯一依据。

## 一、为什么要自选范文

《计算机科学》(Computer Science) 覆盖计算机全学科、栏目众多、含大量专题(专栏)与综述，不同栏目的写法差异
明显（如"综述"与"人工智能"研究长文体例不同）。与其套用泛化模板，不如在**你要投的那个栏目/专题**里找 2~3
篇近 1~2 年的真实刊文作参照。

## 二、挑选步骤

1. 打开官网"当期目录/过刊浏览"（folder1.shtml），定位与你选题最接近的**栏目**或**专题专栏**。
2. 在该栏目下选取 2~3 篇**近 12~24 个月**的文章（越新越贴近现行体例）。
3. 优先选**被引/下载较高**、结构完整的文章作为结构参照；综述稿另找综述范文。

## 三、核验清单（对每篇候选范文逐项记录）

| 维度 | 观察点 |
|---|---|
| 栏目归属 | 属于哪个常规栏目或专题？与你的选题是否同栏 |
| 摘要体例 | 中文摘要是否报道性、含目的/方法/结果/结论？英文 Abstract 结构 |
| 关键词 | 数量、颗粒度、中英文是否一一对应 |
| 章节骨架 | 引言→相关工作→方法→实验→结论的组织与篇幅配比 |
| 图表 | 图/表数量、题注中英文、公式排版 |
| 参考文献 | 是否 GB/T 7714—2015 顺序编码制、条数量级、中英文对照 |
| 篇幅 | 正文字数量级（对照"不少于 8000 字"的建议） |

## 四、使用范文的纪律

- 只**学结构与体例**，不抄内容、不套语句。
- 不要把范文的指标（影响因子、被引数）当作本刊确定属性写进你的稿件或本包正文。
- 若无法在官网确认某篇文章的栏目或年份，则不作为范文使用，也不在任何材料中断言其属性（标 **待核实**）。

## 五、把范文观察转成自查项

将上表逐篇填好后，抽取共性，形成你自己的"投稿前体例对照表"，交由 [`csj-writing-style`](../../skills/csj-writing-style/SKILL.md)
与 [`csj-submission`](../../skills/csj-submission/SKILL.md) 做最终审计。
