# 《计算机科学》审稿意见答复与修回说明

本技能帮你为《计算机科学》(Computer Science, 简称 JSJKX) 准备**审稿意见答复(response letter)** 与**修回说明**。
本刊单盲审稿、期刊式**多轮修回**（小修/大修），一份逐条、有据、可定位的答复信往往是决定下一轮通过的关键。
把修回（尤其大修）当作"第二次投稿"认真对待。

> 提醒：本刊 Computer Science 是**期刊(journal)**，非会议；这里是期刊修回信，不是会议 rebuttal。

## 一、总体原则

- **逐条回应**：每条意见单独回应，不合并、不遗漏。
- **可定位**：说明改在何处（章节/页/图表/公式号），并在修改稿中用颜色或标注标出。
- **有据**：能补的补实验/数据；不能改的礼貌说明理由并给证据。
- **尊重**：先感谢，再回应；对分歧就事论事，不情绪化。

## 二、答复信结构（中文模板）

```
尊敬的编辑、各位审稿专家：
  感谢对本文的审阅与宝贵意见。我们已按意见逐条修改，主要改动如下：
  <改动概述 3~5 点>
  以下逐条回应（【意见】原文 / 【回应】说明 / 【修改】位置）：

—— 审稿专家 1 ——
【意见1】<原文意见>
【回应】<我们的回应>
【修改】见第 X 节 / 图 Y / 表 Z（修改稿第 P 页，已标黄）

【意见2】…

—— 审稿专家 2 ——
…

—— 编辑意见 ——
…

  再次感谢各位专家使本文质量得到提升。
  作者敬上，2026-XX-XX
```

## 三、小修 vs 大修策略

| 类型 | 特点 | 策略 |
|---|---|---|
| 小修 | 局部澄清/补充，通常不重审 | 快速精确修改，答复简明，逐条对应 |
| 大修 | 需补实验/重写，常重新外审 | 优先补最能消除退稿风险的证据，答复详尽，改动显著且可定位 |

## 四、如何回应不同类型意见

- **可直接采纳**：照做并指出改处；一句"已按建议修改，见…"。
- **需补实验**：给出新实验设计与结果表/图（与 `csj-experiments` 一致），说明结论。
- **误解**：礼貌澄清是哪里表述不清，并**改进正文表述**（往往说明你的写作有歧义）。
- **合理分歧**：给出保留理由与证据，承认局限并写入"局限与未来工作"。
- **超范围要求**：说明与本文范围/篇幅的关系，必要时移入附录或未来工作。

## 五、补充实验的呈现

- 新增实验在答复信里给关键结果，在修改稿正文/附录完整呈现，二者数值一致。
- 明确标注哪些是应审稿意见新增，方便专家复核。

## 六、常见失误

| 失误 | 后果 | 纠正 |
|---|---|---|
| 只说"已修改"不给位置 | 专家难复核，易再退修 | 每条给章节/页/图表号 |
| 漏应某条意见 | 复审印象差 | 逐条对照，建清单核销 |
| 对分歧硬顶或情绪化 | 关系恶化 | 就事论事、给证据、认局限 |
| 答复与修改稿数值不一致 | 可信度受损 | 交叉核对一致性 |

## 七、自查清单

1. 是否**每一条**意见都单独回应、无遗漏？
2. 每条是否给出修改位置且在稿中标注？
3. 补充实验是否与正文/附录一致、标明新增？
4. 分歧处是否礼貌、有据、认局限？
5. 小修/大修策略是否对路（大修是否补了关键证据）？
6. 答复信结构是否清晰、感谢得体？

## 八、输出格式

```
【CSJ 修回答复审计】
意见总数：<n> 已逐条回应：<n/n>
定位标注：每条给章节/页/图表 <✓/✗>
补充实验：与正文一致+标明新增 <✓/✗>
分歧处理：礼貌+有据+认局限 <✓/✗>
类型策略：<小修/大修> 对路 <✓/✗>
结论：<可提交修回 / 需补：列点>
```

## 九、修改稿的标注与版本管理

《计算机科学》(Computer Science) 的修回通常需同时提交修改稿与答复信。建议在修改稿中用颜色或下划线标出改动，
或另附"修改对照说明"，让复审专家一眼看到改了哪里。答复信中每条回应给出精确定位(第 X 节、图 Y、表 Z、第 P 页)，
与修改稿颜色标注呼应。保留每一轮的稿件版本与答复信，避免多轮修回后自己也搞混改了什么。

## 十、跨审稿人矛盾意见的处理

多位外审可能给出相互矛盾的意见(如一人要求精简、另一人要求补充)。处理办法：不偏袒任何一方，而是给出你的
权衡与依据，在正文用折中方案(如核心留正文、细节入附录)同时回应两方；在答复信中分别向两位说明你如何兼顾。
若确实无法两全，礼貌说明取舍理由并请编辑裁量。切忌对一方妥协却对另一方沉默。

## 十一、语气与时限

答复信全程保持感谢、专业、克制的语气，即便面对严厉甚至偏颇的意见，也就事论事、以证据回应，不辩解、不
情绪化——单盲下审稿人知道你是谁，专业的态度本身也是形象。务必在编辑部规定的修回期限内提交；确需延期
应提前通过系统或邮件说明并申请。逾期不回易被按退稿处理。

## 十二、修回提交前的最终清单

提交修回前，用这份清单做最后核对，确保为《计算机科学》(Computer Science) 的复审专家提供一份可快速复核的
材料包：

1. 答复信是否覆盖**每一位**审稿人的**每一条**意见，无一遗漏？
2. 每条回应是否给出修改的精确位置(节/页/图/表/公式号)？
3. 修改稿是否已用颜色或标注标出全部改动？
4. 新增实验的数值是否在答复信、正文、附录、复现包四处一致？
5. 对未采纳的意见是否礼貌说明理由并给出证据？
6. 矛盾意见是否给出兼顾方案或请编辑裁量？
7. 是否在编辑部规定期限内提交、格式符合系统要求？

全部打勾后再提交；任一项存疑就回到前面对应小节补齐。高质量的修回材料，是大修翻盘、小修顺利通过的决定
因素。
