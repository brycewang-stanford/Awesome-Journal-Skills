# 《计算机科学》相关工作与文献综述

本技能覆盖两种场景：研究长文的**相关工作**章节，以及本刊鼓励的**综述**稿。两者在《计算机科学》(Computer
Science, 简称 JSJKX) 都很关键——本刊覆盖面广、综述活跃，一篇组织良好的综述本身就是受欢迎的稿件类型，而
研究长文的相关工作则决定单盲外审专家对你**创新性(delta)** 的第一判断。

> 提醒：本刊 Computer Science 是**期刊(journal)**，非会议；引用体例用 GB/T 7714—2015 顺序编码制。

## 一、相关工作（研究长文）：delta 优先

外审专家读相关工作，本质在问：**你比已有工作多做了什么？** 因此不要按时间流水账罗列，而要：

1. **按问题维度分组**：把已有工作归入 2~4 个技术路线/流派，每组一段。
2. **每组落到"局限"**：这组方法在你关注的问题上**缺什么**（假设过强、场景受限、代价过高…）。
3. **收束到你的缺口**：由这些局限自然引出本文补的空白与贡献。
4. **公平引用**：代表性工作、最新工作、以及你将作为**基线**比较的工作都要出现且准确。

一句话模板：`现有 X 类方法通过<机制>解决<问题>，但在<你的场景>下<局限>；本文<delta>。`

## 二、综述稿：本刊的优势路径

若你写的是**综述**（本刊鼓励），需要的不是罗列而是**框架**：

- **明确范围与检索方法**：界定综述覆盖的问题域、时间跨度、纳入/排除标准（可提检索式与来源库），体现系统性。
- **提出分类框架(taxonomy)**：用一张分类图/表把该方向的方法谱系组织起来，这是综述的骨架与主要贡献。
- **批判性比较**：不只描述，还要横向比较各类方法的假设、适用条件、优缺点、代表数据集与指标。
- **未解问题与趋势**：指出公开挑战与未来方向，给后来者路标——这是综述价值的高地。
- **篇幅与文献量**：综述通常长文，参考文献量级较大，覆盖需全面且新。

## 三、引用规范（GB/T 7714—2015）

- 正文上标数字按引用先后编号，文后按序著录；期刊 `[J]`、会议 `[C]`、专著 `[M]` 等类型标识齐全。
- 中文文献建议中英文对照；作者、题名、刊名、年卷期页完整。
- 避免只引英文而漏引本领域重要中文工作，也避免过度自引。

## 四、常见问题与修正

| 问题 | 修正 |
|---|---|
| 相关工作变成"编年史" | 改为按技术路线分组，每组落到局限 |
| 只夸别人不指缺口 | 每类方法明确"在本文场景下缺什么" |
| delta 不清、和基线区别模糊 | 用一句话模板显式写出增量 |
| 综述只有罗列无分类框架 | 补 taxonomy 图/表与批判性比较 |
| 漏引将作基线的工作 | 基线工作必须在相关工作出现且准确 |
| 引用格式非 GB/T 7714 | 统一顺序编码制与类型标识 |

## 五、与实验章节的联动

相关工作里点名的**基线**，必须与 `csj-experiments` 的对比实验一一对应：凡在相关工作说"优于/不同于"的方法，
实验里应有公平比较或明确说明为何不可比。避免"相关工作提一套、实验比另一套"的脱节。

## 六、自查清单

1. 相关工作是否按问题维度分组、每组落到局限？
2. 是否有一句话 delta，和基线区别清晰？
3. 若为综述：是否有范围界定、分类框架、批判性比较、趋势展望？
4. 代表性/最新/基线工作是否都引用且准确？中文重要工作是否漏引？
5. 引用是否 GB/T 7714—2015 顺序编码制、类型标识齐全？
6. 相关工作点名的基线是否与实验对应？

## 七、输出格式

```

类型：<研究长文相关工作 / 综述稿>
分组：<路线1/路线2/…>，每组局限是否明确 <✓/✗>
delta 一句话：<……>
综述专项：范围[✓/✗] 分类框架[✓/✗] 批判比较[✓/✗] 趋势[✓/✗]
引用：GB/T 7714 <✓/✗>；基线是否与实验对应 <✓/✗>

```

## 八、面向本刊读者的相关工作写法

《计算机科学》(Computer Science) 读者横跨计算机各方向，相关工作不宜假设读者都是本细分领域专家。因此在
点明技术路线时，用一两句给出必要背景，让跨方向读者也能跟上，再收敛到你的缺口。这与只面向小圈子的专门刊
不同——适度的"可读性铺垫"在本刊是加分项，但不能喧宾夺主，背景铺垫后必须迅速回到 delta。

## 九、综述型相关工作的分类框架示例

综述稿的核心是分类框架。一个可操作的做法：先选一个**划分维度**(如按问题设定、按方法范式、按应用场景)，
在该维度下把方法分成互斥且穷尽的若干类，用一张表并列各类的代表工作、核心思想、假设、优缺点与适用条件。
若单一维度不足，可用两级分类(先按范式、再按机制)。分类框架要能让读者"按图索骥"，并暴露出尚无人涉足的
空格——那正是未来方向。避免按年份或按作者罗列，那不是分类。

## 十、文献时效与中英文均衡

本刊重视时效，相关工作/综述应覆盖近 1~2 年的最新工作，避免只引陈旧文献显得脱节。同时兼顾中英文文献：
既引国际代表工作，也不遗漏本领域重要的中文成果(尤其是本刊及兄弟刊近年的相关文章)，体现对本土研究脉络的
了解。引用格式统一 GB/T 7714—2015 顺序编码制，中文文献按要求给中英文对照。

## 十一、相关工作与方法章节的分工

相关工作与方法章节常被写重复。清晰分工是：相关工作**对外**——梳理他人做了什么、局限在哪、你的缺口；方法
章节**对内**——完整讲你自己怎么做。相关工作里只需点到别人方法的核心思想与局限，不展开实现细节；把与你
方法最相关的技术背景(如你在其上改进的某个模型)可在方法章节开头简述。为《计算机科学》(Computer Science)
写作时，避免在相关工作里过度技术展开，也避免方法章节重新罗列文献，二者各司其职、相互呼应即可。

## 十二、综述稿的更新维护意识

一篇好的综述在见刊后仍有生命力。写作时就应考虑：分类框架是否具有一定的前瞻性与可扩展性，能容纳后续新
方法？公开问题与趋势的判断是否经得起时间检验？这些决定综述的长期被引价值。为本刊撰写综述时，力求框架
清晰、判断有据，使其成为该方向读者(横跨计算机各领域)的稳定参照，而非很快过时的文献清单。

## 十三、相关工作写作步骤清单

为《计算机科学》(Computer Science) 撰写相关工作时，按步骤推进：

1. 检索近 1~2 年最新工作与领域经典工作，兼顾中英文文献。
2. 按技术路线/问题维度归入 2~4 组，每组挑代表工作。
3. 每组用一句话概括核心思想，再点明在你场景下的局限。
4. 由各组局限收束到你补的缺口，写出一句话 delta。
5. 核对：将作基线的工作是否都已出现且准确？
6. 用 GB/T 7714—2015 顺序编码制统一著录，中文文献给中英文对照。
7. 若为综述，另补分类框架表、批判性比较与趋势展望。

按此步骤，相关工作既有条理又能立住创新性，减少外审对 delta 的质疑。
