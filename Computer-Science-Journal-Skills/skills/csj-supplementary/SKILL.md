# 《计算机科学》附录与补充材料组织

本技能帮你决定《计算机科学》(Computer Science, 简称 JSJKX) 稿件里**什么进正文、什么进附录/补充材料**。本刊
鼓励长文与综述，内容往往较多；但正文仍应保持主线清晰，让单盲外审专家高效抓住核心贡献，把支撑性、验证性
细节放到附录或补充材料。

> 提醒：本刊 Computer Science 是**期刊(journal)**，非会议。补充材料具体投递方式以官网《投稿须知》为准（**待核实**）。

## 一、划分原则："决定性"测试

对每块内容问：**删掉它，读者还能否理解并相信核心结论？**

- **能** → 可移入附录/补充材料。
- **不能** → 留正文。

## 二、正文应保留

- 核心 RQ 与主表/主图（有效性、关键消融）。
- 方法的关键机制与必要公式。
- 支撑摘要量化结论的证据。
- 主要基线比较与显著性结论。

## 三、适合放附录/补充材料

| 类别 | 说明 |
|---|---|
| 完整超参表 | 正文给关键设置，全表入附录 |
| 额外数据集/场景 | 主结论用代表数据集，其余入附录 |
| 定理证明细节 | 正文给命题与证明思路，完整推导入附录 |
| 扩展实验/敏感性曲线 | 支撑性、非决定性的补充实验 |
| 算法伪代码 | 长伪代码、实现细节 |
| 复现信息 | 环境、种子、数据获取、复现包链接 |
| 多媒体附件 | 可视化、演示视频等（若本刊接收） |

## 四、引用与可访问性

- 正文需**显式引用**每个附录/补充项（"详见附录 A""见补充材料 S1"），不留"孤儿附录"。
- 补充材料命名清晰（附录 A/B、S1/S2），与正文引用一致。
- 外部资源（数据、代码、视频）给稳定长期链接与访问日期；大文件外链+校验和（见 `csj-reproducibility`）。

## 五、综述稿的特殊处理

综述常有大体量的**分类表、文献对照表**。可将超长对照表、完整检索式与纳入清单放附录/补充材料，正文保留
分类框架图与关键比较，保持可读性。

## 六、常见问题与修正

| 问题 | 修正 |
|---|---|
| 正文塞满超参与次要实验，主线被淹 | 移入附录，正文只留决定性证据 |
| 关键消融被塞进附录 | 决定"每个模块必要"的消融应留正文 |
| 附录未被正文引用 | 每个附录项都要在正文点名引用 |
| 补充材料链接失效/不稳定 | 用长期仓库+校验和+访问日期 |

## 七、自查清单

1. 每块内容是否通过"决定性"测试判定归属？
2. 正文是否保留核心 RQ 证据与关键消融？
3. 超参全表、扩展实验、证明细节是否移入附录？
4. 每个附录/补充项是否在正文被引用、命名一致？
5. 外部资源是否稳定链接+校验和+访问日期？
6. 综述的大对照表是否合理下沉附录？

## 八、输出格式

```
【CSJ 正文/附录划分】
正文保留：<核心RQ证据/关键消融/关键公式…>
移入附录：<超参全表/扩展实验/证明/伪代码…>
补充材料：<数据/代码/多媒体 链接 + 校验和>
引用完整性：每个附录项在正文引用 <✓/✗>
结论：<划分合理 / 需调整：列点>
```

## 九、本刊长文/综述下的篇幅管理

《计算机科学》(Computer Science) 鼓励长文与综述，容量相对宽松，但"能写长"不等于"什么都堆进正文"。正确的
做法是：正文承载让读者理解并相信核心结论的**主线**，把验证性、参考性、工程性的细节系统地下沉到附录或
补充材料。尤其综述稿，正文放分类框架与关键比较，超长的文献对照大表、完整检索式与纳入/排除清单入附录，
既满足系统性又不牺牲可读性。

## 十、补充材料的可持续可访问

补充材料若依赖外部链接(数据、代码、演示视频)，要保证长期可访问：优先用机构库、Zenodo 等提供 DOI 的平台，
给出校验和(SHA256)与访问日期，避免个人网盘或临时地址在见刊后失效。正文引用补充材料时注明获取方式。对
敏感或受限数据，提供合规的脱敏子集或申请流程说明，不要留"查无此文件"的死链。

## 十一、与投稿系统的配合

不同期刊对补充材料的投递方式不同(随正文打包、单独上传、或仅提供链接)，《计算机科学》(Computer Science) 的
具体要求以官网《投稿须知》与投稿系统内的上传选项为准(**待核实**)。投稿前确认：补充材料该以何种形式提交、
是否计入篇幅、是否参与外审。把这一步与 `csj-submission` 的就绪审计一起完成，避免临投稿才发现材料无处上传。

## 十二、附录组织的可读性

附录不是"杂物间"，也应有结构。为《计算机科学》(Computer Science) 稿件组织附录时：按主题分节(附录 A 超参、
附录 B 证明、附录 C 扩展实验…)，每节有小标题与一句引导；正文在需要处显式指向对应附录节；附录内的图表
延续正文编号规则(如图 A1、表 B2)并各有题注。读者从正文跳到附录应能立刻找到目标，而非在一大段无结构的
材料里翻找。良好的附录结构让外审愿意去看，你补充的证据才真正发挥作用。

## 十三、正文与附录的动态调整

正文与附录的划分不是一次定死，而应随审稿反馈动态调整。若外审认为某个被你放进附录的实验其实很关键，修回
时应把它**上移正文**；反之若外审嫌正文冗长，可把次要内容**下沉附录**。这种调整要与 `csj-author-response`
的逐条回应配合，并在修改稿中清晰标注移动。始终以"决定性"为准绳：决定读者是否相信核心结论的证据留正文，
其余下沉，随评审意见微调，直到正文主线清晰、附录支撑充分。

## 十四、补充材料准备清单

为《计算机科学》(Computer Science) 投稿准备附录/补充材料时，逐项落实：

1. 用"决定性"测试判定每块内容归正文还是附录。
2. 附录按主题分节(超参/证明/扩展实验/伪代码)，各有小标题。
3. 附录图表延续正文编号规则并各有中英文题注。
4. 正文对每个附录/补充项显式引用，无孤儿附录。
5. 外部资源(数据/代码/视频)用稳定平台+校验和+访问日期。
6. 敏感数据提供合规脱敏子集或申请说明。
7. 确认本刊补充材料的投递方式(**待核实**，见《投稿须知》)。

按此清单，补充材料既完整可查，又不拖累正文主线的可读性。
