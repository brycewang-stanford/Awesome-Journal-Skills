# 《计算机科学》中文写作与体例规范

本技能帮助你把稿件写到符合《计算机科学》(Computer Science, 简称 JSJKX) 的中文科技体例。本刊是创刊于
1974 年的计算机全学科中文综合月刊(CCF 会刊、B 类、T2 级)，编辑初审首先看**体例是否规范**：中英文摘要、
关键词、中图分类号、参考文献格式任一不合规都可能在初审被退回。体例过关，外审专家(单盲)才会把注意力放在
学术贡献上。

> 提醒：本刊《计算机科学》英文名 Computer Science，是**期刊(journal)** 而非会议。

## 一、题名（中英文）

- 中文题名准确、具体、可检索，一般不超过 20 字，慎用缩写与"研究""方法"等空词堆叠。
- 英文题名与中文对应，实词首字母大写(或按刊物样式)，避免中式直译与冠词滥用。

## 二、摘要（报道性，四要素）

中文摘要须为**报道性摘要**，包含：

1. **目的**：针对什么问题/缺口；
2. **方法**：提出了什么(方法/模型/系统)、关键机制；
3. **结果**：主要量化结果（相对哪些基线、提升多少、显著性）；
4. **结论**：说明意义与适用范围。

要求：不用"本文"过多起句、不加评价性溢美、不含未在正文出现的结论；一般 200~300 字量级。英文摘要 Abstract
与中文一一对应，建议请专业润色，避免语法与术语错误。

## 三、关键词（中英文）

- 3~8 个，覆盖问题域、方法、对象；中英文一一对应。
- 颗粒度适中：既非过泛（"人工智能"），也非过窄（一次性生造词）。

## 四、中图分类号

- 计算机学科多用 **TP** 类（如 TP391、TP18 等），据文选取；必要时给出文献标识码。
- 依据《中国图书馆分类法》，不确定时参照同栏目近刊文的取号。

## 五、参考文献：GB/T 7714—2015 顺序编码制

这是本刊格式审查的重点：

- **顺序编码制**：正文按引用先后以上标数字标注 `[1]`、`[2]`…，文后参考文献表按该序号排列。
- 各文献类型标识：期刊 `[J]`、专著 `[M]`、会议论文 `[C]`、学位论文 `[D]`、报告 `[R]`、标准 `[S]`、
  电子资源 `[EB/OL]` 等。
- **中文文献建议中英文对照**（部分栏目要求），作者、题名、刊名、年卷期页齐全。
- 著录示例（示意）：

```
[1] 张三, 李四. 面向跨域推荐的图对比学习方法[J]. 计算机科学, 2026, 53(6): 1-10.
[2] Smith J, Doe A. Graph Contrastive Learning for Recommendation[C]//Proc of XXX. 2025: 100-110.
```

- 用支持 GB/T 7714—2015 样式的文献管理器统一生成，避免手工编号错乱。

## 六、量、单位与数字

- 一律使用**法定计量单位**（SI），单位符号正体；变量斜体、矢量黑斜。
- 数字用法遵循国标：统计量、测量值用阿拉伯数字；避免中英文数字混用。

## 七、图表与公式

- 图/表须有**中英文题注与序号**（图 1 / Fig.1、表 1 / Table 1），先见文字后见图表。
- 图分辨率满足印刷，矢量优先；表用三线表。
- 公式居中、右侧编号；符号首次出现即定义，全文一致。

## 八、语言风格

- 科技语体：客观、简洁、准确；少用第一人称口语与夸张评价。
- 术语全文统一，中英文缩写首次出现给全称。
- 避免长难句堆叠，一段一意；逻辑连接词准确。

## 九、体例自查清单

1. 中英文题名/摘要/关键词是否齐全、对应？摘要是否报道性四要素？
2. 中图分类号(TP…) 是否给出？
3. 参考文献是否 GB/T 7714—2015 顺序编码制、类型标识齐全、中文文献中英文对照？
4. 图/表是否中英文题注+序号、三线表、矢量图？
5. 量与单位是否法定计量单位、变量斜体？
6. 术语与缩写是否全文一致、首现给全称？
7. 篇幅是否达标（一般≥8000 字）、是否符合所投栏目/专题体例？

## 十、输出格式

```
【CSJ 体例审计】
题名[中/英]：<✓/✗>
摘要：报道性四要素[目的/方法/结果/结论] <✓/缺…>
关键词[中/英 3~8]：<✓/✗>
中图分类号：<TP… / 缺>
参考文献：GB/T 7714-2015 顺序编码制 <✓/✗>；中文文献中英文对照 <✓/✗>
图表：中英文题注+三线表 <✓/✗>
单位/术语：法定单位 <✓/✗>；术语一致 <✓/✗>
结论：<可过初审 / 需修改：列点>
```

所有格式口径以官网《投稿须知》与官方模板最新版为准。

## 十一、研究长文 vs 综述的体例差异

《计算机科学》(Computer Science) 同时发研究长文与综述，二者体例侧重不同。研究长文的骨架是"引言→相关工作
→方法→实验→结论"，重心在方法创新与实验证据，摘要须给量化结果。综述的骨架是"引言→范围与方法→分类框架
→各类方法批判性比较→公开问题与趋势→结论"，重心在系统性与作者判断，摘要以"梳理了什么、提出何种分类、
指出哪些趋势"为主。写作前先据 `csj-topic-selection` 定下类型，再选对应骨架，避免用长文模板套综述或反之。

## 十二、英文摘要(Abstract)常见问题

本刊要求中英文摘要，英文摘要是国际检索与传播的门面，也是外审关注点之一。常见问题：一是中式直译，句式生硬、
主谓不清；二是时态混乱(方法与结果多用一般现在/过去时需统一)；三是术语与正文不一致；四是把中文的评价性
语句照搬(如"取得了很好的效果")。建议：英文摘要独立按英文科技写作重写而非逐句翻译，请母语或专业人士核对，
关键术语与正文、关键词严格一致。

## 十三、写作纪律与红线

- 术语、符号、缩写全文统一，首次出现给全称与定义。
- 不出现未在正文支撑的结论；摘要数字必须与实验一致。
- 参考文献只引真实、可核查的文献，杜绝"凑引用"与错误著录。
- 图表不得误导（坐标截断、选择性展示）；一图一表都要有正文解读。
- 遵守学术规范，避免自我抄袭与不当重复发表。以上任一红线触碰，都可能在编辑初审或外审被退。
