# 《计算机科学》投稿就绪审计

在把稿件上传到《计算机科学》(Computer Science, 简称 JSJKX) 官网在线投稿系统之前，先跑完这份审计。
《计算机科学》是重庆西南信息有限公司主办、创刊于 1974 年 1 月的**计算机全学科中文综合月刊**，是 CCF 会刊、
CCF 推荐 B 类中文科技期刊、CCF 计算领域高质量科技期刊分级目录 T2 级，同时是 CSCD 来源、北大中文核心。它
**覆盖面广、发文量较大、专题(专栏)与综述活跃**，鼓励综述与长文——这与门槛更高、以长文为主的《计算机学报》
(Chinese Journal of Computers) 定位不同。本刊按期刊稿件走**单盲审稿**（作者不知审稿人、审稿人知作者），
所以投稿被当作一篇要经受编辑初审、专家外审、主编终审的完整学术论文来评判。

> 类型辨析：本刊是**期刊(journal)**，英文刊名 Computer Science，不是会议；请勿与同名会议或英文刊混淆。

以下事实于 2026-07-09 通过官网与 CNKI/CCF 目录交叉核验（见 `resources/official-source-map.md`），凡本刊未
公开确认的数字（版面费金额、审稿/见刊周期、录用率）均标 **待核实**，请以投稿系统内最新《投稿须知》为准。

## 一、投稿系统与唯一通道

- 本刊**仅**接受官网 `https://www.jsjkx.com` "作者投稿"在线系统提交的稿件，其他第三方"投稿代理"域名并非
  官方通道，认准官网入口，避免误投。
- 注册账号信息（题名、作者、单位、通信作者、研究方向）决定编辑分派外审专家的匹配度，务必用**真实终稿信息**，
  不要用占位标题。
- 通信作者邮箱须长期可达；后续初审反馈、外审意见、修回通知均通过系统与邮件送达。

## 二、稿件六件套（缺一不可）

| 组件 | 要求 |
|---|---|
| 中英文题名 | 准确、可检索；英文题名规范大小写，避免中式直译 |
| 中文摘要 | **报道性**：目的—方法—结果—结论四要素齐全，含量化 |
| 英文摘要 Abstract | 与中文对应，结构完整，建议专业润色 |
| 中英文关键词 | 3~8 个，中英文一一对应，颗粒度适中 |
| 中图分类号 | 计算机学科多为 **TP** 类，据文选取 |
| 参考文献 | **GB/T 7714—2015 顺序编码制**，正文上标、文后按序编号 |

正文另需：引言、相关工作、方法、实验、结论；基金项目号、作者简介、通信作者信息、利益冲突声明。

## 三、栏目与专题(专栏)归属

- 《计算机科学》(Computer Science) 栏目众多：计算机科学理论、体系结构、软件、数据库/大数据/数据科学、
  图形学/多媒体、人工智能、人机交互/普适计算、计算机网络、信息安全、交叉前沿等。投稿时**明确你的栏目**。
- 本刊常发布**专题(专栏)征稿**。若你的选题正对接某个正在征稿的专题，应在投稿系统中按专题要求投递，并在
  信中说明——这是本刊区别于兄弟刊的一条"快车道"（详见 `csj-topic-selection`）。
- 综述稿是本刊鼓励的类型，需按综述体例组织（详见 `csj-related-work`）。

## 四、篇幅与格式

- 投稿一般**建议不少于 8000 字**，鼓励长文与综述；具体上限以官方模板与《投稿须知》为准。
- 使用本刊要求的排版格式，提交 PDF（必要时附源文件）；图/表须有中英文题注与序号，公式规范排版，
  量与单位用**法定计量单位**。

## 五、单盲审稿下的署名与伦理

- 本刊为**单盲**：审稿人可见作者与单位，因此**无需匿名化**正文，但须：
  - 完整**利益冲突声明**与基金项目号；
  - 全部作者知情同意、署名顺序无争议、通信作者明确；
  - 遵守学术规范：**严禁一稿多投、重复发表、抄袭与不当署名**。
- 投稿前自查学术不端**总文字复制比**，控制在期刊可接受范围（阈值以编辑部为准，**待核实**）。

## 六、版面费与费用

- 录用后按刊物规定缴纳版面费；**第一作者为 CCF 会员的录用稿件版面费享 8.5 折**（准备好会员信息）。
- 具体版面费金额未公开统一口径，标 **待核实**，以录用通知为准。

## 七、退稿风险分诊（投稿前自评）

| 风险信号 | 触发的补救技能 |
|---|---|
| 选题与本刊栏目/专题不匹配，或更像 CCF-A 长文 | `csj-topic-selection` |
| 摘要非报道性、缺中图分类号、参考文献非 GB/T 7714 | `csj-writing-style` |
| 相关工作堆砌、创新点(delta) 不清 | `csj-related-work` |
| 实验无 RQ、基线不公平、无统计与消融 | `csj-experiments` |
| 结果无法复现、数据划分含泄漏 | `csj-reproducibility` |
| 未走官网系统、六件套不全、疑似一稿多投 | 本技能（重投前修正） |

## 八、提交前最终检查表

1. 是否通过官网 www.jsjkx.com 在线系统投稿？
2. 六件套是否齐全（中英文题名/摘要/关键词、中图分类号、正文、GB/T 7714 参考文献）？
3. 栏目/专题归属是否明确、是否对接正在征稿的专题？
4. 篇幅是否达标（一般≥8000 字）、图表题注中英文是否齐全？
5. 利益冲突声明、基金号、作者与通信作者信息是否完整？
6. 查重是否自查通过、是否确认无一稿多投？
7. 若第一作者为 CCF 会员，是否准备好信息以享版面费 8.5 折？

## 九、输出格式（交给作者的审计结论）

```
【CSJ 投稿就绪审计】
栏目/专题：<常规栏目名 或 专题名(征稿中?)>
六件套：题名[中/英] 摘要[中/英] 关键词[中/英] 中图分类号[TP…] 参考文献[GB/T 7714 是/否]
篇幅：约 <字数>（建议≥8000）
单盲伦理：利益冲突声明[✓] 一稿多投[无] 查重复制比[<自查值>]
版面费：第一作者 CCF 会员[是/否 → 8.5 折]
退稿风险：<高/中/低> → 建议先跑 <技能名>
结论：<可投 / 修改后可投 / 暂不建议投>
```

以上口径以官网《投稿须知》最新版为准；标 **待核实** 项请勿当作确定值对外陈述。

## 十、栏目匹配的常见判断

《计算机科学》(Computer Science) 栏目众多，投稿时若栏目选错，编辑可能改派或退回。给出一些经验判断：偏
理论证明与算法复杂度的稿件多归"计算机科学理论"；偏体系结构、处理器、存储与并行的归"计算机体系结构"；
偏编译、编程语言、软件工程方法的归"计算机软件"；数据管理、数据挖掘、数据科学归"数据库/大数据/数据科学"；
机器学习、深度学习、知识表示、智能体归"人工智能"；协议、路由、无线、数据中心网络归"计算机网络"；密码、
入侵检测、隐私保护归"信息安全"；渲染、可视化、音视频归"图形学/多媒体"。若稿件横跨多栏目，选**方法贡献
最集中**的那个作为主栏目，并在信中说明。对接专题时，以专题征稿通知的范围描述为准。

## 十一、投稿信(cover letter)要点

虽然本刊为在线系统投稿，仍建议在系统备注或投稿信中简述：本文的核心贡献一句话、所投栏目或专题、为何契合
《计算机科学》(Computer Science) 的读者、是否有利益冲突需回避的审稿人、以及声明无一稿多投与全部作者同意
投稿。投稿信是编辑初审时快速理解稿件定位的入口，不宜空泛，也不要夸大。

## 十二、被拒后的再投策略

若被本刊退稿，先分清是**范围/定位问题**还是**质量问题**：属定位问题(更应投 CCF-A 或方向更专的兄弟刊)则
考虑转投，用 `csj-topic-selection` 重新定位；属质量问题则据外审意见系统性补强(实验、写作、创新性)后，方可
考虑改投或在充分修改后再与编辑沟通。切忌原样改投他刊造成一稿多投或重复问题。任何再投都要重新过一遍本
技能的就绪审计与查重自查。
