# 《计算机科学》选题与栏目定位

在为《计算机科学》(Computer Science, 简称 JSJKX) 动笔或定投稿方向前，用本技能确认选题是否契合本刊。
《计算机科学》是创刊于 1974 年、面向**计算机全学科**的中文综合月刊，CCF 会刊、CCF 推荐 B 类、分级目录
T2 级。它的鲜明定位是**覆盖面广、发文量较大、专题(专栏)与综述活跃**，明确鼓励综述与长文投稿，报道涉及面
广的方法论、技术与能起承先启后作用的成果。这是它区别于兄弟刊的核心。

> 提醒：本刊是**期刊(journal)**，英文名 Computer Science，非会议。

## 一、先判"契合度"：本刊 vs 兄弟刊

| 期刊 | 定位侧重 | 何时更合适 |
|---|---|---|
| **《计算机科学》(Computer Science)** | 全学科、覆盖广、专题/综述活跃、面向应用与时效 | 稿件扎实但非顶尖长文；综述；对接专题；希望较快见刊面 |
| 《计算机学报》(Chinese Journal of Computers) | CCF-A、高门槛原创长文 | 理论/系统贡献很强、可写成充分长文 |
| 《软件学报》 | 软件/系统软件为主 | 软件工程、编程语言、系统软件方向 |
| 《自动化学报》 | 控制/自动化 | 控制理论、机器人、自动化 |
| 《电子学报》 | 电子/通信/信号 | 电路、通信、信号处理 |
| 计算机研究与发展 | 综合，偏系统与前沿 | 系统、体系结构、前沿综述 |

若你的工作**创新充分但未达 CCF-A 长文门槛，或是一篇高质量综述，或正好对接本刊某专题**，《计算机科学》
往往是更契合的选择。

## 二、栏目地图（据官网栏目页）

- 计算机科学理论
- 计算机体系结构
- 计算机软件
- 数据库 / 大数据 / 数据科学
- 计算机图形学 / 多媒体
- 人工智能
- 人机交互 / 普适计算
- 计算机网络
- 信息安全
- 交叉与前沿

动笔前先把选题**映射到 1 个主栏目**（必要时备选 1 个）。栏目决定外审专家池与体例参照。

## 三、稿件类型选择

1. **研究长文**：有明确 RQ、方法创新与充分实验；本刊主体类型。
2. **综述**：系统梳理某方向的问题、方法谱系与未来趋势；本刊**鼓励**，是相对兄弟刊的优势路径。需有分类框架、
   批判性比较，而非文献罗列（见 `csj-related-work`）。
3. **应用/技术报告类**：面向广覆盖、时效性强的方法与技术。

## 四、专题(专栏)快车道

- 本刊常发布**专题征稿**（特约/组稿），围绕某前沿方向集中刊发一组文章。
- 好处：主题聚焦、评审对口、见刊节奏可能更紧凑。
- 做法：留意官网/公众号的专题征稿通知，核对**主题范围、截稿时间、客座编辑**，在投稿系统按专题投递并在信中
  说明。若你的选题正落在某专题范围内，这是提升契合度与命中率的有效路径。

## 五、选题自评清单

1. 选题属于哪个栏目？是否有正在征稿的专题可对接？
2. 相对本刊近 1~2 年同栏目刊文，我的**增量(delta)** 是什么？一句话能否说清？
3. 类型选研究长文、综述还是应用类？体例是否匹配？
4. 是否更适合 CCF-A 的《计算机学报》或方向更专的兄弟刊？为何选本刊？
5. 数据/实验/理论证据是否足以支撑外审（单盲，审稿人知作者）？
6. 是否满足本刊广覆盖、重时效、鼓励综述与长文的定位？

## 六、常见误配与修正

| 误配 | 修正 |
|---|---|
| 顶尖理论长文投本刊"求稳" | 评估是否更应冲 CCF-A《计算机学报》 |
| 把文献综述当研究论文投 | 明确按综述体例组织，或补足方法创新 |
| 选题与所有栏目都沾边、都不深 | 收敛到 1 个主栏目，突出单一贡献 |
| 强行蹭无关专题 | 只对接真正落在范围内的专题，否则走常规栏目 |

## 七、输出格式

```
【CSJ 选题定位】
主栏目：<栏目名>（备选：<栏目名>）
专题：<无 / 专题名(征稿中，截稿 待核实)>
类型：<研究长文 / 综述 / 应用>
一句话 delta：<……>
契合度：<高/中/低>；相对兄弟刊为何选本刊：<……>
建议：<推进 / 转投更合适刊 / 补足证据后再定>
```

结论以官网栏目页与专题征稿通知最新版为准，专题截稿等具体信息标 **待核实**。

## 八、综述选题的额外考量

《计算机科学》(Computer Science) 鼓励综述，但综述被拒的常见原因是"只综不述"——缺少作者自己的分类框架与
判断。选综述题时先自问：这个方向近 1~2 年是否有足够新工作值得系统梳理？是否已有权威综述、我的增量在哪
(新视角、新分类、新数据对比)？读者(本刊覆盖全学科的广泛读者)是否需要一篇入门+前沿并重的地图？若这三问
都正面，综述选题才立得住。综述通常长文、参考文献量大，务必预估工作量。

## 九、面向"覆盖广、时效强"定位的选题微调

本刊定位覆盖面广、重视方法论与技术的普适价值与时效。因此在同等质量下，能**惠及多个方向读者**、**方法可
迁移**、**回应当前热点但有扎实证据**的选题更契合。相反，极度专门化、只对小圈子有意义、或纯增量调参的选题
契合度偏低。可用一句话检验："本刊读者(横跨理论到应用)读了会觉得有用吗？"

## 十、选题到写作的交接

选题定稿后，把三件事交给下游技能：主栏目/专题、稿件类型、一句话 delta。`csj-writing-style` 据类型确定体例
(研究长文 vs 综述)，`csj-related-work` 据 delta 组织相关工作，`csj-experiments` 据 RQ 设计实验。选题阶段的
含糊会一路放大到外审，故务必在此收敛清楚，避免"什么都想讲、什么都没讲透"。

## 十一、选题成熟度自评

在正式动笔前，用三档给选题成熟度打分，判断是否值得投入《计算机科学》(Computer Science)：

- **成熟(可推进)**：栏目/专题明确、一句话 delta 清晰、已有初步证据支撑、契合本刊广覆盖定位。
- **待打磨(需补强)**：方向对但 delta 模糊或证据不足，需先补实验或收敛贡献再定。
- **不契合(宜转向)**：更应投 CCF-A 长文刊或方向更专的兄弟刊，或选题本身价值不足。

只有评为"成熟"才进入写作流水线；评为"待打磨"先补强再复评；评为"不契合"则据 `csj-submission` 的再投策略
转向更合适的出口。选题阶段多花半天收敛，往往省下投稿后数月的返工。

## 十二、选题定位输出小结

完成选题定位后，用一句话向自己复述结论并交给下游：本文拟投《计算机科学》(Computer Science) 的
<某栏目/专题>，类型为<研究长文/综述>，核心增量(delta)是<一句话>，相对 CCF-A 的《计算机学报》与方向更专
的兄弟刊，选本刊的理由是<覆盖广/综述活跃/对接专题/时效>。这句话清晰了，写作、相关工作与实验才有统一
锚点；含糊，则一路放大到外审。定位以官网栏目页与专题征稿通知最新版为准，专题截稿等信息标 **待核实**。
