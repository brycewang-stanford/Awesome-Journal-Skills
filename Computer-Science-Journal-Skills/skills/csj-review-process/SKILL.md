# 《计算机科学》审稿流程建模

本技能帮你理解《计算机科学》(Computer Science, 简称 JSJKX) 投稿后的审稿流程，并在每个环节找到**作者可用的
杠杆**。本刊采用**单盲审稿制**（作者不知审稿人姓名、审稿人知作者姓名），按期刊稿件逐轮处理。具体天数/轮次
本刊未公开统一口径，均标 **待核实**（见 `resources/official-source-map.md`），以编辑部通知为准。

> 提醒：本刊 Computer Science 是**期刊(journal)**，非会议；没有会议式的一轮 rebuttal 截止，而是期刊式多轮修回。

## 一、流程总览

| 阶段 | 谁负责 | 主要判据 | 可能结果 |
|---|---|---|---|
| 编辑初审 | 编辑部 | 是否符合范围/体例/学术规范/查重 | 通过送外审 / 格式退修 / 直接退稿 |
| 专家外审 | 同行专家(单盲) | 创新性、正确性、实验、写作、意义 | 录用 / 小修 / 大修 / 退稿 |
| 复审退修 | 编辑+专家 | 修回是否回应意见 | 再送审 / 接受 / 退稿 |
| 主编终审 | 主编 | 综合把关 | 录用 / 退稿 |
| 修回 | 作者 | 逐条回应+修改说明 | 进入下一轮 |

## 二、编辑初审（第一关）

- 查范围与栏目匹配、体例(中英文摘要/关键词/中图分类号/GB/T 7714)、学术规范与查重。
- **作者杠杆**：投稿前用 `csj-submission` 与 `csj-writing-style` 自查，避免因体例/查重在初审被退。
- 选题不匹配则可能直接退稿——先用 `csj-topic-selection` 确认契合度。

## 三、专家外审（单盲）

- 专家可见你的姓名与单位，因此**学术形象与利益冲突声明**重要；但评审仍以稿件质量为准。
- 关注：创新性(delta)、方法正确性、实验公平与可复现、写作与可读性、对本刊读者的意义。
- **作者杠杆**：把 `csj-related-work`、`csj-experiments`、`csj-reproducibility` 做扎实，减少外审疑点。

## 四、修回：小修 / 大修

- **小修**：局部澄清与补充，通常不重新外审。
- **大修**：需补实验/重写章节，**常重新送外审**；把大修当"第二次投稿"认真对待。
- 修回须附**逐条答复(response letter)** + **修改说明**（见 `csj-author-response`），改动在文中可定位。

## 五、复审与主编终审

- 复审看修回是否切实回应；反复未回应易走向退稿。
- 主编终审综合把关，做最终录用/退稿决定。

## 六、时间预期与管理

- 首轮外审周期、见刊周期本刊未公开统一口径（见刊约 13 个月为部分渲染，**待核实**）。
- 管理预期：留足修回时间；对长时间无状态更新，可礼貌地通过系统/邮件询问编辑部。

## 七、各阶段作者行动清单

1. 投稿前：`csj-topic-selection` + `csj-submission` + `csj-writing-style` 自查。
2. 送外审后：准备可能的补充实验与复现材料。
3. 收到意见：用 `csj-author-response` 逐条答复，区分小修/大修。
4. 大修：优先补最能消除退稿风险的实验，改动可定位。
5. 终审后录用：进入 `csj-camera-ready`。

## 八、输出格式

```
【CSJ 流程定位】
当前阶段：<初审/外审/复审/终审/修回>
判据关注点：<该阶段主要看什么>
可能结果：<录用/小修/大修/退稿>
作者杠杆：<该阶段应做的 1~3 件事 + 对应技能>
时间预期：<待核实，以编辑部通知为准>
下一步：<具体行动>
```

所有周期与轮次以官网《投稿须知》与编辑部通知最新口径为准，本技能中标 **待核实** 项不作确定值陈述。

## 九、单盲审稿对作者的具体含义

《计算机科学》(Computer Science) 采用单盲：审稿人看得到你的姓名与单位。这带来两点实务影响。其一，正文与
致谢**无需匿名化**，可正常署名、致谢基金与合作者，但必须给出完整的利益冲突声明，避免与审稿人存在明显
利益关联而未披露。其二，由于身份可见，作者的既往学术信誉与稿件的严谨程度会共同影响印象，因此更要把实验、
写作、参考文献做扎实，用质量而非"名气"说话。若有确需回避的潜在审稿人，可在投稿时按系统规则提出。

## 十、编辑与作者的沟通礼仪

投稿后所有正式沟通以投稿系统与编辑部邮件为准。查询进度时应礼貌、简洁、给出稿号，避免频繁催促；收到意见
后按期回复，确需延期应提前说明。与编辑保持专业、克制的沟通，有助于稿件顺利推进。切勿绕过系统私下联系
审稿人，这违反单盲评审的基本规范。

## 十一、从流程结果到下一步

- **录用/小修**：进入 `csj-author-response` 快速修回，随后 `csj-camera-ready`。
- **大修**：把它当第二次投稿，优先补最能消除退稿风险的证据，逐条答复。
- **退稿**：区分定位问题还是质量问题(见 `csj-submission` 再投策略)；定位问题考虑转投更契合的刊，质量问题
  系统补强后再定。任何情形都不要一稿多投或原样改投造成学术不端。

## 十二、各阶段的典型时长与心态

虽然《计算机科学》(Computer Science) 未公开统一的审稿时长口径(**待核实**)，但可给作者一个心态框架：编辑
初审通常较快(数日到一两周量级)，外审是最不确定的一段(取决于专家响应)，修回轮次视意见多寡可能一到多轮，
终审后到见刊还有排期(见刊约 13 个月，**待核实**)。作者应把整段旅程理解为"以月计"的马拉松而非"以周计"的
冲刺，投稿后同步推进其他工作，收到意见时集中发力。对异常长时间无更新，可礼貌查询，但不必过度焦虑。

## 十三、审稿意见的分级解读

收到外审意见后，先给每条意见分级：**致命级**(质疑创新性/正确性，不解决则退稿)、**重要级**(缺实验/写作不清，
影响录用)、**建议级**(优化性意见)。修回时按此优先级投入精力：先用扎实证据消除致命级质疑，再补齐重要级，
最后从善如流处理建议级。这个分级直接决定 `csj-author-response` 的答复重心，也决定大修能否翻盘。把有限精力
用在最能改变结局的意见上，是修回的核心策略。

## 十四、投稿后进度自我定位清单

投稿《计算机科学》(Computer Science) 后，用系统状态与下列信号自我定位当前阶段：

1. 状态"编辑处理/初审中"→ 编辑初审，尚未送外审。
2. 状态"外审中/专家审理"→ 已送同行专家(单盲)，最不确定的一段。
3. 收到"退修/修改"通知 → 进入修回，分清小修/大修。
4. 状态"复审/再审"→ 修回已提交，专家复核你的回应。
5. 状态"主编审定/终审"→ 接近最终决定。
6. 收到"录用"→ 转入定稿(`csj-camera-ready`)。
7. 收到"退稿"→ 分析定位/质量原因，据再投策略决定去向。

定位清楚当前阶段，才能对应调用正确技能、采取正确行动，而不是盲目等待或催促。
