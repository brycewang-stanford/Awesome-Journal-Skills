# External Tools - COLM

Access date: 2026-07-08

These are the live official surfaces for a COLM cycle plus the author-side passes to
run against them. COLM rewrites parts of its policy every edition — it is three
editions old — so nothing here substitutes for reopening the current pages.

If `colmweb.org` is unreachable from your network (it was blocked by the gateway used
to build this pack), read the pages through a search engine's rendering of the exact
URL and cross-check against the OpenReview group before acting.

## Author-side verification passes

1. **Identity pass** — every author has a current OpenReview profile; the submission
   PDF, appendices, and any linked material contain no acknowledgments, author
   names, lab names, or repository URLs that resolve to a person.
2. **Format pass** — main text within the strict 9-page cap on the current COLM
   template; citations on their own unlimited pages; no template surgery.
3. **Policy pass** — Code of Ethics acknowledged by all authors; LLM-usage
   disclosure completed truthfully against the current policy text, not last
   year's memory of it.
4. **Reciprocal-reviewing pass** — the nominated reviewer qualifies, is not reused
   across your group's other submissions, and anyone with 4+ submissions knows they
   are now in the reviewer pool.
5. **Evidence pass** — model versions (including API model strings and dates),
   decoding parameters, prompts, and compute are reported precisely enough that the
   evaluation could be re-run; contamination exposure is discussed rather than
   hidden.

## Do not infer

- Do not project the 2026 timeline onto 2027; COLM has already moved its dates,
  venue continent, and policy details between its first three editions.
- Do not assume a camera-ready page bonus, an artifact track, an acceptance-rate
  figure, or a virtual-attendance option — none of these was verifiable for 2026.
- Do not treat COLM as sharing machinery with ACL/EMNLP (no ARR, no anonymity
  embargo found) or with PMLR venues (no PMLR volume; papers live on OpenReview).
  Check the current COLM pages instead of importing sibling-venue rules.
- Where official pages and secondary coverage disagree, the newest official COLM
  page or a direct program-chairs message wins; record the conflict in the source
  map.
