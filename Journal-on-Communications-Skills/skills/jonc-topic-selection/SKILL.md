# 《通信学报》选题与栏目定位

在动笔前，先用本技能判断选题是否契合《通信学报》(Journal on Communications, JOC)。JOC 由中国通信
学会主办，是通信学科的中文旗舰**月刊**，只有真正落在**通信/网络/信息安全/信号处理**版图内、且
创新性与实证充分的稿件，才值得投入本刊三审制的时间成本。以下定位口径于 2026-07-09 依据官网栏目、
CNKI 期刊主页与 CCF 推荐中文科技期刊目录核验（见 `resources/official-source-map.md`）。

## 一、本刊学科版图

《通信学报》/Journal on Communications (JOC) 的核心方向：

| 方向 | 典型题目 |
|---|---|
| 通信理论与技术 | 信道编码、调制、信息论界、协作/中继、MIMO |
| 无线/移动通信 | 5G/6G、大规模天线、毫米波、星地融合、资源分配 |
| 网络与交换 | 路由、拥塞控制、SDN/NFV、数据中心网络、时延优化 |
| 信息安全与密码 | 密码协议、认证与密钥管理、物理层安全、隐私保护 |
| 信号处理 | 检测与估计、阵列信号、压缩感知、通信中的机器学习 |

若稿件主线不在上表，需谨慎——很可能更适合兄弟刊。

## 二、与兄弟中文刊的分工

| 期刊 | 侧重 | 与本刊边界 |
|---|---|---|
| 《电子学报》(Acta Electronica Sinica) | 宽电子学科（器件、电路、电磁、信号、通信） | 偏器件/电路/电磁 → 电子学报；偏通信系统与网络 → 本刊 |
| 《计算机学报》(Chinese Journal of Computers) | 计算机全学科综合长文 | 偏算法/体系结构/AI 核心 → 计算机学报；偏通信网络 → 本刊 |
| 《软件学报》(Journal of Software) | 系统软件、软件工程、编程语言 | 偏软件方法学 → 软件学报 |
| 《自动化学报》(Acta Automatica Sinica) | 控制、优化、智能系统 | 偏控制理论 → 自动化学报 |
| 《中国科学:信息科学》(SCIS) | 信息科学基础理论 | 偏基础理论且面向更宽读者 → SCIS |

**判定口诀**：贡献的"承重墙"是否是**通信系统、无线链路、网络协议、通信安全或通信信号处理**？
是 → 本刊；若承重墙是器件/电路/纯算法/控制/软件方法，则改投对应兄弟刊。

## 三、稿件类型选择

| 类型 | 适用 | 提示 |
|---|---|---|
| 学术论文 | 完整原创研究（方法+理论/系统+充分实验） | 本刊主力，篇幅约 12000 字符 |
| 技术报告 | 工程性强、以系统实现与实测为主 | 强调可复现与工程价值 |
| 综述 | 对某方向的系统梳理与展望 | 可延至约 15 页；须有批判性归纳而非罗列 |
| 短文 | 单点创新、篇幅较短 | 贡献聚焦、快速呈现 |

## 四、创新性自评（投前三问）

1. **delta 是否清楚？** 相对最强基线，本文多了什么可验证的改进（理论界更紧/性能更优/开销更低/
   安全性更强）？含糊的"效果更好"不足以过外审。
2. **实证是否够？** 通信类工作通常需要标准信道/拓扑下的仿真或实测、公平基线、消融、蒙特卡洛
   置信区间（见 `jonc-experiments`）。
3. **是否对口本刊？** 通过第二节分工表确认承重墙在通信学科内。

## 五、选题风险信号

| 信号 | 含义 | 建议 |
|---|---|---|
| 主线是器件/电路/电磁 | 更像《电子学报》 | 改投或把通信系统层面做成承重墙 |
| 主线是通用 ML 算法、通信只是应用皮 | 创新性存疑 | 强化通信侧建模与增益机理 |
| 只有单场景单次仿真 | 实证不足 | 补多场景、多信道、统计严谨 |
| 与作者会议版高度重叠 | 学术规范风险 | 明确新增贡献、控制查重 < 20% |
| 选题过于宽泛（"综述式论文"） | 缺创新增量 | 收敛到具体问题或改写为规范综述 |

## 六、立项产出模板

```
拟投栏目/类型：____（学术论文/技术报告/综述/短文）
学科落点：____（通信理论/无线移动/网络交换/信息安全/信号处理）
一句话贡献：相对 ____（最强基线），本文通过 ____ 在 ____ 指标上实现 ____
承重墙是否在通信学科：是/否（若否，改投 ____）
实证计划：信道/拓扑 ____，基线 ____，指标 ____，统计 ____
与会议版关系：无 / 扩展自 ____（新增：____）
```

## 七、热点与本刊契合度参考

以下方向近年在通信学科活跃，且与《通信学报》/Journal on Communications (JOC) 版图契合度高
（具体当期偏好以近期目次为准，**待核实**）：

| 热点方向 | 契合子领域 | 提示 |
|---|---|---|
| 6G 与空天地一体化 | 无线/移动、网络 | 强调通信机理与系统增益，非泛概念 |
| 大规模 MIMO/智能超表面(RIS) | 通信理论、信号处理 | 需信道建模与性能界 |
| 通信中的人工智能 | 信号处理、无线 | 避免"通用 ML 套壳"，突出通信约束 |
| 物理层安全与后量子密码 | 信息安全与密码 | 需威胁模型与安全性论证 |
| 语义通信/信源信道联合 | 通信理论 | 需明确指标与可比基线 |
| 确定性网络/时间敏感网络 | 网络与交换 | 需时延/抖动的系统级验证 |

> 提醒：热点只是加分项，**创新增量与实证充分性**才是录用关键；追热点但 delta 不清同样会被退。

## 八、投稿前的一句话定位测试

在动笔前，用一句话填空自测，若填不出具体内容，说明选题尚未成熟：

```
本文面向【通信子领域____】的【具体问题____】，相对【最强基线____】，
通过【核心方法____】，在【指标____】上取得【可验证改进____】，
该问题属于《通信学报》的【学科落点____】。
```

若"学科落点"一栏勉强或空白，优先重新定位或改投兄弟刊。

## 九、与其它技能的衔接

- 定位通过后 → `jonc-workflow` 排时间线；
- 写作阶段 → `jonc-writing-style`、`jonc-related-work`、`jonc-experiments`；
- 投前 → `jonc-submission` 审计。

## 十、要点回顾

- 先确认承重墙在通信学科（通信理论/无线移动/网络交换/信息安全/信号处理）。
- 用分工表把本刊与《电子学报》《计算机学报》《软件学报》《自动化学报》《SCIS》区分开。
- 据贡献强度选类型：学术论文/技术报告/综述/短文。
- 追热点是加分项，delta 清晰与实证充分才是录用关键。
- 用"一句话定位测试"自检，学科落点勉强或空白就重新定位或改投。

> 目标：确保投入的是《通信学报》/Journal on Communications (JOC) 真正想要的**通信学科原创长文**，
> 而不是投错门的宽电子或纯计算机稿件。
