# 《通信学报》补充材料与正文/附录切分

本技能帮助为《通信学报》(Journal on Communications, JOC) 稿件划定正文与附录/补充材料的边界。JOC 是
中国通信学会主办的通信学科中文旗舰月刊，正文篇幅约 12000 字符（综述可延至约 15 页），因此"什么
进正文、什么进附录"是一门必须处理好的取舍。以下与 2026-07-09 核验的本刊格式口径一致（见
`resources/official-source-map.md`；精确附录政策 **待核实**，以投稿系统为准）。

## 〇、正文/附录切分决策记录模板

对每块内容跑一遍"决定性三问"，用下面的记录固化归属：

```text
[通信学报正文/附录切分 / JOC main-vs-appendix]
内容块：____（如：完整安全性证明 / 额外多普勒场景 BER 曲线 / 超参搜索表）
Q1 移除后主结论是否崩塌？：是→正文 / 否→附录
Q2 复核创新性或正确性时必看？：是→正文 / 否→附录
Q3 仅为完备性佐证？：是→附录/补充材料
归属：正文 / 附录A / 补充材料 / 多媒体附件
正文引用位置：____（如"证明见附录A""完整参数见附录B表B1"）
篇幅影响：正文 __ 字符（目标≈12000；综述≤约15页）
```

## 一、核心判据："决定性三问"

1. **移除它，主结论是否崩塌？** 是 → 正文；否 → 附录。
2. **审稿人复核创新性/正确性时必看吗？** 是 → 正文；否 → 附录。
3. **它只是完备性佐证吗？** 是 → 附录/补充材料。

## 二、正文 vs 附录分配表

| 内容 | 归属 | 理由 |
|---|---|---|
| 核心算法/协议设计 | 正文 | 承载创新性 |
| 关键定理与主要证明思路 | 正文 | 支撑正确性 |
| 冗长证明细节/引理堆叠 | 附录 | 完备性佐证 |
| 主实验（主信道/主指标曲线） | 正文 | 决定性证据 |
| 超参搜索、额外场景、大参数表 | 附录 | 支撑性证据 |
| 系统框图/协议时序（关键） | 正文 | 帮助理解主线 |
| 数据集详细统计、预处理脚本说明 | 补充材料 | 可复现参考 |
| 演示视频、原始日志、代码链接 | 多媒体附件/补充 | 增强可信度 |

## 三、通信学科的典型取舍

- **物理层/信号处理**：主 BER/容量曲线与关键推导进正文；完整推导、额外信道、复杂度细算进附录。
- **网络与交换**：主吞吐/时延对比进正文；完整拓扑参数、更多负载、协议状态机细节进附录。
- **信息安全与密码**：威胁模型与安全性论证主线进正文；完整可归约证明、工具脚本、开销细表进附录。

## 四、篇幅治理

- 目标全文约 12000 字符（约 12 页）：正文只留"决定性"内容。
- 大表压缩为关键行，完整版入附录并在正文引用。
- 多个相似曲线合并为多子图或选代表性场景，其余入附录。
- 文字层面删冗词、合并重复描述（见 `jonc-writing-style`）。

## 五、补充材料规范

- 附录编号（附录 A/B…）或补充材料独立编号，正文明确引用（"详见附录 A"）。
- 补充材料应可独立理解，含必要上下文。
- 多媒体附件说明格式、时长、内容；给可获取方式。
- 代码/数据链接：匿名评审阶段用匿名仓库（见 `jonc-reproducibility`）。

## 六、可获取性与许可

- 公开补充材料：说明许可与版本。
- 受限内容：说明获取方式与 **待核实** 的许可限制。
- 确认本刊是否随刊发布补充材料、是否有大小/格式限制（**待核实**，以投稿系统为准）。

## 七、自查清单

- [ ] 每块内容都过了"决定性三问"。
- [ ] 正文只含决定主结论与复核必看的内容。
- [ ] 冗长证明、超参、额外场景、大表已移入附录。
- [ ] 附录/补充材料编号规范、被正文引用。
- [ ] 多媒体附件与代码数据链接的获取方式清晰。
- [ ] 全文篇幅控制在约 12000 字符（综述≤约 15 页）。

## 八、按方向的补充材料清单模板

不同通信子方向，附录/补充材料的典型内容不同，可作勾选参考：

**物理层 / 信号处理**
- [ ] 完整定理证明与引理
- [ ] 额外信道（不同衰落/多普勒）下的 BER/容量曲线
- [ ] 复杂度详细推导与实测
- [ ] 更多 SNR/码率/天线配置的敏感性结果

**网络与交换**
- [ ] 完整拓扑与流量参数表
- [ ] 更多负载/规模下的吞吐-时延曲线
- [ ] 协议状态机/时序图完整版
- [ ] 仿真脚本与配置说明

**信息安全与密码**
- [ ] 完整可归约安全性证明
- [ ] 形式化验证脚本与结果（ProVerif/Tamarin）
- [ ] 计算/通信开销详表
- [ ] 攻击复现脚本（脱敏）

## 九、正文对附录的引用规范

- 每个附录/补充块必须在正文被引用（"证明见附录 A""完整参数见附录 B 表 B1"）。
- 引用要指到具体编号，不写"见附录"这类模糊指向。
- 附录图表独立编号（图 A1、表 B1），与正文编号体系区分但风格一致。

## 十、反模式

| 反模式 | 问题 | 修法 |
|---|---|---|
| 把关键证明塞进附录 | 正文正确性不自足 | 主线证明回正文 |
| 正文堆超参与冗表 | 篇幅臃肿、失焦 | 移入附录 |
| 附录内容正文从不引用 | 割裂 | 补引用或删除 |
| 补充材料无法独立看懂 | 复核困难 | 补上下文 |
| 用附录规避篇幅（正文内容硬塞附录） | 逻辑断裂 | 决定性内容回正文，压缩文字而非搬运 |

## 十点五、篇幅超限的应急取舍

若终稿超出约 12000 字符（约 12 页），按优先级压缩，切勿删关键证据：

| 优先压缩 | 保留 |
|---|---|
| 冗长的背景铺陈 | 核心贡献与主结果 |
| 重复的方法描述 | 关键算法与定理 |
| 次要场景的多余曲线 | 主信道/主指标曲线 |
| 完整参数大表（移附录） | 关键参数摘要 |
| 过度展开的相关工作 | delta 对比与定位 |

压缩顺序：先删冗词与重复，再移次要内容入附录，最后才考虑合并图表；始终保住"决定性"内容。

## 十一、要点回顾

- 用"决定性三问"划边界：移除后主结论是否崩塌、复核是否必看、是否只是佐证。
- 正文只放决定主结论与复核必看的内容；冗长证明、超参、额外场景、大表入附录。
- 按物理层/网络/安全三方向的清单模板整理附录与补充材料。
- 每个附录块必须被正文以具体编号引用；附录图表独立编号但风格一致。
- 目标全文约 12000 字符（综述≤约 15 页），靠压缩文字而非搬运内容达标。

> 切分定稿后，回到 `jonc-submission` 核对篇幅与格式。目标是让《通信学报》/Journal on Communications
> (JOC) 的正文既完整承载创新性，又不因臃肿而触碰篇幅与可读性红线。
