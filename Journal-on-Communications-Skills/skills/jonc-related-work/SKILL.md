# 《通信学报》相关工作与文献综述

本技能帮助把《通信学报》(Journal on Communications, JOC) 稿件的相关工作写成**创新性论证**，而非
文献流水账。JOC 是中国通信学会主办的通信学科中文旗舰月刊，外审专家熟悉本领域脉络，最反感"堆
文献却说不清自己多了什么"。以下方法与 2026-07-09 核验的本刊定位一致（见
`resources/official-source-map.md`）。

## 一、核心原则：delta 优先

相关工作的每一段都应回答：**前人做到哪，缺了什么，本文补了什么。** 组织主线不是时间或作者，而是
"问题维度 → 现有路线 → 各自局限 → 本文定位"。

## 二、组织范式（按子方向）

| 子方向 | 常见分类维度 | delta 常见落点 |
|---|---|---|
| 信道编码/调制 | 码型、译码复杂度、性能界 | 更紧的界 / 更低复杂度 / 更好 BER |
| 无线/移动传输 | 场景（毫米波/大规模天线/星地）、资源分配目标 | 更高谱效/能效 / 更低时延 / 更强鲁棒 |
| 网络与交换 | 层次（路由/传输/MAC）、指标 | 更高吞吐/公平 / 更低开销 |
| 信息安全与密码 | 威胁模型、原语、开销 | 更强安全性 / 更低计算通信开销 |
| 信号处理 | 检测/估计/学习方法 | 更优估计精度 / 更少样本 / 可解释 |

## 三、写作结构建议

1. **总领句**：点明本文问题所在的研究脉络。
2. **分组归纳**：按 2~4 条技术路线分组，每组代表性工作 + 共同局限。
3. **对照定位**：明确本文属于哪条路线、突破了哪个局限。
4. **收束句**：自然过渡到本文方法（"针对上述不足，本文……"）。

## 四、与兄弟刊读者的差异

- 本刊读者是**通信学科**专家，对信道模型、协议细节、安全模型敏感——综述要落到**通信机理**，
  而非泛泛的"AI/算法"叙述。
- 若引用《电子学报》《计算机学报》《软件学报》类工作，要说明其与本文通信问题的关联，避免让编辑
  觉得选题更适合兄弟刊。

## 五、引用规范（GB/T 7714）

- 顺序编码制，正文角标"[3]"或"文献[3]"。
- **中英文文献并重**：既引国际顶刊/顶会（IEEE Trans./INFOCOM 等），也引本领域中文权威（含本刊
  往期），显示对中外脉络的把握。
- 优先近 3~5 年关键工作；避免只引十年前旧文而漏掉最新进展。
- 每条文献著录完整、可溯（见 `jonc-writing-style` 的 GB/T 7714 示例）。

## 六、反模式清单

| 反模式 | 问题 | 修法 |
|---|---|---|
| 罗列式（"A 做了…B 做了…"） | 看不出 delta | 改为按局限分组 + 对照定位 |
| 漏引近年关键工作 | 外审质疑新颖性 | 补近 3 年顶刊/顶会 + 本刊往期 |
| 只引英文/只引中文 | 视野片面 | 中英并重 |
| 贬低前人 | 学术礼仪失当 | 客观陈述局限，不情绪化 |
| 相关工作与贡献不呼应 | 逻辑断裂 | 让每个 gap 对应一条贡献 |

## 七、创新性论证闭环

把相关工作、引言贡献、实验证据三者对齐：

```
gap_i（相关工作指出）→ contribution_i（引言承诺）→ evidence_i（实验兑现）
```

三者一一对应时，外审很难以"创新性不足"退稿。

## 八、综述体例特别提示

若投**综述**（可延至约 15 页）：

- 须有**批判性归纳**（分类框架、对比表、趋势与挑战），而非文献目录。
- 给出清晰的分类维度与对比表；指出未解难题与研究展望。
- 覆盖面与时效性并重，尤其纳入近年本刊与国际关键进展。

## 九、对比表：让 delta 一目了然

外审最欣赏的相关工作呈现之一，是一张**方法对比表**，把本文与代表性工作沿关键维度对齐：

| 方法 | 场景/信道 | 关键指标 | 复杂度/开销 | 局限 |
|---|---|---|---|---|
| 文献[3] | AWGN | BER 中 | 低 | 未考虑多径 |
| 文献[7] | 多径 | BER 优 | 高 | 复杂度高 |
| 本文 | 多径+多普勒 | BER 优 | 中 | —（本文突破点） |

用一句话在表下点明："本文在保持中等复杂度的同时，覆盖了[7]未处理的多普勒场景"。

## 十、引用礼仪与自引控制

- **客观**：陈述前人局限时对事不对人，避免"该文错误地……"式表达。
- **自引克制**：合理引用自己前序工作，但避免过度自引推高相关度或触碰查重红线（见
  `jonc-submission` 的 20% 口径）。
- **溯源准确**：概念/方法归属到最早或最权威来源，避免张冠李戴。
- **均衡**：既引国际顶刊顶会，也引本刊与国内权威中文文献，体现学科视野。

## 十一、自查清单

- [ ] 每段都能回答"前人缺了什么、本文补了什么"。
- [ ] 按技术路线/局限分组，非按时间罗列。
- [ ] 有一张方法对比表让 delta 一目了然。
- [ ] 中英文文献并重，近 3~5 年关键工作齐全。
- [ ] 引用合 GB/T 7714、著录完整；自引克制、溯源准确。
- [ ] gap—贡献—证据三者一一对应。
- [ ] 落点在通信机理，契合本刊读者。

## 十一点五、检索策略（保证覆盖面）

为避免漏引导致外审质疑，建议多源检索并交叉核对：

| 来源 | 用途 |
|---|---|
| IEEE Xplore | 通信顶刊顶会（Trans. Commun./Wireless、INFOCOM/GLOBECOM 等） |
| CNKI/万方/维普 | 本刊往期与国内权威中文文献 |
| dblp | 作者与会议脉络梳理 |
| Google Scholar | 引用链与最新预印本追踪 |

检索后按第二节子方向分组归档，确保每条技术路线都有代表作与最新进展。

## 十二、要点回顾

- 相关工作是创新性论证，不是文献流水账；主线是"局限—定位"而非时间。
- 按技术路线分组，每组给代表作与共同局限，再对照本文突破点。
- 用一张方法对比表让 delta 一目了然；落点扣在通信机理。
- 中英文文献并重、近三年关键工作齐全、合 GB/T 7714、自引克制。
- 让 gap—贡献—证据三者一一对应，堵住"创新性不足"的退稿口。

> 相关工作定稿后，用 `jonc-experiments` 让每个 delta 有可验证的证据，用 `jonc-writing-style` 统一
> 引用格式。目标是让《通信学报》/Journal on Communications (JOC) 外审一眼看清本文的通信学科增量。
