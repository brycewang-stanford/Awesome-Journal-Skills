# 《通信学报》实验设计与呈现

本技能规范《通信学报》(Journal on Communications, JOC) 通信学科长文的实证部分。JOC 由中国通信学会
主办，外审专家对信道模型、公平基线与统计严谨性有专业期待。以下方法与本刊通信学科定位一致
（2026-07-09 核验，见 `resources/official-source-map.md`）；无独立制品徽章制度（见
`jonc-artifact-evaluation`），但实验可信度直接决定录用。

## 一、用 RQ 驱动实验

先写研究问题，再设计实验：

```
RQ1：在标准信道下，本文方法相对最强基线能否在 ____ 指标上显著改进？
RQ2：改进对关键参数（SNR/天线数/节点数/密钥长度）是否稳健？
RQ3：改进的代价（复杂度/开销/能耗）是否可接受？
```

每个 RQ 对应一组图表与一个明确结论。

## 二、信道与场景（物理层/无线）

| 要素 | 要求 |
|---|---|
| 信道模型 | 明确 AWGN / Rayleigh / Rician / 多径 / 多普勒；给参数 |
| 场景 | 单/多用户、天线配置、频段、带宽、移动速度 |
| 指标 | BER/BLER、容量/谱效、能效、时延、覆盖 |
| 曲线 | SNR-BER、容量-SNR、CDF；标注置信区间 |

## 三、网络与交换实验

| 要素 | 要求 |
|---|---|
| 仿真器/平台 | NS-3 / OMNeT++ / Mininet / 真实拓扑，注明版本 |
| 负载与拓扑 | 节点数、流量模型、拓扑规模 |
| 指标 | 吞吐、时延、丢包、抖动、公平性（Jain 指数）、开销 |
| 对照 | 经典协议 + 近年 SOTA |

## 四、信息安全与密码实验

| 要素 | 要求 |
|---|---|
| 威胁模型 | 攻击者能力、安全目标（机密性/完整性/认证/隐私） |
| 安全性论证 | 可归约证明 / 形式化验证（ProVerif/Tamarin） |
| 性能开销 | 计算开销、通信轮数与比特、存储；与基线对比 |

## 五、统计严谨性（外审重点）

- **蒙特卡洛仿真**：报告仿真次数/独立实验数；给**置信区间**或误差棒，而非单次结果。
- **随机性控制**：固定随机种子与信道实现，保证可复现（见 `jonc-reproducibility`）。
- 显著性：必要时给出统计检验或多次运行的均值±标准差。
- 避免"精挑参数"：说明参数选取依据，提供敏感性分析。

## 六、公平基线原则

| 原则 | 说明 |
|---|---|
| 同信道同场景 | 基线与本文在相同信道、参数、指标下比较 |
| 最强而非稻草人 | 选近年 SOTA 而非过时或故意弱化的方法 |
| 同等调优 | 基线也做合理调参，避免"未调优基线" |
| 报告代价 | 不仅报性能增益，也报复杂度/开销代价 |

## 七、消融与敏感性

- **消融**：逐一移除本文模块，量化各自贡献，证明设计必要性。
- **敏感性**：扫描关键参数（码率、SNR、天线数、节点数、密钥长度），画趋势曲线。
- **边界场景**：低 SNR、高移动、大规模、强攻击等极端条件下的表现。

## 八、图表呈现

- 主结果曲线矢量化（TikZ/Matplotlib），黑白可辨（线型+标记）。
- 坐标轴标量与单位；图例清晰；置信区间可见。
- 复杂度/开销用表格，列时间/空间复杂度与实测。
- 图表编号连续、被正文引用、结论明确（见 `jonc-writing-style`）。

## 九、常见退稿诱因（实验层面）

| 问题 | 后果 | 修法 |
|---|---|---|
| 无置信区间/单次仿真 | 统计不严谨 | 补蒙特卡洛与置信区间 |
| 基线过时或未调优 | 公平性质疑 | 换 SOTA、同等调优 |
| 单场景单信道 | 泛化性存疑 | 补多信道多场景 |
| 缺消融 | 设计必要性不明 | 补模块消融 |
| 只报增益不报代价 | 片面 | 补复杂度/开销 |
| 参数来源不明 | 可疑挑参 | 说明依据 + 敏感性分析 |

## 十、实验设置报告模板

在"实验设置"节用固定结构列全，方便外审复核与他人复现：

```
系统/场景：____（单/多用户、天线配置、频段、带宽、移动速度）
信道模型：____（AWGN/Rayleigh/多径/多普勒 + 参数）
基线方法：____（近年 SOTA，注明来源与调参）
评价指标：____（BER/容量/吞吐/时延/开销/安全性）
仿真设置：蒙特卡洛 ____ 次；随机种子 ____；平台/版本 ____
统计口径：____（置信区间/误差棒/显著性检验）
```

## 十一、结果解读的严谨性

- 只报"提升 X%"不够，须说明**在什么场景、相对什么基线、以什么代价**。
- 区分**统计显著**与**实际显著**：微小但显著的增益要说明工程意义。
- 出现反例或不利场景时**如实报告**并分析原因，比掩盖更能取信外审。
- 曲线交叉/转折点要给出机理解释，而非只描述现象。

## 十二、自查清单

- [ ] 每个 RQ 有对应实验与明确结论。
- [ ] 信道/拓扑/威胁模型与参数完整可复现。
- [ ] 基线为近年 SOTA、同场景同调优。
- [ ] 主结果含蒙特卡洛置信区间/误差棒。
- [ ] 有消融与敏感性分析；报告性能与代价两侧。
- [ ] 实验设置节按模板列全，可复现。
- [ ] 结果解读严谨，交代场景/基线/代价与机理。
- [ ] 图表规范、编号连续、被正文引用。

> 实验定稿后，用 `jonc-reproducibility` 固化参数与代码，用 `jonc-supplementary` 切分正文与附录。
> 目标是让《通信学报》/Journal on Communications (JOC) 外审信服本文证据在通信学科意义上充分、公平、
> 可复现。
