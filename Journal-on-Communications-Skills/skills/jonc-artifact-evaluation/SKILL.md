# 《通信学报》代码与数据可用性

本技能澄清《通信学报》(Journal on Communications, JOC) 在"制品/工件评审"上的**现状**，并给出稳妥
做法。JOC 由中国通信学会主办，是通信学科中文旗舰月刊，走**初审→外审→终审**三审制。

## 一、关键现状（勿误称）

- 据 2026-07-09 核验（见 `resources/official-source-map.md`），本刊**未见**设立独立于正文评审的
  正式"制品徽章"(artifact badge / artifact evaluation) 制度——这与部分国际会议（如 ACM 的
  Artifact Evaluation）不同。
- 因此**不要**在稿件或答复中声称本刊会颁发"可复现""可用"等徽章；是否有相关要求以投稿系统内当期
  规定为准（**待核实**）。
- 但"代码/数据可用"仍是外审判断通信实验可信度的重要加分项，值得主动做好。

## 二、在无正式徽章下的稳妥做法

| 做法 | 说明 |
|---|---|
| 开放复现包 | 提供 config、脚本、种子、信道实现（见 `jonc-reproducibility`） |
| 数据可用性声明 | 在正文/致谢说明数据来源、开放程度与获取方式 |
| 匿名评审仓库 | 外审阶段用匿名链接，避免泄露身份 |
| 参数与信道公开 | 公开仿真参数表与信道模型实现，便于外审复核 |
| 校验和与版本 | 数据给校验和、软件给版本，保证可重建 |

## 三、数据可用性声明模板

```
数据可用性：本文仿真代码与参数配置已公开于 ____（或"可向通信作者索取"）。
所用数据集 ____ 为公开数据（来源 ____，版本 ____）/ 自建数据（生成脚本见 ____）。
受限测量数据因 ____ 无法公开，可提供脱敏样例用于复核。
```

## 四、通信学科的可用性要点

- **物理层/信号处理**：公开信道生成、调制/编码实现、蒙特卡洛脚本。
- **网络与交换**：公开仿真配置（NS-3/OMNeT++ 脚本）、拓扑与流量模型。
- **信息安全与密码**：公开协议实现与验证脚本（ProVerif/Tamarin）；敏感部分脱敏。

## 五、与三审制的衔接

- **初审**：确保数据/代码声明不违反学术规范（无伪造、无侵权）。
- **外审**：外审可能索要复现细节；备好复现包提升说服力。
- **终审/录用**：按编辑部要求补充数据声明或授权（**待核实** 具体栏目）。

## 六、注意事项

| 事项 | 说明 |
|---|---|
| 不误称徽章 | 本刊无官方制品徽章，勿声称获得 |
| 许可合规 | 公开代码数据前确认许可与第三方权利 |
| 隐私脱敏 | 涉用户/网络测量数据须脱敏 |
| 以当期规定为准 | 是否强制提交代码/数据 **待核实** |

## 七、与国际会议 AE 制度的对照（帮助建立正确预期）

许多作者从国际会议投稿迁移而来，容易把会议的"制品评审"经验错误套用到本刊。下表帮助厘清差异：

| 维度 | 部分国际会议（如 ACM AE） | 《通信学报》/ JOC 现状 |
|---|---|---|
| 是否独立评审 | 有独立 Artifact Evaluation 委员会 | 未见独立制品评审（**待核实**） |
| 是否颁徽章 | 颁 Available/Functional/Reproduced 等徽章 | 未见官方徽章制度 |
| 提交时点 | 常在论文录用后单独提交制品 | 无独立环节；随正文评审 |
| 对录用影响 | 徽章独立于录用 | 代码/数据可用性作为外审可信度参考 |

结论：把精力放在**实质可复现**（复现包质量、参数与信道公开）而非"拿徽章"，才符合本刊评审逻辑。

## 八、可用性成熟度自评（从低到高）

| 级别 | 特征 | 建议 |
|---|---|---|
| L0 | 无任何代码/数据说明 | 至少补数据来源与可用性声明 |
| L1 | 声明"可向作者索取" | 明确联系方式与可提供范围 |
| L2 | 提供部分脚本/参数 | 补全 config、种子、信道实现 |
| L3 | 完整复现包，可一键复现主图 | 本包推荐目标（见 `jonc-reproducibility`） |
| L4 | L3 + 公开数据集 + 长期可访问 | 通信实证工作的理想状态 |

## 九、常见疑问

- **问：本刊要求必须开源吗？** 以投稿系统当期规定为准（**待核实**）；即使不强制，开放仍加分。
- **问：涉密或商用数据怎么办？** 说明受限原因、提供脱敏样例或可公开的替代验证。
- **问：匿名评审如何给链接？** 用匿名仓库（如匿名化的托管服务），终稿再换正式链接。

## 十、自查清单

- [ ] 未在任何材料中声称本刊颁发制品徽章。
- [ ] 提供复现包或明确的数据/代码获取方式（目标 L3 及以上）。
- [ ] 数据可用性声明写入正文/致谢。
- [ ] 外审用匿名仓库（如适用）。
- [ ] 公开内容许可合规、敏感数据脱敏。
- [ ] 已对照国际会议 AE 差异，建立对本刊的正确预期。
- [ ] 以投稿系统当期要求核对是否需额外提交（**待核实**）。

## 十点五、数据合规与第三方权利

通信实验常用到第三方数据集、信道测量或开源代码，公开前需核查权利与许可：

| 检查项 | 说明 |
|---|---|
| 数据集许可 | 是否允许再分发；标注来源与许可类型 |
| 网络/用户测量 | 是否脱敏、是否获授权、是否涉隐私 |
| 第三方代码 | 遵守其开源许可（署名、传染性条款） |
| 商用/涉密 | 不得擅自公开；用可公开替代验证 |

合规是学术规范的一部分，初审也会关注数据来源的正当性。

## 十一、要点回顾

- 本刊为中文通信期刊，**未见独立制品徽章制度**，勿声称获得徽章（**待核实** 当期规定）。
- 代码/数据可用性虽非强制，却是外审判断通信实验可信度的重要参考。
- 稳妥做法：开放复现包、写清数据可用性声明、匿名评审用匿名仓库、公开信道与参数。
- 目标成熟度为 L3 及以上（可一键复现主图），配合 `jonc-reproducibility` 落地。
- 涉密/受限数据以脱敏样例或替代验证兼顾开放与合规。

> 目标：在《通信学报》/Journal on Communications (JOC) 没有正式制品评审的前提下，用开放、规范的
> 可用性材料为通信实验的可信度加分，同时不误导读者以为本刊有官方徽章制度。配合 `jonc-reproducibility`
> 落地复现包，配合 `jonc-supplementary` 安排材料归属，配合 `jonc-submission` 在投稿时给出可用性声明。
