# 复现包与仿真代码组织 · 《通信学报》(Journal on Communications, JOC)

本目录给出向《通信学报》(Journal on Communications, JOC) 投稿时，复现包与仿真代码的组织建议。JOC
无独立的"制品徽章"评审制度（见 `jonc-artifact-evaluation`），但提供清晰可复现的代码/数据能显著提升
外审对通信实验可信度的评价。

## 推荐结构

```
repo/
├── README.md              # 环境、依赖、一键复现主图表的命令
├── env/                   # requirements.txt / environment.yml / MATLAB 版本
├── src/                   # 仿真与算法源码
│   ├── channel/           # 信道模型（AWGN/Rayleigh/多径/多普勒）
│   ├── methods/           # 本文方法与基线
│   └── metrics/           # BER/吞吐/时延/开销统计
├── configs/               # 每个实验的参数表（YAML/JSON），含随机种子
├── data/                  # 数据集或生成脚本（大数据给下载脚本+校验和）
├── scripts/               # run_fig3.sh 等，与论文图表编号对应
└── results/               # 输出曲线与原始日志（可选）
```

## 关键约定

- **参数可追溯**：每张主图对应一个 config，写清 SNR 范围、码率、天线数、节点数、蒙特卡洛次数、
  随机种子。
- **图表可回溯**：脚本命名与论文图号一致（如 `run_fig4_ber.py`），README 列"图 X ← 命令"。
- **数据可获取**：公开数据给来源与版本；自建数据给生成脚本与校验和；受限数据说明获取方式与
  **待核实** 的许可限制。
- **随机性可控**：固定种子；报告多次运行的均值与置信区间，而非单次结果。

## 与投稿的衔接

- 在正文"实验设置"节引用复现包地址（匿名评审阶段用匿名仓库）。
- 在数据可用性/致谢处说明代码与数据的开放程度。
- 具体是否要求提交代码、是否有 ORCID/数据声明栏，以投稿系统当期要求为准（**待核实**）。

> 目标：让《通信学报》/Journal on Communications (JOC) 的外审能按 README 在声明环境下复现主结果，
> 这是通信类实证工作可信度的底线。
