# 选题与对口判断（cjms-topic-selection）

## 触发时机

- 开题或改投时问"这个题目《中国管理科学》要不要"
- 稿件核心是问卷量表、案例或政策评估回归，怀疑赛道错配
- 不清楚该对位哪个栏目、按哪条主线组织稿件

## 核心：选题三问

本刊沿承华罗庚创刊传统——**用定量方法解真实管理问题**。选题必须同时过三问：

1. **管理实践真问题**：问题来自可指认的中国管理情境（供应链、能源市场、应急、平台、项目、医疗运营……），而不是从模型倒推出来的习题。
2. **方法有增量**：相对已有文献，模型设定、算法、预测组合或度量方法至少有一处可命名的改进；"把 A 方法用到 B 数据"若无适配性改造，增量不足。
3. **结果可落地**：能从求解或实证结果导出决策规律或对策，而非止于统计显著。

## 栏目对位表

| 主要栏目 | 典型选题形态 | 主线 |
|----------|--------------|------|
| 规划与优化 | 运输、选址、调度、鲁棒/随机优化 | 优化建模 |
| 生产与经营管理 | 供应链契约、库存、平台运营、定价 | 优化建模 |
| 项目管理 | 项目调度、资源冲突、工期-费用权衡 | 优化建模 |
| 市场与投资分析 | 投资组合、风险度量、市场相依与传染 | 金融工程实证 |
| 预测与决策 | 能源/碳价/需求预测、组合预测、多准则决策 | 数据驱动预测 |
| 管理信息系统 | 数据智能、信息系统运营决策 | 数据驱动/优化 |

近年热点可参照本刊当期目次与"中国管理科学学术年会"（编辑部参与主办，1999 年起连办二十余届）征文方向：供应链韧性、平台经济、双碳与能源转型、应急管理、数智化运营。

## 姊妹刊分流（投错赛道是第一大退稿源）

| 稿件形态 | 更合适的期刊 |
|----------|--------------|
| 定理-证明链条厚、贡献在数理性质 | 《管理科学学报》 |
| 系统工程方法论、大型工程系统应用 | 《系统工程理论与实践》 |
| 案例研究、问卷-SEM、政策评估叙事 | 《管理世界》《南开管理评论》 |
| 资产定价、货币金融、以经济含义为主 | 《金融研究》 |
| 软科学、科技政策论述 | 《中国软科学》 |

判据一句话：去掉数据后若只剩"文献 + 观点"，不对口；去掉情境后若只剩"定理 + 证明"，考虑《管理科学学报》；本刊要**情境、模型、检验三者都在**。

## 自检清单

- [ ] 一句话说清管理情境中的决策者、决策变量与冲突
- [ ] 指认了对位栏目，且近三年该栏目发过同型选题
- [ ] 方法增量可命名（改进点 ≤ 2 个，但每个都实）
- [ ] 有可得的真实数据或有依据的参数来源支撑检验
- [ ] 已对照姊妹刊分流表，排除赛道错配
- [ ] 预期结论能写成"条件—决策—效果"式的管理启示

## 本刊初审的选题淘汰逻辑

编辑部初审与外审第一轮，按"对口—增量—证据可行"三档快筛。常见淘汰信号与修法：

| 淘汰信号（审稿常用语） | 根因 | 本刊期望的修法 |
|------------------------|------|----------------|
| "选题不属于本刊范围" | 纯定理推演或纯政策论述 | 按分流表改投，或把交付物改造成"模型+检验"闭环 |
| "创新性不足" | 标准模型换个行业名词 | 找出情境的结构性差异（约束/信息/主体关系），让它进模型 |
| "与已有研究区别不明显" | 增量说不出、藏在正文深处 | 引言第三段用一句话命名改进点 |
| "问题的管理意义不清" | 从方法倒推问题，无人决策 | 写出决策者、决策变量与冲突，给可指认的行业事实 |
| "数据/参数可行性存疑" | 检验方案悬空 | 开题即锁定数据来源或参数标定路径 |

## 微型走查：一个选题的三问检验

虚构选题《考虑充电排队的城市电动物流车队调度》：

- **真问题**：调度员在续航与排队时间不确定下决定路线与充电时机——决策者、变量、冲突齐备，且可指认行业数据 → 过。
- **方法增量**：既有车辆路径模型多把充电站等待视为常数；本题把"排队时间依赖到站流量"写进约束，产生新的耦合结构 → 可命名，过。
- **可落地**：预期给出"电价峰谷差—错峰充电比例"的决策参照 → 参数校准型启示，过。
- **栏目对位**：规划与优化；近三年本刊有电动车辆路径与共享出行调度刊文 → 有对话对象。

反例对照：《数字化转型对企业绩效的影响——基于上市公司的实证》三问全不过（无决策变量、无方法增量、启示必然口号化），属《管理世界》系赛道。

## 反模式

- 用"重要性叙事"替代问题定义：宏观背景两页，决策问题一句没有
- 蹭热点但无方法增量：把"双碳"当形容词贴在标准模型上
- 数据可得性倒逼选题，问题本身无人决策、无人受益
- 同一稿件既想讲定理又想讲政策，两头都不达标

## 输出格式

```
【主线判定】优化建模 / 数据驱动预测 / 金融工程实证
【对位栏目】<栏目名>（近三年同型文章：有/待查）
【三问检验】真问题<过/不过> 方法增量<可命名的改进点> 可落地<预期启示形态>
【姊妹刊分流】不适配风险：<无 / 指向某刊及理由>
【下一步】cjms-literature-review
```
