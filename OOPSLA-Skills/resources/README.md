# OOPSLA Resources

Working materials that let an agent do OOPSLA-shaped tasks — model the two-round calendar,
benchmark a draft against verified OOPSLA papers, rewrite an introduction into the venue's
register, and pressure-test an artifact — while keeping every venue fact traceable to an
official source with an access date.

## Directory map

| File | What it is for |
| --- | --- |
| [`official-source-map.md`](official-source-map.md) | The fact backbone: every OOPSLA 2026 / SPLASH 2026 / PACMPL claim in this pack, its URL, its access date, and the explicit 待核实 list. Read first when anything deadline- or policy-shaped is asked. |
| [`external_tools.md`](external_tools.md) | The live systems (SPLASH track page, HotCRP, artifact track, ACM DL) and what to inspect in each before advising an author. |
| [`exemplars/library.md`](exemplars/library.md) | Six dblp/ACM-DL-verified OOPSLA papers spanning the venue's contribution shapes, plus the misattribution cut list. |
| [`worked-examples/01-introduction.md`](worked-examples/01-introduction.md) | A fictional before/after introduction rewrite showing the tool-advertisement-to-design-insight transformation. |
| [`code/README.md`](code/README.md) | Pointer to the shared reproducibility kit with a PL-artifact translation table. |

## How the resources and skills divide labor

The twelve [`../skills/`](../skills/) entries carry judgment: round selection, revision
strategy, response drafting, routing. These resources carry *ground truth and worked
material*. When a skill and a resource seem to disagree about a date or rule, the source map
wins inside the pack, and the live official page wins over both.

## Suggested passes through the pack

1. **Fit pass** — [`../skills/oopsla-topic-selection`](../skills/oopsla-topic-selection/SKILL.md)
   against the exemplars library, to confirm the contribution shape has an OOPSLA precedent.
2. **Evidence pass** — [`../skills/oopsla-experiments`](../skills/oopsla-experiments/SKILL.md)
   and [`../skills/oopsla-reproducibility`](../skills/oopsla-reproducibility/SKILL.md) with
   [`code/README.md`](code/README.md) for the artifact scaffolding.
3. **Prose pass** — [`../skills/oopsla-writing-style`](../skills/oopsla-writing-style/SKILL.md)
   with [`worked-examples/01-introduction.md`](worked-examples/01-introduction.md) open.
4. **Calendar pass** — [`../skills/oopsla-workflow`](../skills/oopsla-workflow/SKILL.md) plus
   [`official-source-map.md`](official-source-map.md) to place the paper in a real round.

## Scope guard

This pack serves the OOPSLA research-paper track only. Onward! Papers, Onward! Essays,
posters, workshops, and the co-located ISSTA program at SPLASH 2026 have their own calls and
their own bars — route to their pages via [`external_tools.md`](external_tools.md) rather
than stretching these skills to cover them.
