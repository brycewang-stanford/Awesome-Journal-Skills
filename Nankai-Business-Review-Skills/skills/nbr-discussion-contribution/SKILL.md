# 讨论与理论贡献（nbr-discussion-contribution）

## 触发时机

- 讨论部分只复述"H1 成立、H2 成立"
- "理论贡献"段落把假设又抄一遍，没说推进了什么
- 引言立的缺口，讨论里没有回扣闭环

## 讨论应回答的三件事

1. **回扣缺口**：引言说某理论欠什么，本文结果如何填补
2. **推进理论**：用一种明确方式推进——
   - **扩展（extension）**：把理论延伸到新构念/新机制/新层级
   - **边界条件（boundary）**：指明理论在何种情境/水平下成立、削弱或反转
   - **整合（integration）**：把两套理论接合，解释原先各自解释不了的现象
3. **意外/不成立的处理**：未支持的假设要给理论解释，而非略过

## 三类贡献的写法对照

| 贡献类型 | 关键句式 |
|----------|----------|
| 扩展 | "本文将 <理论T> 从 <原域> 拓展到 <新域>，揭示 <新机制>。" |
| 边界条件 | "本文表明 <理论T> 的 <效应> 仅在 <条件W> 下成立/反转，廓清其边界。" |
| 整合 | "本文以 <机制M> 接合 <T1> 与 <T2>，解释了二者单独难以解释的 <现象>。" |

## 改写演练：从复述到推进

原稿讨论："本研究发现算法管控显著降低骑手留任意愿，H1–H3 均获支持，丰富了零工经济研究。"——典型的复述加空话。按本刊标准改写：

```
改写前：本文验证了算法管控对留任意愿的负向影响，丰富了相关研究。
改写后：既有工作设计理论默认控制来自人类管理者；本文表明当控制
主体变为算法时，"自主性损失→离职"的机制由心理契约破坏中介，
且仅在平台依赖度高的骑手中成立。这把工作设计理论的适用边界从
"人对人管理"廓清到"算法管理"，并指明其情境条件。
```

改写要点四件套：点名理论（工作设计）、指明贡献类型（边界条件）、说出机制（心理契约破坏）、给出条件（平台依赖度）。

## 结构建议（讨论部分）

- 结果概述（简）→ **理论贡献**（重，对话具体理论）→ **情境贡献**（中国情境进入理论，见 nbr-china-context）→ **实践启示**（适度，勿喧宾夺主）→ **局限与未来研究**（与方法局限对应）

## 本刊讨论层次校准

- 已刊文章的"结论与讨论"普遍以编号小点呈现 2–3 条理论贡献，每条对话一个具体理论，而非合并成一段泛述
- 理论贡献先于管理启示；管理启示面向**企业管理者**而非政策制定者——这是本刊与政策导向期刊（如管理世界）落点上的关键差异
- 摘要中的"贡献"要素须与讨论部分的贡献小点一一对应；篇幅分配以期刊最新投稿指南与近期刊文为准
- 未支持的假设紧跟贡献小点之后处理：给出竞争机制或边界解释，并预告未来研究设计，不要塞进局限里一句带过

## 贡献段的评审追问

| 追问 | 修复 |
|------|------|
| "与已有 XX 研究的区别仅是换了样本" | 指出机制或边界层面的新判断，回 nbr-theory-gap 重立缺口 |
| "写了三点贡献，其实是一点拆三份" | 合并同源贡献：宁要一条扎实的边界条件，不要三条空泛的"丰富" |
| "管理启示太宏观" | 把启示落到可操作的管理动作（招聘、考核、授权设计），删掉口号句 |

## 自检清单

- [ ] 讨论开头回扣引言缺口，形成闭环
- [ ] 贡献明确是扩展 / 边界 / 整合之一，并指名具体理论
- [ ] 说清"改变了我们对什么的理解"，不是复述假设
- [ ] 未支持假设有理论性解释
- [ ] 中国情境贡献与一般化命题并存
- [ ] 实践启示克制，局限与方法对应、不空泛

## 反模式

- "本文验证了 H1–H4"当贡献
- "丰富了 XX 研究 / 提供了新视角"——空话
- 局限只写"样本有限、未来可扩大样本"
- 实践启示堆"加强、完善、推进"口号

## 输出格式

```
【缺口回扣】引言缺口 <…> → 本文如何填补 <…>
【贡献类型】扩展 / 边界条件 / 整合｜对话理论 <T>
【一句话】本文改变了我们对 <…> 的理解
【意外结果】<未支持假设的理论解释>
【情境贡献】<中国情境如何进入理论>
【实践启示】<克制、具体>｜【局限】<与方法对应>
【下一步】nbr-submission
```
