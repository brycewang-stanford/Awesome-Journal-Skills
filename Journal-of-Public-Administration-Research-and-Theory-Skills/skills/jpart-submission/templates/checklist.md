# JPART Pre-Submission Self-Check

Verify all volatile specifics (word/keyword limits, formatting, fees, editorial team, data policy
wording) against the current **Instructions to Authors** on JPART's official page before relying on them.
Portal: **Editorial Express** (`editorialexpress.com/jpart`).

## 1. Fit & contribution

- [ ] Delivers a contribution to **public-management theory** (not a stand-alone policy finding)
- [ ] A named **mechanism** explains why the relationship holds
- [ ] Sits in JPART's core (PSM, red tape, representation, performance, governance, behavioral PA)
- [ ] The PA conversation it advances can be named in one sentence
- [ ] Confirmed JPART fit vs. **PAR** (broad/practitioner), **JPAM** (policy analysis), **Governance**

## 2. Theory & argument

- [ ] Constructs defined and distinguished from neighbors (red tape vs. burden, PSM vs. satisfaction)
- [ ] Mechanism stated explicitly (actors + incentives/identity/institutions)
- [ ] Observable implications derived (what we should and should not see)
- [ ] Scope conditions stated (sector / level of government / task)
- [ ] Theory move clear: extend / test / bound / overturn

## 3. Design & evidence

- [ ] Design justified as following from the question
- [ ] Experiments: preregistered, powered (MDE reported), on the right population (public employees/citizens)
- [ ] Observational: estimand + identifying assumption stated and defended
- [ ] PA-specific threats addressed: **common-method/common-source bias**, **self-selection into public service**
- [ ] Multilevel nesting modeled (clustered SEs / random effects; ICC reported) where relevant
- [ ] Confirmatory vs. exploratory analyses clearly separated

## 4. Exhibits

- [ ] Effects shown with intervals, not stars-only
- [ ] Mechanism visible in an exhibit (mediation / first stage / scope-condition interaction)
- [ ] Every exhibit self-contained (title, units, N, inference, notes)
- [ ] Every exhibit numbered and referenced in text; values match the deposited code

## 5. Writing & craft

- [ ] Theoretical contribution stated by end of the introduction
- [ ] Mechanism named in plain language
- [ ] Abstract follows: theory → method/data → results → implications for theory
- [ ] Single theoretical thread runs through every section; light signposting

## 6. Length, references & keywords

- [ ] Manuscript ≤ ~12,000 words **including abstract, tables, and references**
- [ ] OUP author-date style; reference list alphabetical **with DOIs**
- [ ] Every in-text citation in the list and vice versa
- [ ] 3–5 keywords; first three signal **theory, research theme, and method**

## 7. Anonymization (double-blind)

- [ ] Main document free of author names, affiliations, funding, acknowledgments
- [ ] Self-citations phrased in the third person
- [ ] File metadata/properties scrubbed; file names contain no author names
- [ ] Cover sheet (separate) carries name/funding/acknowledgments
- [ ] Pre-registration report blinded (anonymized hyperlinks) or placed on the cover sheet

## 8. Transparency & editorial system

- [ ] **Data Availability Statement** drafted (what / where / access terms)
- [ ] Data + software code staged for **public release** in a citable repository (or exemption + editor approval)
- [ ] Manuscript uploaded as a single PDF (title, abstract, text, references, tables/figures, appendices)
- [ ] Declarations completed (originality, conflicts, data/ethics as applicable)
- [ ] Confirmed not under review elsewhere
- [ ] Online supplement/appendix uploaded if used
