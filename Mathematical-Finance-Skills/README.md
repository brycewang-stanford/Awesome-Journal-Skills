# Mathematical Finance Skills

Twelve Codex/Claude skills for preparing manuscripts for *Mathematical Finance*,
the Wiley-Blackwell financial-mathematics journal affiliated with the Bachelier
Finance Society.

This pack is theorem-first. It is for rigorous financial mathematics: stochastic
analysis, stochastic calculus, probability, derivative pricing, risk, portfolio
theory, and market microstructure. It is not an empirical-finance template.
Numerical experiments belong here only when they support rigorous theoretical
developments.

## Skills

| Skill | Use it for |
| --- | --- |
| `mathfin-workflow` | Route a paper through the theorem-first lifecycle. |
| `mathfin-topic-selection` | Test whether the problem is a rigorous financial-mathematics fit. |
| `mathfin-literature-positioning` | Position the result against precise prior theorems. |
| `mathfin-identification-strategy` | Audit assumptions, theorem statements, and proof architecture. |
| `mathfin-contribution-framing` | Explain why the theorem matters for financial modelling. |
| `mathfin-data-analysis` | Keep numerical experiments rigorous and subordinate to theory. |
| `mathfin-tables-figures` | Prepare theorem-supporting figures, tables, and appendices. |
| `mathfin-writing-style` | Polish definitions, theorem exposition, proofs, and financial intuition. |
| `mathfin-replication-and-data-policy` | Prepare Wiley data availability and reproducible code notes. |
| `mathfin-review-process` | Understand single-blind editor/AE/referee review. |
| `mathfin-submission` | Run Wiley Research Exchange LaTeX preflight. |
| `mathfin-rebuttal` | Respond to referee concerns about novelty, rigor, and exposition. |

## Typical Flow

Start with `mathfin-workflow`. If the theorem is not yet stable, go to
`mathfin-identification-strategy`. If the mathematics is sound but the paper does
not explain its financial-modelling value, use `mathfin-contribution-framing`.
