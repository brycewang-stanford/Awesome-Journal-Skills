# PiHG — Submission Preflight Checklist

Copy this into your submission notes and tick every box before uploading to **SAGE Track**. Verify volatile
specifics (cap, abstract length, keyword minimum, ORCID requirement, OA/fee, policy wording) on the
official SAGE / PiHG pages first (检索于 2026-07-16；以官网为准).

## 1. Genre & fit
- [ ] The piece is a **critical review / theory essay**, NOT an empirical study or a detailed case
- [ ] Delete-the-data test passed: the spine is an argument about a literature, with no findings section
- [ ] Genre correct: **review article** (open submission) vs. **Progress Report / PiHG Lecture** (invited only)
- [ ] One-line statement of the intervention/contribution for the cover letter

## 2. The argument (why PiHG will want it)
- [ ] There is a synthetic thesis the sources add up to (not an annotated bibliography)
- [ ] A clear critical intervention: what the field gets wrong and how you re-order it
- [ ] A portable, forward-looking agenda (not "more research is needed")
- [ ] "Why this review, why now?" is answered

## 3. Length (cap is INCLUSIVE of endnotes)
- [ ] Review article **~4,000–8,000 words**, **including endnotes**, **excluding bibliography**
- [ ] Word count computed over the whole essay, not just the body
- [ ] Paper-by-paper walk-throughs and padding cut to fit the short cap

## 4. Abstract & keywords
- [ ] Unstructured abstract **~100 words**, stating the intervention (not just the topic)
- [ ] **Minimum 5 keywords**, as specific to the topic as possible

## 5. Anonymity (double-anonymous)
- [ ] No author names, affiliations, funding, or acknowledgments in the manuscript
- [ ] Self-citations neutralized; no "as we argued in…"
- [ ] File metadata stripped (document properties, comments)
- [ ] Separate, non-anonymous title page prepared as a distinct file

## 6. Style & references
- [ ] SAGE Harvard (author–date) referencing, consistent throughout
- [ ] Reference list complete; every in-text citation resolves; attributions precise
- [ ] Coined terms defined on first use

## 7. Conceptual exhibits
- [ ] Each exhibit does synthetic work (typology / framework / genealogy / synthesis table)
- [ ] Claim-bearing captions; grayscale-legible; colorblind-safe; vector where possible
- [ ] Any bibliometric/data graphic labeled as field-evidence, not the argument
- [ ] Exhibit count disciplined for a short review

## 8. Ethics & policy
- [ ] Competing interests / funding disclosed per SAGE policy
- [ ] Generative-AI use disclosed per SAGE policy (if any)
- [ ] Positionality statement where relevant to the argument
- [ ] For a structured/systematic review: search documented (databases, dates, queries) if applicable
- [ ] ORCID ready for the submitting author (verify requirement)

## 9. Final
- [ ] Cover letter states genre, the intervention, and fit with PiHG
- [ ] Files match SAGE Track's requested set
- [ ] Re-read `../SKILL.md` (phg-submission) and `../../phg-review-process/SKILL.md` once more
