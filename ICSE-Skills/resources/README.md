# ICSE Resources

Working material behind the ICSE skill pack. The `skills/` directory tells an
agent what to do; this directory holds what that advice stands on — the
verified official sources, award-certified exemplar papers, a worked
first-page rewrite, and pointers to reusable package-checking tooling.

## Scope note

This is an empirical-software-engineering conference pack. The evidence
culture it encodes — research-question contracts, threats-to-validity
craft, real-subject benchmarks, replication packages as scored review
material — differs from the ML conference packs (no leaderboard framing) and
from the journal packs (no standing editorial board: ICSE re-decides its own
rules, including its cycle count, every edition).

## Suggested route through the pack

1. **Fit first:** [`../skills/icse-topic-selection`](../skills/icse-topic-selection/SKILL.md)
   with the exemplars library open beside it — the routing table and maturity
   ladder are cheapest before any writing exists.
2. **Design the evidence:** [`../skills/icse-experiments`](../skills/icse-experiments/SKILL.md)
   and [`../skills/icse-reproducibility`](../skills/icse-reproducibility/SKILL.md),
   with [`code/README.md`](code/README.md) for package smoke checks — threats
   named at design time can still be mitigated.
3. **Write and package:** [`../skills/icse-writing-style`](../skills/icse-writing-style/SKILL.md)
   against the worked example, then
   [`../skills/icse-supplementary`](../skills/icse-supplementary/SKILL.md) for
   the 10-page/package split and
   [`../skills/icse-submission`](../skills/icse-submission/SKILL.md) for the
   HotCRP audit.
4. **Verify last:** reopen everything in
   [`official-source-map.md`](official-source-map.md) — the 2027 snapshot in
   this pack ages by design, and the cycle structure is the first thing to
   re-check.
