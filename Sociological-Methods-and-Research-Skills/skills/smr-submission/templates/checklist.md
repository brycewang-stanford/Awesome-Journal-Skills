# Sociological Methods & Research — Pre-Submission Checklist

> Verify all volatile specifics (ScholarOne portal, review model, abstract limit, citation style,
> data-and-code policy, AI policy, fees, file formats) on the official SAGE author page before relying
> on this list. Access basis: 检索于 2026-06；以官网为准.

## 1. Methods-paper completeness (the SMR scope gate)

- [ ] The manuscript is a methodological contribution — it develops, evaluates, or critically assesses
      a method — not an application of an existing method to new data
- [ ] The contribution sentence states the method and the problem it solves vs. existing methods
- [ ] Analytical properties (bias / consistency / efficiency / coverage) are derived or argued, with conditions
- [ ] A Monte Carlo simulation includes the real competing methods and reports the matching metrics
- [ ] A real-data empirical illustration shows the method changes a substantive conclusion
- [ ] Usable, released software/code accompanies the paper

## 2. Double-anonymized review

- [ ] A separate title page carries all author identity, affiliations, and acknowledgments
- [ ] The main document, exhibits, and code contain no author-identifying content
- [ ] PDF/file metadata and repository metadata are stripped of author identity
- [ ] No "in our previous work" / institutional file paths remain in the body

## 3. Abstract and front matter

- [ ] Abstract is ≤150 words
- [ ] Abstract contains no parenthetical citations (non-parenthetical "Smith (2015)" is allowed)
- [ ] Abstract names the method, the problem it solves, and the evidence types
- [ ] Keywords entered in ScholarOne match the manuscript

## 4. Style and references

- [ ] In-text citations and references follow ASA style
- [ ] Dataset references follow DataCite style
- [ ] Notation is consistent across equations, tables, and figures
- [ ] Equations are numbered; estimand and estimator are visually distinct

## 5. Exhibits

- [ ] The simulation results grid compares methods within each cell and marks the headline + boundary cells
- [ ] Monte Carlo precision (replication count / standard error) is reported
- [ ] Figures/tables are self-contained (notes define abbreviations and Monte Carlo details)
- [ ] File formats and resolution meet the live SAGE requirements

## 6. Data, code, and reproducibility

- [ ] A data-and-code availability statement describes how data, code, and materials can be permanently accessed
- [ ] Code and materials are deposited in a trusted public repository (with DOI/URL)
- [ ] Restricted data state the reason and access conditions in the statement
- [ ] A master script with fixed seeds reproduces the tables, figures, and simulation
- [ ] An exhibit→script map lets a replicator find each result
- [ ] The review-copy materials are anonymized (blinded archive); public DOI revealed at acceptance

## 7. Disclosures and logistics

- [ ] Generative-AI tool use is disclosed in the back matter (none needed if no AI tool was used)
- [ ] Conflict-of-interest declaration included
- [ ] Funding statement prepared (required at acceptance stage)
- [ ] Preregistration statement included if applicable
- [ ] ORCID self-linked in ScholarOne (strongly encouraged)
- [ ] File format is Word, LaTeX, or PDF; all coauthors approved the final version
