# 《计算机辅助设计与图形学学报》审稿意见答复与修回说明

本技能帮助为《计算机辅助设计与图形学学报》(Journal of Computer-Aided Design & Computer
Graphics, JCAD&CG) 撰写修回材料。本刊是 CCF A 类图形学/CAD 方向中文月刊，走同行评议与多轮
修回；修回质量直接决定复审与主编终审结果。核验见 `resources/official-source-map.md`
（2026-07-09）。先用 `jcadcg-review-process` 判断当前轮次(大修/小修)。

## 一、修回材料的两份文件

1. **点对点答复(Response Letter)**：逐条列出审稿意见 → 本文回应 → 对应修改处(章节/图/表)。
2. **修回说明**：概述本轮主要改动(新增实验、重写章节、补图形证据)，方便编辑与主编快速把握。

## 二、点对点答复的写法

对**每一条**意见(含编辑与各外审)：

```
【审稿人一 · 意见3】质疑基线不公平：本文与基线面片预算不同。
【回应】感谢指出。我们已统一面片预算重做对比(表2)，并在4.1节说明设置；
       结论不变，本方法在相同预算下 Hausdorff 误差仍低 x%。
【修改处】正文4.1节第2段、表2、图4新增；见修订稿标黄。
```

- **逐条、不合并、不遗漏**：即使不采纳也要回应并给理由。
- **用证据回应**：图形学质疑多需**新增图形证据**(渲染对比、几何误差色图、消融、开销、用户
  研究)，而非仅文字辩解。
- **标注定位**：明确修改在正文何处，便于复审核对(可用标色/行号)。

## 三、大修 vs 小修策略

| 类型 | 策略 |
|---|---|
| 大修(常重新送审) | 把每条意见当作必须闭环的任务；补关键实验、重写薄弱章节；答复要让**原审稿人**信服 |
| 小修 | 精准修正细节(体例、表述、个别数据)，答复简洁；避免引入新问题 |

## 四、不采纳意见的处理

- 可以据理不采纳，但要**礼貌、有据**：说明为何该建议不适用(如超出本文范围、与本刊定位不符、
  证据不支持)，并尽量给替代方案(如在局限中讨论)。
- 不与审稿人争辩语气；把分歧转化为对读者更清楚的解释。

## 五、图形学修回常见质疑与应对

| 质疑 | 应对 |
|---|---|
| 只有指标无视觉对比 | 补渲染对比图、放大关键差异区域 |
| 基线不公平/不够强 | 统一预算、纳入更强基线重测 |
| 缺消融/贡献归因不清 | 补逐模块消融 |
| 泛化性/鲁棒性存疑 | 补更多模型/数据集、失败案例 |
| 开销未报告 | 补时间/显存/可扩展性 |
| 用户研究不足 | 补被试、统计口径、伦理说明 |

## 六、修回自查清单

- [ ] 每条意见(编辑+各外审)都有独立回应，无遗漏。
- [ ] 承诺的修改在正文真的落地，并标注定位。
- [ ] 图形类质疑用新增图形证据回应，口径与正文一致。
- [ ] 不采纳项礼貌据理说明并给替代方案。
- [ ] 修回说明概述主要改动，主编可快速把握。
- [ ] 修订稿匿名合规(双盲)；在系统时限内提交。

## 七、常见修回失误对照

| 失误 | 后果 | 修正 |
|---|---|---|
| 合并/跳过意见 | 复审判定敷衍 | 逐条独立回应 |
| 只辩解不改稿 | 质疑未闭环 | 用修改+证据回应 |
| 修改无定位 | 复审找不到 | 标章节/图表/行号 |
| 语气对抗 | 观感差 | 礼貌、就事论事 |
| 引入新错误 | 反复修回 | 改后整体自查 |
| 逾期提交 | 按新稿处理风险 | 守时限 |

## 八、修回说明信的结构（模板）

修回说明置于答复文件开头，让编辑与主编在读逐条答复前把握全局：

```
尊敬的编辑、各位审稿专家：
  感谢对本文《<题名>》的评审意见。我们已按意见完成修改，主要改动如下：
  1. 针对基线公平性(审稿人一意见3、审稿人二意见1)，统一面片预算重做全部对比(表2、图4)。
  2. 新增消融实验验证特征项与穿透检测的独立贡献(表4)。
  3. 补充时间与显存开销、失败案例与局限讨论(表5、4.4节)。
  4. 按 GB/T 7714 订正参考文献 X 条，补中文文献英文对照。
  下文对每条意见逐条回应，修改处在修订稿中以<标色/行号>标示。
```

## 九、答复语气与"证据优先"原则

- 图形学外审的多数质疑要靠**新增图形证据**平息，而非语言辩解：能补渲染对比就补图，能补
  几何误差色图就补图，能补用户研究就补数据。
- 语气始终**致谢、克制、就事论事**；把"审稿人误解"表述为"我们此前未讲清，现已在 X 节澄清"，
  避免指责。
- 对相互冲突的两位审稿意见，说明本文的权衡取舍并给依据，而非机械二选一。

## 十、修回时间管理与版本控制

| 动作 | 要点 |
|---|---|
| 建意见台账 | 把编辑与各外审意见拆成带编号的条目表，标注负责人与状态(待办/进行/完成) |
| 冻结改动范围 | 只做意见要求 + 必要连带修正，避免顺手引入未经审阅的新内容 |
| 修订稿标色 | 用统一标色/行号标出改动，方便复审逐条核对 |
| 匿名复核 | 修订稿与新增图/视频仍需匿名(双盲) |
| 时限提交 | 在系统要求时限内提交；逾期风险见 `jcadcg-review-process` |

## 十一、多轮修回的心态与策略

- 大修常需**重新送审**给原审稿人，故本轮要让原审稿人信服，而非只让编辑满意。
- 第二轮若仍有意见，聚焦"上一轮未完全闭环"处，避免反复拉锯；必要时用简短表格展示"意见→
  上一轮回应→本轮进一步改动"。
- 若某意见确实无法满足(如超出本文范围)，在局限中诚实讨论并说明后续工作方向。

## 十二、输出格式

```text
[JCAD&CG 修回] 就绪 / 需完善（轮次：大修/小修）
[覆盖度] 意见总数__，已逐条回应__，遗漏__
[证据回应] 需补图形证据的质疑__条，已补__条（渲染/几何/消融/开销/用户研究）
[定位标注] 修改处均标注章节/图表？（是/否）
[不采纳项] __条，均礼貌据理 + 替代方案？（是/否）
[匿名/时限] 修订稿匿名合规、系统时限内提交？（是/否）
[待办] <逐条列出未闭环意见>
```
