# 《计算机辅助设计与图形学学报》代码与数据可用性

本技能帮助为《计算机辅助设计与图形学学报》(Journal of Computer-Aided Design & Computer
Graphics, JCAD&CG) 准备制品(代码、数据、网格、模型)与**诚实的可用性声明**。本刊是 CCF A 类
图形学/CAD 方向中文月刊。核验见 `resources/official-source-map.md`（2026-07-09）。

## 一、政策现状（务必按当期核实）

- 本刊是否设**正式的制品评审/artifact evaluation 徽章**、是否有**强制代码/数据可用性**
  要求，均属 **待核实**（以官网当期《投稿须知》为准）。
- 与英文会议(如带 AE 徽章的 ACM 会议)不同，中文期刊多以**同行评议**为主，制品公开常为
  鼓励/加分项而非硬性门槛。**不要臆造本刊有徽章制度。**
- 无论政策如何，公开可复现制品都会增强外审信任、提升论文影响力。本技能给出自律标准。

## 二、图形学制品应包含什么

| 类型 | 内容 |
|---|---|
| 代码 | 方法实现、训练/推理脚本、渲染与几何处理脚本 |
| 数据 | 网格/点云/纹理/图像/视频数据集(或获取方式) |
| 模型 | 预训练权重/参数(若适用) |
| 复现 | run_all 脚本、配置(种子/渲染参数)、指标计算脚本 |
| 结果 | 论文表图对应的渲染帧、几何误差、演示视频 |
| 文档 | README(硬件/依赖/一键复现/耗时)、许可证 |

## 三、双盲外审下的匿名化

- 制品在外审阶段也要**去作者身份**：清理仓库用户名、邮箱、机构路径、致谢、基金显名、图源
  与**视频水印**。
- 用匿名托管(如匿名化的仓库镜像/归档)提供，勿直接给带真实身份的个人主页链接。
- 录用后再补真实链接与致谢(见 `jcadcg-camera-ready`)。

## 四、可用性声明的写法

- 在正文(常见于结尾或脚注)写明：提供了什么(代码/数据/模型/视频)、如何获取、许可与版本。
- **诚实**：只声明真实提供的部分；受限数据说明原因(隐私/版权/规模)与**替代核验途径**(子集/
  合成数据/复现说明)。
- 声明与实际提供的制品**一致**，不夸大(如声称"全部开源"却缺关键脚本)。

## 五、制品自查清单

- [ ] 干净环境能一键复现主结果(见 `jcadcg-reproducibility`)。
- [ ] 代码/数据/模型/渲染脚本齐全或给出稳定获取途径。
- [ ] 大数据用可分配 DOI 的归档或 release，固定版本。
- [ ] 外审阶段制品已匿名化(含视频水印、仓库身份痕迹)。
- [ ] 可用性声明与实际一致，受限项说明替代途径。
- [ ] 第三方数据/代码许可合规、注明出处。

## 六、常见问题对照

| 问题 | 后果 | 修正 |
|---|---|---|
| 臆造本刊有 AE 徽章 | 误导作者 | 标注政策**待核实**，以官网为准 |
| 声明"全开源"却缺脚本 | 诚信风险 | 声明与实际一致 |
| 制品泄露身份(视频/路径) | 违反双盲 | 全面匿名化 |
| 大数据无法获取 | 不可复现 | 稳定托管 + 固定版本 |
| 第三方数据无许可说明 | 合规风险 | 注明出处、版本与许可 |

## 七、与其他技能衔接

- 复现细节 → `jcadcg-reproducibility`；补充材料/视频归属 → `jcadcg-supplementary`。
- 录用后补真实链接/致谢 → `jcadcg-camera-ready`；投稿前匿名核查 → `jcadcg-submission`。

## 八、图形学制品的目录组织（建议）

图形学/CAD 论文的制品往往含大体积二进制资源(网格、纹理、视频、模型)，建议按用途分层组织，
让审阅者能快速定位与运行：

```
artifact/
  README.md          # 一键运行、硬件/依赖、与论文表图的对应关系
  LICENSE            # 许可证(代码与数据分别声明)
  src/               # 方法与渲染/几何处理实现
  scripts/run_all.sh # 从零复现主结果
  configs/           # 种子、渲染参数、实验配置
  data/README.md     # 网格/点云/纹理/视频获取方式(稳定托管)
  models/            # 预训练权重(或下载脚本)
  results/           # 渲染帧、几何误差、指标、演示视频
```

## 九、可用性声明写作模板（示例措辞）

在正文可用性声明中，用一句到几句话把"提供了什么、怎么获取、什么受限"讲清，例如：

> 本文的方法实现、训练与渲染脚本、以及用于复现主结果的配置见 <稳定托管链接/DOI>(许可证:
> <MIT/CC-BY 等>)。所用公开数据集 <名称/版本> 按其原始许可获取，获取方式见 data/README。
> 因 <版权/隐私/规模> 原因，<某数据/资源> 不便公开，我们提供 <子集/合成数据/复现说明> 供
> 核验。

- 外审阶段把上述链接替换为**匿名托管**，录用后再换正式链接(见 `jcadcg-camera-ready`)。

## 十、制品评审与本刊定位的关系（避免误用英文会议范式）

- 英文会议的 artifact evaluation(AE) 徽章、Available/Functional/Reusable 分级是**会议机制**，
  **不要**直接假设本刊照搬。本刊以中文期刊同行评议为主，制品公开更多是**增信与影响力**手段。
- 因此本技能的落点是"**无论政策如何都值得做的自律**"，而非追逐某个徽章；政策具体要求以官网
  当期《投稿须知》为准，标 **待核实**。
- 若确需引用本刊制品政策，先在 `resources/official-source-map.md` 双源核实，未确认者不写入
  面向作者的确定结论。

## 十一、许可与合规提醒

- 代码与数据**分别声明许可**：代码常用 MIT/Apache-2.0，数据/模型常用 CC-BY 等；混用需注意
  兼容性。
- 使用第三方数据集/模型时遵守其原始许可，注明出处、版本与许可类型，不擅自二次分发受限资源。
- 含人相关数据(如用户研究素材)须去标识并符合伦理与知情同意，方可随制品提供。

## 十二、输出格式

```text
[JCAD&CG 制品] 就绪 / 需完善 / 政策待核实
[政策] 本刊是否强制可用性/有无 AE 徽章：待核实（以官网为准）
[内容] 代码__ 数据__ 模型__ 渲染脚本__ 结果/视频__（齐否）
[托管] 稳定托管 + 固定版本？（是/否）
[匿名化] 外审阶段制品去身份(含视频水印/路径)？（是/否）
[声明] 可用性声明与实际一致、受限项说明替代途径？（是/否）
[待办] <逐条列出需补的制品/声明项>
```
