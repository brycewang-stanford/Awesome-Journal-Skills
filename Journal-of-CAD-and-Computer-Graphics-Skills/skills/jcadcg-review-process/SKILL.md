# 《计算机辅助设计与图形学学报》审稿流程

本技能说明《计算机辅助设计与图形学学报》(Journal of Computer-Aided Design & Computer
Graphics, JCAD&CG) 的审稿链路，帮助你在投稿后定位稿件所处环节并规划应对。本刊是 CCF A 类
图形学/CAD 方向中文月刊，以**同行评议**为基础。以下流程于 2026-07-09 通过官网、中科院计算所
页与 CNKI 交叉核验，具体**周期、录用率**属经验值/**待核实**，以编辑部当期口径为准
（见 `resources/official-source-map.md`）。

## 一、总体链路

```
投稿(官网 jcad.cn 在线系统)
   → 编辑初审(宗旨契合、创新性、学术规范、格式)
   → 专家外审(双盲，图形学/CAD 领域同行)
   → 复审 / 退修(大修常重新送审；小修编辑核对)
   → 栏目主编组织编委终审(决定录用与否)
   → 录用 → 定稿校样 → 见刊
```

## 二、各环节要点

| 环节 | 谁在做 | 关注什么 |
|---|---|---|
| 编辑初审 | 编辑部/责任编辑 | 是否属图形/CAD 版图、创新性、格式六件套、查重、一稿多投 |
| 专家外审 | 领域同行(双盲) | 图形/几何贡献、证据充分性、公平比较、可复现、写作 |
| 复审退修 | 作者 + 外审 | 大修按意见改并**重新送审**；小修编辑核对 |
| 主编终审 | 栏目主编/编委会 | 综合外审意见与修回，决定录用/退稿 |

- **综述稿**：通过初审后一般由 **3~4 名专家**审阅(经验值)，对系统性要求更高。
- 外审为**双盲**：稿件与制品需匿名(见 `jcadcg-submission`、`jcadcg-artifact-evaluation`)。

## 三、常见初审退稿理由

- 不符宗旨（非图形/几何/可视化/CAD 方向）。
- 无新意、重复前人工作、创新点模糊。
- 图形证据单薄（只有指标无视觉对比、基线不公平）。
- 学术规范问题（查重超标、一稿多投、会议扩展版未声明）。
- 体例不合（缺中英文摘要/关键词/中图分类号/GB/T 7714）。

## 四、修回轮次

| 决定 | 含义 | 应对 |
|---|---|---|
| 大修(重大修改) | 认可方向但证据/写作需实质改进，常重新送审 | 逐条答复 + 补实验/证据，见 `jcadcg-author-response` |
| 小修(minor) | 基本认可，细节修正 | 精准修正，附修回说明 |
| 退稿 | 不达录用标准或不符宗旨 | 按意见重构或改投更对口刊 |

- 大修不是录用承诺；把每条意见当作必须闭环的任务。
- 修回请在系统要求时限内提交，逾期可能按新稿处理(**具体规则待核实**)。

## 五、状态跟踪与沟通

- 在**官网在线系统**跟踪稿件处理流程与当前状态；按编辑部要求阅读意见并上传修改稿。
- 长时间无进展可礼貌邮件(`jcad@ict.ac.cn`)询问，避免频繁催审。
- 尊重双盲：不试图联系或猜测审稿人身份。

## 六、时间线预期（经验值，需核实）

- 具体审稿周期、各轮时长、录用率**均属 待核实**，个案差异大；请以系统状态与编辑部通知为准，
  不以二手数字为定论。
- 规划投稿节奏时预留多轮修回时间，尤其大修重新送审。

## 七、自查/应对清单

- [ ] 投稿前已过初审关口(宗旨、创新、格式、查重、匿名)。
- [ ] 理解当前状态(初审/外审/退修/终审)并知道下一步动作。
- [ ] 收到意见后逐条建表、区分大修/小修。
- [ ] 修回在时限内提交，附点对点答复与修回说明。
- [ ] 不联系审稿人、不催审过频、不一稿多投。

## 八、常见误区对照

| 误区 | 后果 | 修正 |
|---|---|---|
| 把大修当录用 | 修回敷衍被拒 | 逐条闭环、补证据 |
| 忽视某条意见 | 复审失分 | 每条都答复(含不采纳的理由) |
| 逾期修回 | 可能按新稿处理 | 时限内提交 |
| 催审过频/试探审稿人 | 观感差、违规 | 礼貌少量询问、守双盲 |
| 引用二手周期当承诺 | 预期落空 | 以系统状态为准，标待核实 |

## 九、稿件状态词典（在线系统常见状态解读）

在线系统的状态描述可能因编辑部而略有差异，下表帮助理解各状态大致含义(具体以系统为准)：

| 状态(示意) | 含义 | 作者动作 |
|---|---|---|
| 新投稿/待编辑处理 | 编辑部接收、准备初审 | 等待，核对提交完整 |
| 初审中 | 编辑判断契合/创新/规范/格式 | 无需动作 |
| 外审中/送审 | 已分派同行专家评审 | 等待，勿催审过频 |
| 退修/返修 | 需按意见修改并重新提交 | 用 `jcadcg-author-response` 逐条闭环 |
| 复审中 | 修改稿重新送审或编辑复核 | 等待 |
| 终审中 | 栏目主编/编委会决定 | 等待 |
| 录用/待发表 | 通过终审 | 进入 `jcadcg-camera-ready` |
| 退稿 | 未达标或不符宗旨 | 按意见重构或改投 |

## 十、如何合理与编辑部沟通

- 沟通渠道：在线系统消息与编辑部邮箱 `jcad@ict.ac.cn`、电话 010-62562491 / 010-62600342。
- **催审礼仪**：给足评审时间后再礼貌询问，一次一问、说明稿号与投稿日期；避免频繁催促。
- **状态异常**(如长期无变化)可询问是否需补充材料；不试图打听或联系审稿人身份。

## 十一、退稿后的理性选择

| 情形 | 建议 |
|---|---|
| 因不符宗旨退稿 | 用 `jcadcg-topic-selection` 判更对口刊(如综合刊《计算机学报》)后改投 |
| 因证据不足退稿 | 补图形证据与实验后，考虑再投本刊或对口刊 |
| 因创新性不足退稿 | 回到选题重估 delta，勿原样改投 |
| 收到有价值意见 | 无论去向都据以改进，避免下一投重蹈覆辙 |

- 改投务必**撤回确认**、避免一稿多投；重投他刊时按新刊体例重排(勿留本刊格式痕迹)。

## 十二、输出格式

```text
[JCAD&CG 审稿状态] 初审 / 外审 / 退修(大修|小修) / 终审 / 录用 / 退稿
[本轮动作] <当前该做什么>
[意见清单] 已建表逐条：是/否；大修__条 小修__条
[匿名] 稿件/制品/视频匿名合规？（是/否）
[时限] 修回截止：___（周期为经验值/待核实）
[下一步] <指向 author-response / camera-ready / topic-selection(改投)>
```
