# 《计算机辅助设计与图形学学报》附录与补充材料

本技能帮助为《计算机辅助设计与图形学学报》(Journal of Computer-Aided Design & Computer
Graphics, JCAD&CG) 划分正文与补充材料。本刊是 CCF A 类图形学/CAD 方向中文月刊，图形学论文
常有大量视觉材料(渲染图、演示视频、交互录屏)，如何取舍关乎正文可读性与证据完整性。核验见
`resources/official-source-map.md`（2026-07-09）；本刊对补充材料/视频的**具体规格与硬性
要求**属 **待核实**，以官网当期《投稿须知》为准。

## 一、划分原则：决定性 vs 扩展性

| 归属 | 内容 |
|---|---|
| **正文** | 支撑核心主张的**决定性**证据：主结果表、关键渲染对比、几何误差色图、消融主表、方法关键推导 |
| **附录/补充** | **扩展性**材料：额外数据集结果、更多渲染样例、完整推导、超参表、失败案例集、演示视频、交互录屏 |

判据：**读者只读正文能否确信主张成立**？能，则该证据属正文；补充材料只是"想深入的人再看"。

## 二、图形学特有的补充材料

- **演示视频**：动态结果(动画、时序稳定性、交互)最适合视频呈现；标注时长、关键片段、
  参数设置。
- **交互录屏**：可视化/VR/交互类工作用录屏展示可用性与响应。
- **高分辨率渲染图/网格**：正文放缩略与关键对比，高清大图入补充。
- **额外几何结果**：更多模型上的误差色图、鲁棒性样例。

## 三、正文自足性（关键纪律）

- 正文的主张、方法关键步骤、主结果**不得依赖补充材料才能读懂**；补充材料是加强而非前提。
- 正文引用补充材料时明确指向(如"更多渲染结果见补充视频第 2 段")。
- 审稿人可能不看补充材料，故决定性证据必须在正文。

## 四、格式与匿名化

- 电子附件格式据本刊支持(DOC/PDF/ZIP/RAR，视频压制为常见编码如 H.264/MP4；**具体规格
  待核实**)。
- 补充材料同样**双盲匿名**：去视频水印、去仓库/路径身份、去致谢基金显名。
- 视频体积控制在合理范围，必要时提供稳定托管链接(录用后可换正式链接)。

## 五、自查清单

- [ ] 决定性证据(主表、关键渲染对比、几何误差、消融主表)在正文。
- [ ] 扩展材料(额外结果、完整推导、演示视频)入附录/补充。
- [ ] 正文不依赖补充材料即可读懂主张。
- [ ] 正文对补充材料的引用明确指向。
- [ ] 补充材料/视频已匿名化(水印/路径/致谢)。
- [ ] 附件格式与视频编码合规(按当期《投稿须知》)。

## 六、常见问题对照

| 问题 | 后果 | 修正 |
|---|---|---|
| 关键证据塞补充 | 外审看不到、质疑 | 移回正文 |
| 正文臃肿堆砌样例 | 可读性差 | 扩展样例入补充 |
| 正文依赖视频才成立 | 自足性不足 | 正文补足文字/图表证据 |
| 视频带水印/身份 | 违反双盲 | 匿名化后再提交 |
| 补充材料无指向 | 读者找不到 | 正文明确引用位置 |

## 七、与其他技能衔接

- 何为决定性证据 → `jcadcg-experiments`；制品与视频的可复现 → `jcadcg-reproducibility`、
  `jcadcg-artifact-evaluation`；匿名核查 → `jcadcg-submission`。

## 八、演示视频的组织建议

图形学论文的演示视频常是补充材料的核心，建议按以下结构组织，让审稿人几分钟内看到关键点：

| 片段 | 内容 |
|---|---|
| 开场(可选) | 一句话问题与方法概述(去身份，不出现作者/机构) |
| 主结果 | 动态渲染/动画/时序结果，与论文主图对应 |
| 对比 | 本方法 vs 基线的并排动态对比，突出差异 |
| 交互/可视化 | 交互录屏(可视化/VR/交互类)展示可用性与响应 |
| 鲁棒性/失败 | 更多输入下的表现，含诚实的失败案例 |

- 视频中标注关键参数(如帧率、场景、分辨率)；避免只放"好看"片段而回避弱项。

## 九、按材料类型的归属决策表

| 材料 | 正文 | 附录/补充 | 判据 |
|---|---|---|---|
| 主结果表 | ✔ | | 支撑核心主张 |
| 关键渲染对比 | ✔ | | 决定读者是否信服 |
| 几何误差主图 | ✔ | | 量化核心质量 |
| 消融主表 | ✔ | | 贡献归因 |
| 完整数学推导 | 关键步骤 | 完整过程 | 正文留必要步骤 |
| 超参数全表 | | ✔ | 可复现细节 |
| 更多数据集样例 | 代表若干 | 其余 | 避免正文臃肿 |
| 演示视频/录屏 | | ✔ | 动态证据外置 |
| 失败案例集 | 代表 1~2 | 其余 | 诚实但不喧宾夺主 |

## 十、补充材料与可复现/制品的关系

- 补充材料中的**演示视频、额外结果**应能被复现包重现，口径与正文一致(见
  `jcadcg-reproducibility`)。
- 若补充材料含代码/数据链接，遵守 `jcadcg-artifact-evaluation` 的匿名与诚实声明要求。
- 补充材料不是"倾倒未整理内容"的地方：仍需组织清晰、有指向、可被引用定位。

## 十一、常见规格与容量注意

- 附件总体积、单文件大小、视频时长/编码等**具体规格属待核实**，以官网当期《投稿须知》与
  投稿系统提示为准；超限时改用稳定托管链接并在正文引用。
- 提交前再次确认补充材料**无身份泄露**(视频水印、仓库路径、致谢基金显名)。

## 十二、输出格式

```text
[JCAD&CG 补充材料] 就绪 / 需调整
[正文决定性] 主表/关键渲染对比/几何误差/消融主表 在正文？（是/否）
[补充扩展性] 额外结果/完整推导/演示视频/交互录屏 已外置？（是/否）
[自足性] 正文不依赖补充材料可懂主张？（是/否）
[引用] 正文对补充材料指向明确？（是/否）
[匿名/格式] 补充材料去身份 + 编码/格式合规？（是/否 / 待核实规格）
[待办] <逐条列出需移动/匿名/补充项>
```
